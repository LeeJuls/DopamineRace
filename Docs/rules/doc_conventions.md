# DopamineRace 문서 규칙 (상세)

> 파일명 규칙: `CLAUDE.md` 참조. 이 파일은 내용·형식 상세 가이드.

## SPEC 명세서 필수 섹션
1. **배경** — 왜 필요한가 (1~2문장)
2. **요구사항** — R1, R2... 번호 형식, 각 1줄
3. **설계** — 클래스/계층 구조 (ASCII 다이어그램 또는 표)
4. **스프린트 분해** — Sprint 0→N, 각 완료 조건 명시
5. **QA 체크리스트** — TC 표 (아래 형식)
6. **수정 파일 목록** — `파일 | 변경` 표 형식

### QA 체크리스트 표 형식
| TC | 내용 | Pass 기준 |
|----|------|----------|
| TC-S0-01 | StringTable 키 존재 확인 | Loc.Get() 반환값 비어있지 않음 |
| TC-S1-01 | 컴파일 에러 0건 | MCP 콘솔 에러 없음 |

## HIST 히스토리 필수 섹션
1. **작업 개요** — 한 줄 요약 + 날짜/브랜치
2. **완료 작업** — Sprint/Phase별 변경 내용 (불릿)
3. **검증 결과** — QA 결과 체크리스트 (✅/❌)
4. **수정 파일 목록** — `파일 | 변경` 표
5. **비고** — 수동 작업, 알려진 이슈

## 작성 금지 사항
- SPEC에 구현 코드 삽입 금지 (의사코드는 허용)
- HIST에 미확인 결과 ✅ 표시 금지
- 날짜·파일명 불일치 금지 (파일명 = 실제 작업 날짜)
