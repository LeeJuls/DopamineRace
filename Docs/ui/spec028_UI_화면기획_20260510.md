## U1. 베팅 화면 헤더 — 통화 표시 (총점 영역 교체)

### 변경 요약 (UC1·UC9 반영)

기존 베팅 화면 **최상단 가로 헤더 바**의 `총점` / `획득 포인트` 영역을 **🟦 젤리 X / 💎 스톤 Y** 표시로 **교체**. 어차피 새 룰에서 "총점"은 의미가 없어지므로 제거하고 통화 표시로 대체. 베팅 타입별 라벨도 "획득 포인트" → "획득 도파민 스톤(💎)"으로 변경 (UC10).

### 와이어프레임 (Before / After)

**Before (현재 — `총점` / `획득 포인트` 의미 없어짐)**:
```
┌──────────────────────────────────────────────────┐
│ 1라운드   배팅 선택   배당률: 8배   1착 맞추기    │
│           [단승] [연승] [복승] [쌍승] ...          │
│  ☑ 캐릭터 정보 안 보기   총점:0   획득 포인트     │
│                          ↑      ↑                │
│              의미 없어짐       삼복승 획득 포인트  │
└──────────────────────────────────────────────────┘
```

**After (UC9·UC10 적용)**:
```
┌────────────────────────────────────────────────────┐
│ 1라운드 / 4   배팅 선택   배당률: 8배   1착 맞추기 │
│              [단승] [연승] [복승] [쌍승] ...        │
│  ☑ 캐릭터 정보 안 보기   🟦 100   💎 0   삼복승 획득 💎: 10 │
│                           ↑       ↑       ↑       │
│                          교체     교체    라벨변경  │
│                                          (텍스트→💎 또는 "도파민 스톤")│
└────────────────────────────────────────────────────┘
```

### 컴포넌트

| 이름 | 종류 | 위치 | 역할 |
|------|------|------|------|
| `CurrencyHeader` | UGUI Component (신규) | 헤더 우중간 (총점 자리) | 🟦 젤리 / 💎 스톤 표시 + WalletManager 구독 |
| `roundProgressLabel` | TMP_Text (기존) | 헤더 좌측 | "1라운드 / 4" 표시 (Loc 키 변경) |
| `myPointLabel` (기존) | 폐기 | 우측 | **제거** (`총점:0` 라벨 — 의미 없음) |
| `payoutLabel` (기존, 라벨만 변경) | TMP_Text | 우측 | "삼복승 획득 💎: 10" 등 (UC10) |

### 상호작용

- 베팅 확정 → `WalletManager.TryBet(amount)` → CurrencyHeader 즉시 갱신 (`OnChanged` 이벤트)
- 라운드 결과 → `WalletManager.Reward(jelly, stone)` → CurrencyHeader 즉시 갱신
- 게임 오버 → CurrencyHeader는 사라지지 않고 0 / 누적 표시 (다음 화면에서 GameOverPanel이 덮음)
- 베팅 타입 선택 → `payoutLabel`이 해당 타입의 보상 표기로 갱신 ("단승 획득 💎: 5" 등)

str.bet.payout.trio               = "삼복승 획득 💎: {0}"

```

### 임시 → 프리팹화

- 임시: 기존 `myPointLabel` GameObject 비활성화 → `CurrencyHeader` 신규 GameObject 같은 위치에 배치 (TMP_Text + Image[icon] 2쌍)
- 검증: Play 모드에서 베팅 → 결과 흐름에서 통화 변동 정상 표시
- 프리팹화: `BettingUIPrefabCreator.cs`에 `CreateCurrencyHeader()` 메뉴 추가 → `Assets/Resources/Prefabs/UI/CurrencyHeaderPrefab.prefab` 생성

## U2. 베팅액 입력 모달 ★ 핵심 신규

### 컨셉

START 버튼 클릭 시 화면 중앙에 모달 팝업. 베팅액만 결정 후 라운드 시작. **기존 베팅 UI는 손대지 않음** (오너 단순화 채택).

        │                                      │

        │   ▰▰▰▰▰░░░░░  [ 25 ▮ ]            │
        │   ↑ 슬라이더    ↑ 키보드 입력         │
        │                                      │
        │   [-10] [-1] [+1] [+10]              │

### 컴포넌트

| 이름 | 종류 | 역할 |
|------|------|------|
| `BetAmountModal` | UI 컨트롤러 (신규) | 모달 전체 상태 관리 |
| `Backdrop` | Image | 어두운 반투명 (모달 외부 클릭 차단) |
| `betInfoSection` | Layout | 종목·선택·배당 표시 |
| `amountSlider` | Slider | 1 ~ 보유 젤리 |
| `amountInputField` | TMP_InputField | 키보드 직접 입력 (양방향 동기화) |
| `quickButton_-10` / `-1` / `+1` / `+10` | Button × 4 | 빠른 변경 (1·10 단위 ±) |
| `previewWin` / `previewLose` | TMP_Text × 2 | 적중·실패 시 결과 |
| `cancelButton` | Button | 모달 닫기 (베팅 미확정) |
| `confirmButton` | Button | 베팅 확정 + 라운드 시작 |

### 상호작용

| 이벤트 | 동작 |
|--------|------|
| 모달 진입 | `betAmount = 보유 젤리의 25%` 기본값. 슬라이더 25% 위치. InputField에 값 표시 |
| 슬라이더 변경 | InputField 즉시 갱신 + 미리보기 재계산 |
| InputField 입력 | 슬라이더 위치 갱신 + 미리보기 재계산. 보유 젤리 초과 시 자동 클램프 |
| 빠른 버튼 클릭 | 해당 값 적용 (+10/+25/+50은 현재값 + N, 최대는 보유 젤리) |
| `[취소]` | 모달 닫기, 베팅 미확정 (캐릭터 선택 유지) |
| `[배팅 확정]` | `WalletManager.TryBet(betAmount)` → 라운드 시작 → 모달 닫기 |
| 보유 젤리 < 1 (R12 BQ6) | 모달 진입 자체 차단 → 즉시 GAME OVER 화면 |

### 미리보기 계산 (Mathf.CeilToInt)

```
적중 시 받는 젤리 = Mathf.CeilToInt(betAmount × odds)
표시 = 적중 시 받는 젤리 - betAmount  (즉, 순이익)
스톤 보너스 = betAmount

실패 시 손실 = -betAmount
```

## U3. GAME OVER 화면 ★ 신규 프리팹

### 컨셉

### 와이어프레임

```
                [어두운 반투명 백드롭]
        ┌──────────────────────────────────────┐
        │                                      │
        │           💔  GAME OVER             │
        │                                      │
        │   도파민 젤리가 모두 소진됐습니다     │
        │                                      │
        │      3 / 4 라운드에서 종료           │
        │                                      │
        │      랭킹에 등록되지 않았습니다       │
        │                                      │
        │            [ 다시 도전 ]             │
        │                                      │

| 이름 | 종류 | 역할 |
|------|------|------|

| `Backdrop` | Image | 화면 전체 어두운 반투명 |

## U4. Finish 화면 (확장) — 메인/부수 분리 (UC4)

### 변경 요약

오너 인사이트 반영: "결과 화면에서 중요한건 배팅 성공 여부 + 획득한 스톤. 나머지는 부수적인 정보." → **메인 영역(큰 글씨, 강조)** vs **부수 영역(작은 글씨, 접힘 가능)** 으로 분리.

        │  ║   [메인 영역 — 가장 큰 글씨, 강조 색]        ║  │
        │  ║                                              ║  │
        │  ║      획득한 도파민 스톤: 540 💎              ║  │
        │  ║                                              ║  │
        │  ║   R1: ✓ 적중   (+25 💎)                     ║  │

### 상호작용

- 진입 시 부수 영역은 **기본 접힘** 상태 (모바일 친화)

- 메인 영역은 항상 표시 + 페이드인 애니메이션 (드라마틱 강조)

### 예외 처리 강화 (UC5)

- **중복 방지**: 같은 닉네임이 이미 등록돼 있으면 "이미 사용 중인 닉네임입니다" 에러
  - UGS Leaderboard에서 익명 PlayerID 기반으로 등록되며, 다른 PlayerID가 같은 닉네임 시도 시 차단
  - 구현: `LeaderboardService.SubmitStoneAsync` 내부에서 사전 중복 체크 (전체 entries 페이지네이션 조회 후 닉네임 비교) — 클라이언트 1차 필터
- **빈 입력**: 1자 이상 필수
- **길이 초과**: 8자 초과 시 자동 잘림 (TMP_InputField characterLimit)
- **특수 문자**: 한글·영문·숫자 외 차단 (CharValidator: `Custom`)
- **차단어**: `BadwordList.csv`에서 부분 일치 검사
- **네트워크 오류**: 재시도 안내. 등록 실패 시 닉네임 입력 유지 (취소하지 않으면 보존)
