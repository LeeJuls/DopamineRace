# DopamineRace - 작업 문서 관리

## 디렉토리 구조

```
Docs/
├── README.md          ← 이 파일 (문서 관리 규칙)
├── specs/             ← 작업별 계획/명세서
│   └── SPEC-XXX-제목.md
└── history/           ← 완료된 작업 히스토리
    └── HIST-XXX-제목.md
```

## 문서 번호 규칙
- **SPEC-XXX**: 작업 계획/개발 명세서 (XXX = 순번 001~)
- **HIST-XXX**: 완료 히스토리 (SPEC 번호와 1:1 매칭)

## 명세서 (SPEC) 포함 내용
1. 오더 원문
2. 작업 목표 및 범위
3. 단계별 개발 계획 (중간 테스트 포인트 포함)
4. 파일 변경 목록
5. 테스트 계획

## 히스토리 (HIST) 포함 내용
1. 작업 요약
2. 실제 변경된 파일 목록 + 주요 변경 사항
3. 테스트 결과
4. 특이사항 / 추후 작업 메모
5. 관련 커밋 해시

## 작업 흐름
```
오더 접수 → SPEC 작성 → 사용자 승인 → 개발 → 테스트 → HIST 작성 → 커밋
```
