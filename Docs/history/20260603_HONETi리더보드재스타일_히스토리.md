# 20260603 HONETi 리더보드 재스타일 히스토리

> 명세: `Docs/specs/spec036_HONETi리더보드재스타일_20260603.md`

## 요약
HONETi 픽셀 GUI를 **부분 적용**(스프라이트만). Result는 효과 미미로 원복, **리더보드**에 집중해 RANKING 디자인 구현.

## 진행 경과
| 단계 | 내용 | 결과 |
|------|------|------|
| 판단 | 부분 vs 전체 → 부분(한글 폰트 보존). client·qa·design 에이전트 검토 | 가드레일 도출 |
| Result 시도 | 타이틀바·윈도우 프레임·다크 배너·오렌지 버튼 베이크 | 이미 스킨돼 효과 미미 → **원복** |
| 리더보드 STAGE1 | 배팅 화면 임시 랭킹 버튼 추가 → 오버레이 오픈 | ✅ |
| 리더보드 STAGE3 | 행 파란 박스 + 골드 타이틀바 + 닫기버튼 | ✅ |
| 피드백 반영 | 외곽 윈도우 프레임 + 금색 코너 + 순위 `[N]`→배지 박스 + 헤더 제거 + 행 상향 | ✅ |
| 미세조정 | 배지 56→67, 바 상하 여백 +7 | ✅ |
| 구조 정리 | 행을 `LeaderboardEntry.prefab`으로 분리(nested) → 독립 수정 가능 | ✅ |

## 시행착오 / 교훈
| 이슈 | 원인 | 대응 |
|------|------|------|
| Result HONETi 효과 미미 | 프리팹이 이미 크림 패널로 스킨됨 (생성기 코드와 괴리) | 원복, 리더보드로 전환 |
| StringTable 변경 미반영 | Loc 캐시 (편집모드) | ForceUpdate 임포트 + Play 재진입 |
| 인스턴스에 변경 미반영 | 팝업이 stale 프리팹으로 빌드 | 프리팹 재임포트 후 Play 재시작 |
| AssetImportWorker 크래시 | 반복 ForceUpdate 재임포트 + Play 사이클 누적 | 재임포트 최소화, 메인 에디터는 정상(데이터 손실 없음) |
