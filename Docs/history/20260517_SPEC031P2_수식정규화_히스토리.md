# 20260517 SPEC-031.P2 — 수식 정규화 + INT 재스윕 + 동결

**날짜**: 2026-05-17 | **SPEC**: spec031 P2 | **결과**: 동결 완료

## 배경

이전 정밀 스윕(2026-05-17 오전)에서 16셀 모두 σ FAIL:  
shift/floor 어떤 조합도 β 개선 + σ ≤ 5를 동시 달성 불가.  
→ 근본 원인이 파라미터 값이 아닌 **수식 구조** 자체임을 진단 → P2 코드 수정.

## 근본 원인 2가지

| # | 현상 | 원인 수식 | 결과 |
|---|------|-----------|------|
| 1 | INT β 타입별 2.6× 편차 | shift = (end-start) × intMod × v4_intBurstShift | 구간 폭 비례 → Runner shift 1.75× Chaser |
| 2 | 드레인 할인도 타입 편차 | 절약 HP = burstDrainMul × (1-floor) × zoneWidth | 폭 비례 → 동일 floor라도 Runner 더 절약 |

→ 단순 파라미터 스윕으로 두 메커니즘을 동시 해소 불가.

## 해결: 수식 정규화 (Step 1)

### Shift 절대화

```csharp
// 구: 타입 폭 비례
float shift = (end - start) * intModifier * gs.v4_intBurstShift;

// 신: 타입 무관 절대 이동
float shift = gs.v4_intBurstShiftAbs * intModifier;
```

### 드레인 플로어 정규화

```csharp
float avgZoneWidth = (runnerW + leaderW + chaserW + reckonerW) / 4f;
float normFloor = 1f - (1f - gs.v4_intBurstDrainFloor) * avgZoneWidth / zoneWidth;
drain *= gs.v4_burstDrainMul * Mathf.Lerp(1f, normFloor, t);
```

**수학적 보장**: 총 절약 HP = burstDrainMul × (1-globalFloor) × avgWidth → 타입 무관 동일.

## Step 2 — 9셀 그리드 스윕 결과

INT β 하네스 (sweepStat=INT, N=300, equalStats OFF 로테이션):

| shiftAbs | floor | R β | L β | C β | Rk β | max/min | avg β |
|----------|-------|-----|-----|-----|------|---------|-------|
| 0.025 | 0.55 | 1.8 | 1.6 | 1.6 | 1.9 | 1.19× | 1.73 |
| 0.015 | 0.70 | **2.1** | **2.0** | **1.6** | **2.0** | **1.59×** | **1.92** |
| 0.025 | 0.70 | 1.9 | 1.8 | 1.6 | 1.9 | 1.19× | 1.80 |
| 구 동결 (shift=0.5 비율) | 0.55 | 3.9 | 3.4 | 1.5 | 2.3 | 2.6× | 2.78 |

**최적 선택**: `(shiftAbs=0.015, floor=0.70)` — max/min 1.59×, avg β 1.92, 전타입 β>0 ✅

> 잔여 0.09× 편차는 구조적: Runner(0.08–0.36)가 Chaser(0.44–0.60)보다 부스트 후 레이스 거리 길어 HP 이점을 더 오래 사용 → 구간 재배치 없이 해소 불가.

## 교훈

- 단일 파라미터 스윕이 실패하면 **수식 구조**를 의심. 두 메커니즘이 동일 편향 방향으로 결합되어 있으면 파라미터 조정만으로 해소 불가.
- 드레인 절약 = burstDrainMul × (1-floor) × **width** → width가 다르면 floor 조정이 비선형.
- 정규화 수식으로 "구조적 공정" 달성 후 스윕 → 유의미한 그리드 탐색 가능.
