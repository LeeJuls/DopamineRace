# his043 — 트랙별 날씨 효과 세션

**날짜**: 2026-04-26
**관련 SPEC**: SPEC-027
**소요**: 1일 (다수 반복)

## 작업 흐름

### Phase 1: 자산 준비
- **Pixel Weather FX 1.13** 임포트 (`Assets/PixelWeatherFX/`)
- 프리팹 카탈로그: Rain×3, Snow×6, Wind×4, Fog×3, Tornado×3, Lightning×2, Cloud×9

### Phase 2: Rainy 트랙 ✅
1. **PW_2D_Rain02_Moderate** nested instance로 Track_Rainy.prefab에 추가
2. SE_RainSplash 서브이미터 시도 → **회색 사각형으로 표시되는 버그** → `m_IsActive=0` 비활성
3. **PS_Splash 신규 생성** (MCP execute_csharp_script + 리플렉션)
   - v1: 정지형 + colorOverLifetime → 페이드 안 보임
   - v2: 낙하형 → 비처럼 보임 (사용자 거부)
   - **v4: 정지형 + 강한 페이드 인/아웃 + 사이즈 곡선** → 채택
4. 최적화: shadowCastingMode=Off, reflectionProbeUsage=Off → 깜빡임 해결

### Phase 3: 트랙 선택 드롭다운 (SPEC-028 통합)
1. `TrackChoice` enum 신규 (Random, Normal, Rainy, Snow, Desert, Highland)
2. `GameSettings.roundTracks`: string[] → TrackChoice[] 마이그레이션
3. **GameSettings.asset YAML 직접 편집** (Editor 종료 상태에서 — Unity가 타입 불일치 시 default(0)로 초기화하는 문제 회피)
4. `TrackDatabase.ApplyTrackForRound()`이 GameSettings 지정값 우선 참조

### Phase 4: 입상 sortingOrder
- `RaceManager.SetRacerSortingOrder()` 추가
- 1등 49 > 2등 48 > ... 형식, 라운드 리셋 시 0으로

### Phase 5: 타이틀 스폰 가속
- `TitleCharacterRunner` 상수 3개 변경 (50% 단축)

### Phase 6: Snow 트랙 ✅
- PW_2D_Snow02_Pixel_Moderate nested
- **SE_Melt 서브이미터 비활성** (rain SE_RainSplash와 동일 패턴)
- PS_Fog 보너스 안개 살림 (분위기 ↑)

### Phase 7: Desert 트랙 ❌ 롤백
- Wind01_Pixel + Fog02 시도
- ShapeModule 박스 변경, gravityModifier override → 수직 낙하 (눈처럼)
- VelocityModule.x/y 곡선 기반(minMaxState=3)이라 prefab override의 scalar가 무시됨 (회색 점만 찍힘)
- **결정**: Pixel Weather FX 프리팹은 3D + 일정 위치 가정으로 설계된 것 → 풀스크린 + 사선 바람 + 빈도 조절은 prefab override 메커니즘 한계 초과 → 롤백

### Phase 8: Highland 트랙 ✅
- 클라이언트 에이전트 검토:
  - Fog02 (α=0.067) vs Fog01 (α=0.27, 너무 진함) vs Fog03 (2-PS 컴포지트)
  - **Fog02 선택** (가장 옅고 단순)
  - Wind 사고 회피: 곡선 필드 일절 미수정
- ShapeModule.scale (50, **0**, 3) → (22, **12**, 3) 화면 전체 커버
- "모락모락" 펄스 튜닝:
  - 동시 파티클 ~625 (원본) → ~80 (너무 적어 안 보임) → ~240 → ~420 (적정)
  - lifetime 25→12s, rate 25→35/s
  - alpha override 시도(0.067→0.35) → **Unity nested Color.a 미지원**으로 실패 → 파티클 밀도로 대체

### Phase 9: 배회 bounds 축소
- Highland에서 캐릭터가 트랙 밖 산악 배경으로 빠짐
- center(3.5, 0)/size(12, 6) → center(3.0, 0)/size(10, 4.6)
- 가장 작은 트랙 기준으로 공통 축소

## 발견된 Unity Prefab Override 한계

| 한계 | 우회 방법 |
|------|----------|
| Nested struct (Color.r/g/b/a) 변경 불가 | 사용 안 함, 다른 수단으로 해결 |
| MinMaxCurve의 maxCurve.m_Curve.Array.data[*] 변경 불가 | 곡선 필드 미수정, scalar만 (Constant 모드일 때) |
| string[] → enum[] 타입 변경 시 자동 초기화 | YAML 직접 편집 |

## MCP 사용 노트

- `mcp__unity__execute_csharp_script_in_unity_editor`로 Unity Editor 내에서 PS_Splash 생성
- `UnityEngine.ParticleSystemModule` 어셈블리가 Roslyn 참조에 없음 → `Type.GetType` + 리플렉션으로 우회
- `using var` C# 8.0 문법 미지원 → `var scope = new ...; scope.Dispose()` 패턴
- AssetImportWorker 크래시(UnityCodeMcpServerSettings 시작 시점 경합) 발생 — 메인 에디터에는 영향 없음
