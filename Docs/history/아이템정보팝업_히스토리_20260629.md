# 아이템 정보 팝업 히스토리 (SPEC-050)

> 2026-06-29 · 도파민 젤리·스톤 클릭 설명 팝업 (모듈식)

## 작업 요약

배팅 화면 CurrencyHeader의 젤리·스톤 아이콘 클릭 시 이름+설명 팝업을 띄운다.
트랙 설명창 스타일. 다른 화면 아이템에도 재사용 가능하도록 트리거/팝업 분리 모듈화.

## 구현

| 산출물 | 내용 |
|--------|------|
| `Assets/Scripts/UI/ItemInfoPopup.cs` | 팝업 컴포넌트. `Instance` 싱글턴, `Show(nameKey,descKey)`/`Hide()`, 폰트 라우팅 |
| `Assets/Scripts/UI/ItemInfoTrigger.cs` | 클릭 트리거(IPointerClickHandler). nameKey·descKey만 지정 → 모듈 재사용. 토글 동작 |
| `Assets/Prefabs/UI/ItemInfoPopup.prefab` | Box(흰 9-slice)+NameLabel+DescLabel+Backdrop |
| StringTable.csv | `str.dopamine.item.{name,desc}.{dopaminejelly,dopaminestone}` 4키 × 8언어 |
| BettingPanel.prefab | 팝업 인스턴스 1개 + Jelly/Stone 컨테이너에 투명 raycast Image + 트리거 |

설명 출처: SPEC-040 §8 (아레나 경제 — 젤리=기대 에너지, 스톤=결정화된 믿음).

## 핵심 결정

| 항목 | 결정 |
|------|------|
| 키 명명 규칙 | `str.dopamine.item.{name\|desc}.{아이템id}` (오너 지정, 향후 표준) |
| 닫기 방식 | **토글** (같은 아이템 재클릭=닫기, 다른 아이템=원클릭 전환) |

| 생성 방식 | 프리팹 (코드 절차 생성 금지) — his046 국소 수정 패턴으로 배선 |

## 트러블

- **전환 막힘**: 초기 설계의 전체화면 backdrop(클릭=닫기)이 최상위라 젤리↔스톤 직접 전환을 가로챔 → backdrop raycast OFF, 트리거 토글 방식으로 변경.
- **`InstantiatePrefab` Awake 지연**: 에디터 API `PrefabUtility.InstantiatePrefab`는 Play 중 Awake가 지연돼 `Instance` 미등록처럼 보임. 실제 게임의 일반 인스턴스화(`Object.Instantiate`)에선 정상 등록 확인.
- **🔴 비활성 루트 → Instance null (실사용 버그)**: TitleScene에서 정식 실행 시 팝업이 안 뜸. 원인 = 팝업 프리팹 root가 `activeSelf=False`로 저장됨 → Awake 미실행 → `Instance` null → 클릭해도 무반응. raycast는 트리거에 정상 도달(차단 아님)했으나 `ItemInfoPopup.Instance==null`이라 무시됨.
  - **근본 수정**: 루트를 `SetActive(false)` 하던 방식 폐기. **루트는 항상 활성, 자식 `Box`만 토글**하도록 재설계(`box` 필드 추가). 루트가 항상 활성 → Awake 항상 실행 → Instance 항상 등록. 프리팹 재생성(root 활성) + BettingPanel 중첩 인스턴스 재배선(활성). backdrop 제거(토글 방식이라 불필요). Play 실클릭(raycast) 검증 통과.
