# 20260516 SPEC-030 — 결승선 트랙 이탈 수정 세션 히스토리

**날짜**: 2026-05-16
**관련 SPEC**: spec030
**개발 방식**: Harness Engineering (사전 진단 → 수정 → 결정적 검증)

## 작업 흐름

### 진단 (스크린샷 기반)
- 증상: 도착 후 일부 레이서가 트랙 밖 정지 (오너 확인: "이미 도착·정지")
- Explore 에이전트 초기 분석은 진행바 UI를 지목 → 스샷은 월드 화면이라 재검토
- 코드 직접 확인: `RacerController.GetOffsetPosition` 도착 스냅에 `deviationOffset` 미초기화 합산
- MCP로 `GameSettings.asset` 실제값 확인: `pathDeviation=0.3`, `laneOffset=0.19`
  → 도착 최악 1.155 > 레인밴드 0.855 (0.3 이탈) 수치 확정

### 수정 (RacerController.cs)
- `UpdateDeviation()` 최상단: `if (headingToFinish) { deviationOffset = 0f; return; }`
- `GetOffsetPosition()`: `dev = headingToFinish ? 0f : deviationOffset` +
  `Mathf.Clamp(laneOffset + dev, -laneBand, laneBand)` (`laneBand = RACER_COUNT/2 * laneOffset`)

### 검증 (강제 재컴파일 후)
- Gate A: 도착/레이스중 max|lateral| = 0.855 ≤ 0.855 PASS, 수정 전 1.155 재현
- Gate B: L880·L947 dev 게이트, L952-953 클램프 정적 확인
- Gate D: 컴파일 0 에러, RacerController 로드 정상
- Gate C: MCP Play 좀비 리스크로 오너 수동 검증 핸드오프

## RaceBacktestWindow 미러 판단

- CLAUDE.md "RacerController ↔ RaceBacktestWindow 동시 수정" 규칙은 **속도/HP 로직** 대상
