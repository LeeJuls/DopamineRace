## 1. 리더보드 빈 기록 안내 개선

**문제**: 기록 없을 때 행 템플릿(LeaderboardEntry)을 복제해 리스트 상단에 표시 → ①좌상단 정렬 ②템플릿 SummaryText 기본값 "R1:Win+0"이 안 지워져 잔재로 노출.

**해결**:
- 빈 상태 전용 안내 라벨을 **윈도우 중앙**에 표시 (행 복제 폐기 → SummaryText 잔재 원천 제거)
- 처음엔 코드 동적 생성 → 오너 요청으로 **프리팹 `EmptyLabel` 노드**로 전환 (LeaderboardPanel.prefab 루트, Inspector에서 위치/크기/폰트 직접 조정 가능)
- `SceneBootstrapper.Leaderboard.cs`: MkText 동적 생성 → `root.Find("EmptyLabel")` 캐싱
- 다국어: `str.leaderboard.empty` 일본어(ja)가 1줄뿐이었음 → 둘째 줄 보강 (7개 언어 2줄 통일)

## 2. 캐릭터 정보창 거리별 순위 스크롤 통합 동기화

**오너 요구**: ①하나라도 화면 넘으면 단/중/장 3개 함께 스크롤 ②이동 픽셀 동일.

**기존 결함** (normalizedPosition 동기화):
- 짧은 콘텐츠는 Elastic이라 자동 0복원 → 안 따라옴
- normalized는 폭 비율 → 폭 다르면 픽셀 이동 제각각

**해결** (client·qa 에이전트 검토 반영):
- 프리팹 3개 `RanksScroll` movementType Elastic(2)→**Unrestricted(0)**
- `CharacterInfoPopup.cs` LateUpdate: `content.anchoredPosition.x` **절대 픽셀** 동기화
- 공통 clamp = `Max(0, 가장 긴 콘텐츠 폭 - 뷰포트)`, 3개 동일 적용 (**빈 공간 허용** — 오너 결정)
- 마스터 = `_lastX` 대비 변화 감지(픽셀 임계 0.5), clampedX 저장
- `UpdateRecentRecords`: `ForceRebuildLayoutImmediate`로 폭 확정 + 스크롤 좌측 리셋 (캐릭터 전환 대응)
- 상세 plan: `Docs/plans/charinfo_scroll_sync_20260603.md`

## 3. DecorDots(파란 원 도트) 제거

**경위**: 이전 커밋(ffbc509)에서 BetAmountModal(9개)·ResultPanel(20개) 하단에 Img_Rank 파란 원 도트를 가로 띠로 추가했으나 두드러짐.

**해결**: 두 화면 모두 배경 스프라이트 `Img_BG_ListSlot_BG_01`이 캐릭터 카드와 동일 → 배경 자체에 우하단 은은한 점 무늬가 내장돼 있음. **DecorDots 제거** → 배경 무늬에 맡겨 캐릭터 카드와 동일 원리로 자연스럽게.
