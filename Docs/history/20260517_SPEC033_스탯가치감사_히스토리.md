## 흐름
- 의심: SPEC-031/032 후 스탯간 가치 드리프트 (지능5=스피드1식 의도비율 변동)
- 인프라 구축: 통제 단일스탯 스윕 계측(버킷 로테이션 위치편향 상쇄, G0 PASS) +
  숨은 보정 레이어(HiddenStatWeights.csv + 로더 + RacerController_V4 8배선 +
  RaceBacktestWindow 7미러). hidden=1.0 무변화 sanity 확인
- G1: β 측정 → SPD 6.0 = 타 스탯 ~4배 지배 확인 (드리프트 실재)
- 튜닝 시도: hidden_speed 감쇠 → SPEC-032 타입 σ 회귀(7.22) + 고요 다중-max 잔존 → 기각
- 오너 결정: 휴면 복원(1.0) / 고요=등장률 / 3그룹 공정성 감사만
- 버그 규명: HiddenStatWeights static `_loaded` 캐시 → 일반 백테스트가 stale
  스윕값으로 측정(σ 7.92 아티팩트). 정정: RunSimulationCore `EnsureLoaded`→`Reload`
- 복원 검증: 진짜 hidden=1.0서 σ=3.30 (SPEC-032 범위, 무회귀 확정)
- 감사: 동일 Total 코호트 스프레드 4~12pp(90/10 없음), 지배=Total티어+고요 단독 아웃라이어
- 결론: 스탯 재보정 불필요, 휴면 유지, 고요=등장률(별도)

## 교훈
- static 캐시 로더는 에디터 반복 실행서 stale 위험 → 측정 도구는 매 실행 Reload
- 단일 스탯 레버(SPD)가 타 동결 시스템(SPEC-032 타입경제)과 결합 → 격리 재보정 불가
- β 드리프트(스탯가치)와 로스터 공정성(캐릭 분포)은 별개 — β 이상이 곧 불공정은 아님
