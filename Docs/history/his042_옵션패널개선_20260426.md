## 완료 작업

### 1. ESC 키 패널 미표시 버그 수정

**원인**: `SceneBootstrapper.Awake()`에서 OptionPanel을 `transform`(SceneBootstrapper)에 붙임.
`BuildUI()`가 Canvas를 씬 루트에 독립 생성하므로 OptionPanel이 Canvas 바깥 → UI 렌더링 안 됨.

**수정**: `_mainCanvas` 필드 추가, `BuildUI()` 내 Canvas 참조 저장 → Instantiate 시 `_mainCanvas.transform` 부모 지정.

| 파일 | 변경 |
|------|------|
| `SceneBootstrapper.cs` | `_mainCanvas` 필드 + BuildUI()에서 저장 + Instantiate parent 수정 |

### 3. 확인 팝업 텍스트 변경 (7개 언어)

| 키 | 변경 전 | 변경 후 |
|----|--------|--------|
| `str.ui.option.confirm_restart` | 정말 재시작할까요? | 모든 기록이 사라지고 처음부터\n도파니와 함께 모험을 떠날 수 있습니다. |
| `str.ui.option.confirm_quit` | 정말 종료할까요? | 다시 도파니들과 함께 달리러 오실꺼죠?\n기다릴께요. |

- `\n` 줄바꿈 삽입 (Loc.cs `Replace("\\n","\n")` 처리 확인)
- MessageLabel `HorizontalOverflow` → Wrap, 팝업 크기 290px로 확대

### 4. ConfirmPopup 상태 리셋 버그 수정

**현상**: 재시작 버튼 클릭 → ESC 닫기 → ESC 다시 열면 확인 팝업이 열린 채로 표시.

**원인**: `Hide()` 시 패널 전체 `SetActive(false)`로 ConfirmPopup의 `true` 상태가 보존됨.

**수정**: `Hide()` / `Show()` 양쪽에서 `confirmPopup?.gameObject.SetActive(false)` 호출.

---

### 5. 재시작 시 TrackTransition 페이드 연출

`OnRestartConfirmed()` → `TrackTransition.Instance.PlayTransition(0.4f, StartNewGame)` 적용.
TitleScene→SampleScene 전환과 동일한 페이드 인/아웃 연출. fallback(Instance null) 대비.
