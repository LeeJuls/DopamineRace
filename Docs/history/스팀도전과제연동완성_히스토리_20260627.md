# 스팀 도전과제 연동 완성 히스토리 (2026-06-27)

> SPEC-046 후속. 코드 스캐폴드 + 파트너 사이트 등록(VDF·아이콘 Publish) 완료 상태에서
> **실제 Unity 연동(플러그인 임포트 + 심볼 + 런타임 검증)**을 마무리한 작업.

## 1. 배경

SPEC-046에서 코드(`SteamManager`·`SteamAchievements` + 10종 훅)와 파트너 사이트 등록(10개 언어 VDF·아이콘 Publish, 2026-06-27)은 완료됐으나, Unity 프로젝트에 **Steamworks.NET 플러그인 미임포트 + DR_STEAM 심볼 미정의** 상태라 실제 동작 불가했음.

## 2. 작업 내용

### 2.1 Steamworks.NET 2025.163.0 임포트 (MCP 전 과정 자동화)
- 오너 수동작업 0 — Unity MCP `execute_csharp`로 전 단계 자동화
- **다운로드**: Bash `curl` 샌드박스 차단 → Unity 내부에서 `System.dll` reflection으로 `WebClient` 호출(TLS 1.2=3072 강제), GitHub release(S3 302 redirect) 다운로드. 크기 1,002,663 bytes 무결성 일치 확인
- **임포트**: `AssetDatabase.ImportPackage(path, false)` (비대화식 전체 임포트)
- 코어 위치: `Assets/com.rlabrecque.steamworks.net/` (Editor/Runtime/Plugins, asmdef 2개, .cs 113)

### 2.2 충돌 해소 (CS0101)
- 패키지에 포함된 예제 `Assets/Scripts/Steamworks.NET/SteamManager.cs`가 자체 `SteamManager`와 클래스명 중복 → `CS0101` + `CS0579`
- `AssetDatabase.DeleteAsset`으로 예제 제거 (자체 구현이 `RuntimeInitializeOnLoadMethod` 자동 부트스트랩 상위호환)

### 2.3 DR_STEAM 심볼 추가
- `PlayerSettings.SetScriptingDefineSymbols(NamedBuildTarget.Standalone, ...)` — 기존 `AMPLIFY_SHADER_EDITOR` 보존하고 `DR_STEAM` 추가
- **Standalone 전용** — Steamworks.NET asmdef `includePlatforms`에 WebGL 없음 → WebGL에 켜면 컴파일 에러 (SPEC-046 §4 "모든 빌드 타겟" 기술은 오류였음, 본 작업에서 정정)
- 코어 asmdef `autoReferenced: true` 확인 → Assembly-CSharp 자동 참조 (자체 코드 별도 asmdef 불필요)

### 2.4 보조 작업 (커밋 81c005c)
- `DopamineBuildVerify`: DR_STEAM 빌드 시 `steam_api64.dll` 존재 검증 추가
- `SteamAchievements`: `[MenuItem] DopamineRace/Debug/Reset All Steam Achievements` 추가
- `SteamManager`: 설치 주석 정정 (씬 수동 배치 → RuntimeInitialize 자동 생성)

## 4. 알려진 사항 / 참고

- **미출시 게임 → 유저 프로필/커뮤니티 도전과제 페이지에 미표시 (Steam 정상 동작)**. 개발자 계정 해금·팝업은 작동. 실제 해금 카운트는 Steamworks 파트너 사이트 Stats & Achievements에서 확인. 출시 후 자동 노출.
- 나머지 9종(완주·파산·고배당 등 조건 생성 번거로운 것 포함) 전수 Play 테스트는 미실시 — 메커니즘 동일 경로로 입증됨.
- 누적 조건(예: N회 승리)은 Steam **Stat** 필요 — 본 범위 밖 (SPEC-046 §5).
