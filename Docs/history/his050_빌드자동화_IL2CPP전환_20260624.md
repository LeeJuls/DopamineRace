## 배경
Steam 출시 빌드 타입 확인 중 현재 Mono 빌드임을 발견. spec023에서 IL2CPP로 결정했으나 설정 유실. 보안 Layer C(SPEC-044) + 성능 + 표준 위해 IL2CPP 재전환 + 빌드 자동화. 큰 작업이라 에이전트 팀(director·build 신설 + qa)으로 구동.

## 에이전트 협업
- build·qa 에이전트 코드검증 검토로 plan 업그레이드(정정 5건).
- 신설: `director`(빌드/릴리스 PM, game-design leader와 분리) + `build`(빌드 엔지니어).

## 수행 (완료)
1. `.gitignore`: `steam_appid.txt`·`*.mac` 추가 (Builds/는 기존 규칙 제외 확인).
2. PlayerSettings IL2CPP 정정(검증): backend IL2CPP·bundleId com.Dopamine.DopamineRace·version 0.1.0·managedStripping Disabled·stripEngineCode 0. ProjectSettings.asset 반영 확인.
3. 빌드 스크립트: `DopamineBuilder.cs`([MenuItem]+batchmode CIRelease) + `DopamineBuildVerify.cs`(화이트/블랙리스트). 컴파일 0.
4. 문서·위키: SPEC-045 + 위키 `워크플로우/빌드_배포` + 에이전트가이드(director·build).

## 핵심 교훈 (build 엔지니어 실측)
- Standalone scriptingBackend 키 부재 → "신규 set". stripEngineCode:1 이미 ON → 0. 첫 빌드 MCP 금지(좀비). AhnLab V3 예외 필요. 보안코드는 AOT 안전(stripping Disabled면 첫 빌드부터 동작).
