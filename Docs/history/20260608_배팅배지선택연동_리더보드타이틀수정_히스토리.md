# 20260608 배팅 배지 선택연동 + 리더보드/타이틀 버그수정 히스토리

> 명세: `Docs/specs/spec037_캐릭터배당배지_선택연동_20260608.md` (배당 배지)
> 리더보드/타이틀은 버그픽스 (SPEC-028 R5 / TitleScene 전환)

## 요약
한 세션에서 **4건** 처리: ① 캐릭터 배당 배지 선택 연동 ② 리더보드 등록값 정합성 복원 ③ 점수 등록 결과 팝업 ④ 타이틀 전환 데드락 수정.

## 완료 작업
| # | 작업 | 내용 | 결과 |
|---|------|------|------|
| 1 | **배당 배지 선택연동** | v1 단독환산 → v2 종류별 → **v3 선택연동**(`GetPreviewOdds`). 복승 0명 1.1, 선택 시 조합값, 완성 후 교체 1.1. `UpdateButtonVisuals`에 갱신 통합 | ✅ |
| 2 | **리더보드 스톤 정합성** | 리더보드 등록값이 `ScoreManager.CurrentGameScore`(적중점수)였음 → SPEC-028 R5대로 `WalletManager.Stone`(누적 스톤)으로 교정. `LeaderboardScore` 프로퍼티 신설 | ✅ |
| 3 | **결과 팝업 + 터치 닫기** | 점수 제출 후 rank 결과가 깜빡이던 문제 → `NameEntryModal`을 결과 팝업 모드로(입력 UI 숨김·중앙 확대), 화면 터치/클릭/Enter로 닫기. `ForceRefetch` 백그라운드화 | ✅ |
| 4 | **타이틀 전환 데드락** | "새 게임→타이틀" 빌드업 연출(1.2s) 중 입력→`StartTransition`이 `SceneTransitionManager` 무시로 `isTransitioning` 고착 → 화면 멈춤. `StartTransition`에 STM 전환중 가드 추가 | ✅ |
