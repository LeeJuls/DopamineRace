# his047 — 환전 재설계: 고양이의 힘(전부환전) + 구제 분리 (2026-06-21)

## 배경 / 기획 변경

기존 환전(SPEC-028 R16~R20): **라운드당 1회, 비율만큼 스톤 → 젤리 1개**.
오너가 "한 번에 최대치 환전"을 요청 → balance·leader 검토 후 다음 3개를 확정:

| 결정 | 내용 |
|------|------|
| 안전망 분리 | "고양이의 힘"(게임당 N회·전부환전) + "구제"(라운드당 1회·파산 안전망) **별개 공존** |
| 환율 | 라운드마다 2~10 리롤 **유지** (전략 타이밍) |
| 사용 횟수 | 고양이의 힘 = `GameSettings.catPowerUsesPerGame` (기본 1, 0=비활성) |

## 두 능력

| | 🐾 고양이의 힘 | 구제 |
|---|---|---|
| 빈도 | 게임당 N회 (`_catPowerUsesThisGame`) | 라운드당 1회 (`_rescuedThisRound`) |
| 조건 | `uses<max && stone≥rate` | `!rescued && jelly==0 && stone≥1` |
| 효과 | `jelly += floor(stone/rate)`, `stone -= floor(stone/rate)*rate` | `jelly += 1`, `stone -= min(stone,rate)` |
| 리셋 | Awake·ResetForNewGame (게임) | RollExchangeRate (라운드) |

## 핵심: GameOver 단순화 (qa CR-1 해소)

```
ShouldGameOver = (jelly == 0 && stone == 0)
```
- **이유**: 구제가 매 라운드 리셋 → 스톤 1개라도 있으면 다음 라운드 구제로 생존 가능. 따라서 GameOver는 젤리·스톤 모두 0일 때만.
- **CR-1**: 기존 안(`jelly==0 && !CanRescue() && !CanUseCatPower()`)은 `GameManager.NextRound`에서 GameOver 체크(L359)가 라운드 플래그 리셋(L369)보다 **먼저**라 "이번 라운드 구제 썼으면 거짓 GameOver"(스톤 쥐고 죽음) 발생. 플래그 비의존식으로 원천 차단 + NextRound 재정렬 불필요.

## 검토 요지 (4개 에이전트)

- **balance**: 올인 GameOver 68→99% 우려, "구제 per-round 분리 + 전부환전 2회" 권장 → 오너가 분리 채택, 횟수는 설정화.
- **leader**: 안전망 붕괴·지배전략·갑툭튀 헛점 지적 → 분리로 안전망 보존. "마네키네코 변덕" 프레이밍 제안(추후 폴리시).
- **client**: 모달 버튼1개 우선순위, GetAvailableAction 단일판정, GameSettings 런타임 클램프(`Mathf.Max`) 필수, 터치포인트 전수(소비처 ExchangeModal+GameManager뿐).
- **qa**: CR-1(Critical) + 카운터리셋·무효소비·N회음수·모달상태(Major) → 전부 반영. TC-B4-02~09/spec029 GATE 재작성 필요(아래).
