# 럭키 잭팟 재설계 + 0젤리 게임오버 버그픽스 히스토리 (SPEC-051)

> 2026-07-01 · 고양이의 힘 → "마네키네코" 럭키 잭팟 전환 + 0젤리 즉사 버그 해소 + 연출

## 작업 요약

경기 중 젤리 0 + 스톤 보유로 START 시 즉시 game over 되던 버그를 고치고, 스톤→젤리 교환을
**럭키 잭팟**(도박형 럭키 리워드)으로 재설계. 구제(rescue)를 잭팟에 통합.

## 핵심 결정 (오너 + 에이전트 합의)

| 항목 | 결정 | 근거 |
|------|------|------|
| 버그 본체 | `TryShowBetAmountModal`의 `Jelly<1→즉사`가 `ShouldGameOver`(jelly&&stone)와 모순 | 스톤 있어도 game over |
| 게임오버 SSOT | `ShouldGameOver = jelly==0 && !CanJackpot()` | 0젤리라도 잭팟 가능하면 생존 |
| 잭팟 산식 | 보유 스톤 **전부 × N젤리**, N=2~8 가중, **무캡** | 항상 이득(1스톤≥2젤리) |
| 무캡 안전성 | balance 40~60만회 백테스트 — 무캡=캡=잭팟없음 | 스톤=점수 선소모 + 하우스엣지(−17%) + 2라운드 천장 3중 자기교정 |
| 빈도 | 게임당 1회 (자발 고양이클릭 + 0젤리 자동 공유, P1 긴장) | floor 5 컴백 완충 |
| 구제 | 잭팟에 통합 (별도 메커니즘 제거) | 오너 지정 |
| 0젤리 자동모달 | 강제(닫기·backdrop·ESC 차단) | 반드시 당겨야 진행 |

> ⚠️ **무캡 안전성은 (2라운드 + 하우스엣지 + 잭팟1회)에 의존** — 셋 중 하나라도 바뀌면 balance 재검증.

## UI 명명·연출 (오너 피드백 반영)

- 이름: Lucky Jackpot → **마네키네코**(招き猫), 버튼 "당기기" → **고양이의 보은**(猫の恩返し)
- 연출: 검정 반투명 판 + 밝은 노랑 롤(115pt) + 입력 잠금, 결과 "+{N} 도파민 젤리 획득" + 젤리 아이콘(130px)
- "스톤=최종 랭킹 점수" 문구 제거

## 검증

- 컴파일 0, StringTable Validate 0(`str.jackpot.*` 9키 완비).
- 단위(execute_csharp): CanJackpot 4조합 / ShouldGameOver(0젤리 버그 해소) / 산식·floor / E14 무효소비 미차감 / autoFloor 200회 min5 / RollJackpotN 분포.
- 오너 Play 스모크 반복(잭팟·0젤리 자동·연출·아이콘) — 통과.
