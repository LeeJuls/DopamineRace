# 번체 추가 히스토리 (SPEC-048)

> 2026-06-27 · tw 언어 + Fusion Pixel 폰트

## 작업 절차

1. **탐색** — TitleScene에 언어 드롭다운 이미 존재(TitleSceneManager), Loc CSV 파서 동적 확인
2. **Stage 1** — Loc.cs(langIndex tw=8, ChineseTraditional 분리) + TitleSceneManager 배열(tw/TW)
3. **Stage 2** — cn(간체) 422행 추출 → 6개 병렬 에이전트로 대만식 번체 변환 → C#로 CSV 병합(따옴표 안전) → 검증
4. **폰트** — Fusion Pixel(OFL) 12px ttf zh_hans/zh_hant를 GitHub 릴리스에서 받아 Resources/Fonts 설치 → FontHelper 라우팅
5. **Stage 4** — 문서/CLAUDE.md

## 발견 / 트러블

- 테스트 중 TW 선택 시 전체 키로 표시 → 원인은 tw 컬럼 미적용 상태(데이터 생성 완료, Assets 미반영). CSV 적용 후 해소.
- 중국어가 "기본 폰트"로 보임 → PFstardust에 CJK 글리프 없어 시스템 폴백. cn·tw 공통 문제였고 Fusion Pixel 도입으로 해결.
