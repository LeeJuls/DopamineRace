## 1. V4 스탯 불균형 발견 및 통일

### 문제
- 오너가 플래시(Flash) 캐릭터의 HP가 유독 빨리 소모된다고 보고
- 조사 결과: **CharacterDB_V4.csv의 스탯이 V3 CharacterDB.csv와 완전히 다른 값**으로 입력되어 있었음
  - 예: Flash — V3 Stamina=20, **V4 Stamina=13** (초기 기획 당시 차별화 값)
- 캐릭터별로 Speed/Accel/Stamina 등이 모두 달라 밸런스 기준 파악 불가

### 해결
- CharacterDB_V4.csv 전체 캐릭터(12명) 모든 스탯을 **20으로 통일**
- 목적: 타입(Runner/Leader/Chaser/Reckoner) 특성 차이만 보기 위한 테스트 기준선 확보
- 이후 밸런싱 단계에서 캐릭터별 개별 스탯 차별화 예정

### 추가 작업
- **디버그 오버레이 V4 스탯 테이블** 추가 (`RaceDebugOverlay.cs`)
  - 레이스 중 F4 디버그 오버레이 "상태" 탭 하단에 캐릭터별 SPD/ACC/STA/POW/INT/LCK/합계 표시
  - 접이식(foldout) 섹션, 현재 순위순 정렬
- **캐릭터 정보창 레이더 차트 V4 연동** (`CharacterInfoPopup.cs`)
  - V4 모드 활성 시 레이더 차트가 V4 스탯(CharacterDatabaseV4) 기준으로 표시
  - Legacy 모드: 기존 power/brave/calm/endurance 스탯 유지
  - V4 모드: speed/accel/stamina/power/intelligence/luck 스탯 사용
  - StringTable.csv에 3개 키 추가: `str.ui.char.stat.accel`, `str.ui.char.stat.stamina`, `str.ui.char.stat.intelligence` (7개 언어)

## 2. HP 드레인 — 진짜 절대 거리 기반으로 수정

### 문제
- 오너가 2바퀴와 5바퀴 레이스에서 HP 잔량이 같다는 것을 발견
- "단거리는 HP 걱정 없고 장거리는 HP 관리 필수"가 되어야 하는데 반영 안 됨

### 원인 분석
- 기존 공식: `drain = v4_drainAbsolutePerTrack × progressDelta`
- `progressDelta`는 0~1 정규화된 값이므로, **총 진행도 합산이 항상 1.0** → 바퀴 수 무관하게 동일 총 드레인
- 2바퀴든 5바퀴든 완주 시 소모량이 같았음

### 해결
- 공식 변경: `drain = v4_drainPerLap × totalLaps × progressDelta`
- 필드 이름 변경: `v4_drainAbsolutePerTrack` → `v4_drainPerLap`
- **캘리브레이션**: 지구력 20 (MaxHP 140) 기준, 5바퀴 완주 시 HP 20% 잔여
  - 총 소모 = 140 × 0.80 = 112 HP
  - `drainPerLap = 112 / 5 = 22.4`

### 검증 결과
| 바퀴 수 | HP 잔여 | 평가 |
|--------|--------|------|
| 2바퀴 | ~68% | 단거리 — HP 여유 |
| 3바퀴 | ~52% | 중거리 — 적당한 소모 |
| 5바퀴 | ~20% | 장거리 — HP 관리 필수 ✅ |

## 3. Runner 타입 테스트 — 버스트 구간 확장

### 배경
- 부스트 구간(6~25%)이 전체 레이스의 19%뿐 → Runner가 후반부에서 HP 없이 normal 속도로 달림
- Runner가 항상 뒤처지는 현상 발생

### 조치
- 테스트 목적으로 `v4_runnerBurstEnd` 확장: 0.25 → **0.86**
- 사실상 6~86% 전 구간 버스트 → HP 소진될 때까지 풀스피드
- 결과: Runner가 일관되게 선두 유지 확인

> **테스트값 주의**: 이 값은 밸런싱 전 검증용. M3b AI 구현 후 적절한 값으로 재조정 필요.

## 4. 부스트/스퍼트 드레인 배율 적용

### 문제
- `v4_burstDrainMul = 1.0`, `v4_spurtDrainMul = 1.0` — 부스트로 2배 빠르게 달려도 HP 소모가 기본과 동일
- 속도에 따른 HP 트레이드오프가 없어 전략적 의미 희박

### 조정
| 파라미터 | 변경 전 | 변경 후 |
|---------|--------|--------|
| `v4_burstDrainMul` | 1.0 | **1.8** |
| `v4_spurtDrainMul` | 1.0 | **2.5** |

### 설계 의도
- 부스트(Vmax×2.0) → HP 1.8배 소모 → "빠름의 대가"
- 최종 스퍼트(Vmax×1.5) → HP 2.5배 소모 → "피날레 연소" 느낌
- 기본 달리기는 1.0 기준 유지
