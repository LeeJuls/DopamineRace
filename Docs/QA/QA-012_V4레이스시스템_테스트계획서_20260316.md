## 1. 테스트 전략 개요

### 1.1 기본 원칙

V4 시스템은 V3와 **완전히 격리된 신규 시스템**이므로 두 가지 축으로 동시 검증한다.

- **수식 정확성**: 각 스탯 공식이 명세서 수치와 일치하는지 단위 검증
- **행동 의도성**: 포지션별 레이스 양상이 우마무스메 역기획 의도와 일치하는지 시나리오 검증

### 1.2 V4 전용 검증 캐릭터 (CharacterDB_V4.csv 기반)

| 검증 목적 | 캐릭터 | 특징 |
|----------|--------|------|
| 최고 Speed | 태풍(Runner), 폭풍(Runner) | Speed 20 |
| 최저 Speed | 로켓(Leader) | Speed 12 |
| 최고 Stamina | 폭풍(Runner), 선녀(Reckoner) | Stamina 20 |
| 최저 Stamina | 섬광(Chaser) | Stamina 13 |
| 최고 Intelligence | 번개(Leader) | Intelligence 18 |
| 최저 Intelligence | 태풍(Runner), 혜성(Reckoner) | Intelligence 8 |
| 최고 Power | 불꽃(Chaser), 천둥(Chaser), 해일(Reckoner) | Power 20 |
| 최저 Power | 유성(Runner), 폭풍(Runner) | Power 10 |

### 1.3 테스트 환경 전제

- `GameSettingsV4.asset` 파라미터를 명세서 기본값과 일치시킨 상태에서 시작
- 백테스트는 시뮬레이션 횟수 100회 이상 (통계 유의성 확보)
- 스탯 극단 테스트는 균등 스탯 모드(equalStats) 활용

## 2. M1: 깡통 주행 테스트

> **목표**: 단일 캐릭터가 V4 스탯 공식대로 달리는지 수식 단위 검증

### 2.1 초기화 검증 (M1-2: InitV4)

#### TC-M1-001: MaxStamina 계산 정확성

| 항목 | 내용 |
|------|------|
| 전제 조건 | V4 활성, `staminaBase=80`, `staminaPerStat=3` |
| 입력 | 스태미나 스탯 = 20, 15, 10 |
| 기대값 | MaxStamina = 140, 125, 110 (명세서 수치 일치) |
| 검증 방법 | Console 로그로 `v4MaxStamina` 출력 확인 |
| 실패 기준 | 오차 0.01 초과 |

#### TC-M1-002: 도주 타입 즉시 Spurt 상태

| 항목 | 내용 |
|------|------|
| 입력 | Runner 타입 단일 캐릭터 |
| 기대값 | `InitV4()` 호출 직후 `v4Phase == Spurt`, `v4IsSpurting == true` |
| 검증 방법 | 디버그 로그 또는 에디터 인스펙터 |
| 주의 | 선행/선입/추입은 Spurt 상태가 아니어야 함 |

### 2.2 속도 계산 검증 (M1-3: CalcSpeedV4)

#### TC-M1-003: Vmax = 1.0 + Speed × 0.02 검증

| 항목 | 내용 |
|------|------|
| 입력 | Speed = 20, 15, 10 |
| 기대값 | Vmax = 1.40, 1.30, 1.20 |
| 검증 방법 | 레이스 시작 직후 최고속도 도달 시 로그 확인 |

#### TC-M1-004: 스퍼트 Vmax = 기본 Vmax × 1.20 검증

| 항목 | 내용 |
|------|------|
| 입력 | Speed=20인 Runner (즉시 스퍼트) |
| 기대값 | 실제 최고속도 ≈ 1.68 (= 1.40 × 1.20) |
| 검증 방법 | 속도 그래프 또는 최고속도 로그 |

#### TC-M1-005: Accel Lerp 가중치 검증

| 항목 | 내용 |
|------|------|
| 입력 | Accel=20 캐릭터 vs Accel=10 캐릭터 |
| 기대값 | Accel=20이 목표속도에 더 빠르게 수렴 |
| 검증 방법 | 레이스 시작 후 1초 시점의 속도 비교 |
| 정량 기준 | 같은 목표속도를 향해 달릴 때 Accel=20이 목표의 90% 이상 도달하는 시간이 Accel=10보다 짧아야 함 |

#### TC-M1-006: 스퍼트 AccelBonus × 2.0 검증

| 항목 | 내용 |
|------|------|
| 입력 | 스퍼트 미발동 Accel=10 vs 스퍼트 발동 Accel=10 |
| 기대값 | 스퍼트 발동 시 가속 가중치가 1.00으로 동작 (기본 0.50 × 2.0) |

### 2.3 스태미나 소모 검증 (M1-4: ConsumeStaminaV4)

#### TC-M1-007: 기본 드레인 = currentSpeed × 0.08 검증

| 항목 | 내용 |
|------|------|
| 입력 | 슬립스트림 없음, 오버페이스 없음, currentSpeed=1.0 |
| 기대값 | 1초당 스태미나 소모 ≈ 0.08 |
| 허용 오차 | ±0.005 |

#### TC-M1-008: 스태미나 0 → 속도 급감 검증

| 항목 | 내용 |
|------|------|
| 입력 | Stamina=1인 극단 설정 (MaxHP=83), 고속 주행 |
| 기대값 | 스태미나 소진 시 Vmax가 기존의 50% 이하로 하락 |
| 기대값 추가 | `exhaustDecel=2.0` → 초당 속도가 2.0씩 감소하는 구간 발생 |
| 시각 확인 | 레이스 뷰에서 캐릭터가 눈에 띄게 느려지는 것 확인 |

#### TC-M1-009: 완주 가능성 검증

| 항목 | 내용 |
|------|------|
| 입력 | Speed/Stamina 균등 스탯 (값=10), 5바퀴 트랙 |
| 기대값 | 모든 캐릭터가 결승선 통과 (스태미나 0 도달 후에도 완주) |
| 실패 기준 | 캐릭터가 무한루프하거나 진행도 멈춤 |

## 3. M2: FSM + 페이스메이커 테스트

> **목표**: 9명 그룹 주행 시 포지션별 행동 및 페이스메이커 데드락 방지 검증

### 3.1 FSM 페이즈 전환 검증 (M2-2)

#### TC-M2-001: 페이즈 전환 타이밍 정확성

| 항목 | 내용 |

#### TC-M2-002: 스퍼트 트리거 타이밍 (타입별)

| 타입 | 트리거 조건 | 기대값 |

## 8. 포지션별 행동 시나리오

> **목표**: 4가지 포지션의 레이스 행동이 기획 의도와 시각적으로도 일치하는지 검증

### 8.1 도주(Runner) 시나리오

#### SC-R-001: 단거리 압도적 독주

**설정**: 태풍(Runner, Speed=20, Accel=12, Stamina=18) + 다른 포지션 8명, 2바퀴

**단계별 기대 양상**:
1. 스타트 직후: 즉시 Spurt 상태 → 전력질주로 선두 독점
2. 중반: 1위 유지, 2~3위와 점점 격차 벌어짐
3. 막바지: HP 소진 직전까지 선두 — 결승 통과
4. 완주 직후 스태미나 잔량: 0에 가까워야 함 (단거리 = 스태미나 거의 다 씀)

**실패 기준**: Runner가 단거리에서 초반 1위를 못 잡는 경우

#### SC-R-002: 장거리 후반 스태미나 소진 → 역전당함

**설정**: 폭풍(Runner, Speed=20, Stamina=20) + Reckoner(Stamina=20) 포함, 5바퀴

**단계별 기대 양상**:
1. 초반: Runner 선두 달리기
2. 중반 이후: HP 소진 → exhaustSpeedFloor 적용 → 속도 급감
3. 스퍼트 구간: Reckoner가 HP 풀 보존 상태로 폭발적 추격
4. 결승: Runner가 역전당하여 하위권 완주

**검증 포인트**: 장거리에서 Runner 최종 순위 ≥ 5위인 비율이 ≥ 40%

### 8.2 선행(Leader) 시나리오

#### SC-L-001: 도주 뒤 슬립스트림 크루징 → 막판 역전

**설정**: 번개(Leader, Int=18, Stamina=15) + 태풍(Runner), 3바퀴 중거리

**단계별 기대 양상**:
1. 포지셔닝: 2~4위 범위 진입
2. 크루징: Runner 바로 뒤에서 슬립스트림 유지 → 드레인 70%
3. 스퍼트(남은 20%): Runner보다 스태미나 더 보존된 상태 → 추월 시도
4. 결승: 중거리에서 Leader 1위 비율이 Runner보다 높아야 함

**기대값**: 슬립스트림 ON 쪽이 결승선 통과 시 잔여 스태미나 ≥ 정상의 1.3배

2. 크루징: 마군 속에서 슬립스트림 찾아 스태미나 비축
3. 스퍼트(남은 30%): Power=20으로 마군 충돌 페널티 없이 돌파
4. 결승: 중거리에서 Chaser 상위 3위 진입 비율 ≥ 35%

#### SC-C-002: Power 차이에 따른 마군 돌파력 비교

### 8.4 추입(Reckoner) 시나리오

#### SC-RC-001: 장거리 극적 역전 시나리오

**설정**: 선녀(Reckoner, Accel=18, Stamina=20) + Runner/Leader/Chaser 포함 9명, 5바퀴

**단계별 기대 양상**:
1. 초반 (0~50%): 7~9위 범위 유지, Vmax의 70~80% 속도
2. 중반 (50~75%): 여전히 후미, 스태미나 풍부
3. 스퍼트(남은 40%): Accel=18의 폭발적 가속 + 아웃코스 우회
4. 결승: 스태미나 소진 직전 1~3위 돌입

**검증 포인트**:

## 9. 거리별 밸런스 검증

> **목표**: 백테스트를 통해 Run_balance.md 목표 승률에 수렴하는지 정량 확인

### 9.1 검증 방법론

#### 표준 편성

9명 레이스에서 포지션별 균등 배분:
- Runner × 2 (태풍, 폭풍)
- Leader × 3 (번개, 로켓, 질풍)
- Chaser × 2 (불꽃, 천둥)
- Reckoner × 2 (혜성, 해일)

#### 균등 스탯 편성 (통제 실험)

모든 스탯 = 10으로 균일화한 상태에서 포지션 전략 로직만 검증:
- 이 경우 승률이 포지션 전략 순수 효과를 반영

#### 실제 스탯 편성 (종합 검증)

CharacterDB_V4.csv 실제 수치 사용: 스탯 우열 + 전략 효과 합산

### 9.2 밸런스 스윕 테스트

#### TC-BAL-001: drainBaseRate 영향도 테스트
