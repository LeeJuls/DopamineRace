## 사전 주의 사항

- M1~M4는 **순서대로 게이팅** — 이전 Phase QA PASS 확인 후 다음 진행
- 각 Phase 완료 후 client가 QA 에이전트에 검증 요청 송신 필수
- `RacerController_V4.cs` 수정 시 **`RaceBacktestWindow.cs`도 반드시 동시 수정** (미러링 원칙)
- `useV4RaceSystem` 플래그가 `false`일 때 V3 동작에 영향이 없는지 항상 확인

## Phase M1 QA 체크리스트 — 깡통 주행 (단일 개체 테스트)

### 기능 테스트

- [ ] **V4 초기화**: `GameSettings.useV4RaceSystem = true` 설정 후 Play → `InitV4()` 호출 로그 확인 → `[RacerController V4] InitV4()` 이외 에러 없음
- [ ] **MaxStamina 계산**: Speed=10, Stamina=10 캐릭터 단독 실행 → `v4MaxStamina = 80 + 10×3 = 110.0f` (오차 ±0.01 허용)
- [ ] **MaxStamina 계산**: Stamina=20 캐릭터 → `v4MaxStamina = 140.0f` (오차 ±0.01)
- [ ] **Vmax 계산**: Speed=10 → `Vmax = 1.20f`, Speed=20 → `Vmax = 1.40f` (F1 디버그 오버레이 또는 로그로 확인)
- [ ] **가속 Lerp**: Accel=20 캐릭터가 Accel=5 캐릭터보다 목표 속도에 더 빨리 도달 → 레이스 시작 후 0.5초 시점 현재속도 비교 (Accel=20이 더 높아야 함)
- [ ] **Vmax 클램프**: 어떤 조건에서도 `v4CurrentSpeed > Vmax` 상태가 발생하지 않음 → 10회 Play 반복 확인
- [ ] **스태미나 드레인**: 단일 캐릭터 완주 후 `v4CurrentStamina >= 0` (음수 없음)
- [ ] **스태미나 0 도달 시 패널티**: MaxHP가 낮아 조기 고갈되도록 설정 → 고갈 직후 속도가 `기존 Vmax × 0.50` 이하로 감속
- [ ] **스태미나 고갈 감속**: 고갈 후 `v4_exhaustDecel = 2.0` 초당 추가 감속 적용 → 고갈 전 후 속도 로그 비교
- [ ] **완주 처리**: 단일 V4 캐릭터가 트랙 완주 후 `IsFinished = true`, `FinishOrder` 정상 기록

### 예외 케이스

- [ ] **Stamina=1 극단값**: MaxHP = 83 → 초고속 드레인으로 조기 고갈 → 크래시 없이 exhaustSpeedFloor 적용 후 완주
- [ ] **Speed=20 + Stamina=1**: Vmax=1.40 상태에서 드레인 최대 → 고갈 후에도 완주 (멈추지 않음)
- [ ] **dt 스파이크**: 프레임 드랍 시뮬레이션 (dt=0.1초 강제) → `v4CurrentStamina` 음수 없음, 클램프 정상 (`Mathf.Max(0f, ...)` 가드 확인)
- [ ] **V4 미활성 상태**: `useV4RaceSystem = false` → V3 로직으로 동작, V4 변수(`v4CurrentStamina` 등) 0 또는 초기값 유지

### 합격 기준

- 위 체크리스트 전 항목 PASS
- Unity 콘솔 에러 0건
- `v4CurrentStamina` 음수 발생 0회 (10회 Play 기준)
- Vmax 초과 발생 0회 (10회 Play 기준)
- 단일 캐릭터 완주율 100% (Speed/Stamina 스탯 조합 5가지 이상 테스트)

## Phase M2 QA 체크리스트 — FSM + 가상 페이스메이커 (그룹 주행)

### 기능 테스트

- [ ] **페이스메이커 생성**: Play 시 `VirtualPacemaker` 오브젝트가 씬에 생성, 레이스 종료 시 정리됨
- [ ] **페이스메이커 속도**: `globalSpeedMultiplier × pacemakerSpeedRatio` 기준으로 `pacemakerProgress`가 매 프레임 증가 (로그 확인)
- [ ] **FSM 전환 - Positioning → Cruising**: OverallProgress >= 0.15(15%) 시점에 모든 V4 캐릭터의 `v4Phase == Cruising` 전환 확인
- [ ] **FSM 전환 - Cruising → Spurt**: 마지막 랩 진입 후 타입별 트리거 비율 도달 시 `v4Phase == Spurt` 전환
  - 선행: 남은 진행도 20% 시점
  - 선입: 남은 진행도 30% 시점
  - 추입: 남은 진행도 40% 시점
- [ ] **도주 즉시 스퍼트**: 도주 타입 캐릭터는 `InitV4()` 호출 즉시 `v4Phase == Spurt`, `v4IsSpurting = true`
- [ ] **포지셔닝 목표 순위 진입**:
  - 도주 캐릭터: OverallProgress 15% 시점에 1~2위권 (허용 범위: 1~3위)
  - 선행 캐릭터(3명): 2~4위권 (허용 범위: 1~6위)
  - 선입 캐릭터(3명): 4~6위권 (허용 범위: 3~8위)
  - 추입 캐릭터(3명): 7~9위권 (허용 범위: 5~12위)
- [ ] **추입 크루징 속도 제한**: 크루징 중 추입 캐릭터 `v4CurrentSpeed <= Vmax × 0.80f` 항시 유지
- [ ] **페이스메이커 없을 때 vs 있을 때**: `useVirtualPacemaker = false` → 데드락(1위 캐릭터가 0.0f/s로 멈춤) 발생 가능성 확인 → `useVirtualPacemaker = true` 후 동일 조건에서 데드락 없음
- [ ] **스퍼트 속도 적용**: 스퍼트 진입 직후 Vmax가 `기본 Vmax × 1.20f`로 상향, 가속 가중치가 `기본 가속 × 2.0f` 적용
- [ ] **9명 완주**: 9명 그룹 주행 시 전원 완주, 순위 1~9위 중복/누락 없음
- [ ] **거리별 페이스메이커 속도 보정**: 단거리(2바퀴) vs 장거리(5바퀴) 실행 비교 → 단거리에서 페이스메이커가 더 빠른 속도 계수 적용

### 예외 케이스

- [ ] **동일 타입 9명**: 도주 9명 실행 → 포지셔닝 목표 충돌 시 크래시 없이 처리 (우선순위 또는 타이브레이크 로직 확인)
- [ ] **극단적 Vmax 차이**: Speed=20 도주 + Speed=1 추입 → 포지셔닝 중 추입이 목표 순위보다 너무 뒤처질 때 무한 가속 루프 없음 확인
- [ ] **페이스메이커 완주 후**: `pacemakerProgress >= 1.0f` 초과 시 동작 정상 (캐릭터들이 레이스 계속 완주)
- [ ] **2라운드 반복**: 레이스 1회 완료 후 재시작 → `v4Phase` 리셋, `v4CurrentStamina` 재초기화, 페이스메이커 재생성 정상

### 합격 기준

- 위 체크리스트 전 항목 PASS
- Unity 콘솔 에러 0건
- 9명 전원 완주율 100% (5회 반복 실행)
- 포지션별 순위 분포 (OverallProgress 15% 시점, 5회 평균):
  - 도주: 평균 순위 1.5 이하
  - 추입: 평균 순위 7.5 이상
- 데드락 발생 0회 (페이스메이커 ON 상태에서 5회 실행)
- Play 모드 진입·종료 반복 3회 후 상태 정상 (NullReferenceException 없음)

## Phase M3 QA 체크리스트 — Y축 차선 + 지능 AI + 충돌

### 기능 테스트

#### Y축 차선

- [ ] **5차선 초기 배치**: 레이스 시작 시 9명이 5차선 내에 중복 없이 분산 배치 (같은 차선에 최대 2명)
- [ ] **차선 변경 속도 감소**: 차선 변경 시작 ~ 완료 구간 동안 `v4CurrentSpeed` 소폭 감소 확인 (변경 없을 때 대비 5% 이상 차이)
- [ ] **차선 변경 1칸 제한**: 단일 판단 틱에서 1차선 이상 이동하지 않음 (추입 아웃코스 우회 2칸 제외)
- [ ] **차선 변경 쿨다운**: 차선 변경 후 0.5초 이내 재변경 없음 (타이머 로그 확인)
- [ ] **차선 범위 클램프**: 차선 1(인코스) 이하, 차선 5(아웃코스) 이상으로 이동 없음

#### 슬립스트림

- [ ] **슬립스트림 발동 조건**: 같은 Y축 차선 + 앞 캐릭터와 진행도 차이 ≤ `slipstreamRange` → `v4InSlipstream = true`
- [ ] **슬립스트림 드레인 감소**: `v4InSlipstream = true` 상태의 드레인이 미발동 상태보다 30% 적음 (`drainBaseRate × 0.70` 확인)
- [ ] **도주 슬립스트림 불가**: 도주 캐릭터는 선두이므로 `v4InSlipstream` 항시 `false`
- [ ] **차선 이탈 시 슬립스트림 해제**: 앞 캐릭터와 차선이 달라지면 즉시 `v4InSlipstream = false`

#### 지능 AI

- [ ] **판단 틱 주기**: Int=20 캐릭터의 `v4ThinkTimer` 주기 0.20초, Int=5 캐릭터 0.80초 (로그 또는 디버거 확인)
- [ ] **P_smart 회피 확률**:
  - Int=20: `P_smart = 0.667f` (67% 회피) — 10회 충돌 상황 시 6~8회 회피 허용
  - Int=5: `P_smart = 0.333f` (33% 회피) — 10회 충돌 상황 시 2~5회 회피 허용
- [ ] **지능 슬립스트림 탐색**: Int=20 vs Int=5 캐릭터 비교, 장거리 레이스 5회 평균에서 Int=20의 슬립스트림 발동 누적 시간이 Int=5보다 30% 이상 많음
- [ ] **오버페이스 발동**:
  - Int=5 캐릭터 100회 백테스트 → P_panic=0.225 기준 오버페이스 발동률 18~27% (±20% 허용)
  - Int=20 캐릭터 → P_panic=0.00, 오버페이스 발동 0회
- [ ] **오버페이스 드레인 증폭**: 발동 중 드레인이 정상 상태 대비 1.80배 (±5% 허용)
- [ ] **오버페이스 지속 시간**: 발동 후 `v4_panicDuration = 2.0`초 이후 정상 드레인 복귀
- [ ] **동적 스퍼트 타이밍**: Int=20 캐릭터는 고정 트리거 비율보다 ±5% 범위 내에서 스퍼트, Int=5 캐릭터는 정확히 고정 트리거 비율에 발동

#### 충돌 시스템

- [ ] **V4 충돌 공식**:
  - Power=20: 충돌 시 `V_drop = 0.00f` (속도 손실 없음)
  - Power=10: `V_drop = 0.175f` (±0.005 허용)
  - Power=5: `V_drop = 0.263f` (±0.005 허용)
- [ ] **Luck 충돌 회피**:
  - Luck=20: `dodgeChance = 0.40f` (40% 회피) — 10회 충돌 상황 시 3~6회 회피 허용
  - Luck=1: `dodgeChance = 0.02f` (2% 회피) — 50회 충돌 상황 시 0~3회 회피 허용
- [ ] **마군 돌파 - 선입**: 스퍼트 진입 후 전방 장애물 발생 시 차선 1칸 이동으로 우회 시도
- [ ] **마군 돌파 - 추입**: 스퍼트 진입 후 아웃코스 방향 2칸 이동으로 우회 가능

#### Luck 크리티컬

- [ ] **크리티컬 발동률**: Luck=20 → `critChance = 0.10f`, 3초마다 판정 → 60초 레이스에서 15~25회 발동 (±30% 허용)
- [ ] **크리티컬 효과**: 발동 시 속도가 `critBoost = 1.30` 배수로 1.5초간 상승

### 예외 케이스

- [ ] **동일 차선 9명 동시 밀집**: 모든 캐릭터가 차선 3에 몰릴 때 → 충돌 발동 시 스택 오버플로우 없음, 전원 완주
- [ ] **블로킹 상태**: 빠른 캐릭터가 느린 캐릭터 뒤에서 차선 이동 불가 상태 → 속도 강제 하향 후 쿨다운 내 이탈 시도 (무한 블로킹 없음)
- [ ] **Power=0 캐릭터**: `V_drop = 0.35 × (1 - 0/20) = 0.35` 최대 손실 → 크래시 없이 처리 (음수 속도 없음)
- [ ] **오버페이스 + 슬립스트림 동시**: 두 상태 동시 활성 시 드레인 계산 `baseSpeed × drainRate × 0.70 × 1.80` 정상 적용 (중복 적용 누락 없음)
- [ ] **차선 변경 중 충돌**: 이동 중간 좌표에서 다른 캐릭터와 겹칠 때 Power 판정 정상 처리

### 합격 기준

- 위 체크리스트 전 항목 PASS
- Unity 콘솔 에러 0건
- Int=20 vs Int=5 슬립스트림 활용 시간 차이: 30% 이상
- Int=5 오버페이스 발동률: 18~27% 범위 내 (100회 백테스트)
- Power=20 충돌 시 V_drop = 0.000f (오차 0)
- 마군 돌파 성공률: Int=20 캐릭터 60~75%, Int=5 캐릭터 30~40% (각 50회 충돌 상황)
- 전 캐릭터 완주율 100% (다양한 스탯 조합 10회 실행)

## Phase M4 QA 체크리스트 — 백테스트 확장 + 밸런싱

### 기능 테스트

- [ ] **V4 백테스트 블록 존재**: `RaceBacktestWindow.cs`에 V4 시뮬레이션 탭 또는 토글 존재
- [ ] **공식 미러링 일치 확인**:
  - Vmax 계산: 백테스트 결과 = RacerController_V4 런타임 결과 (Speed=10, 단일 캐릭터 비교)
  - MaxStamina 계산: 동일 조건 백테스트 vs 런타임 오차 ±1.0f 이내
  - 드레인 계산: 동일 조건 백테스트 vs 런타임 오차 ±2% 이내 (100회 평균)
- [ ] **페이즈별 순위 스냅샷 출력**: 백테스트 실행 시 OverallProgress 25%/50%/완료 시점 순위 로그 출력
- [ ] **스태미나 소모 추이 로그**: 각 캐릭터의 10% 구간마다 남은 스태미나 기록 출력
- [ ] **오버페이스/슬립스트림 발동 횟수 로그**: 레이스 1회 완료 시 캐릭터별 누적 횟수 출력
- [ ] **단거리 승률 — 도주 45%**: 단거리 설정, 100회 백테스트 → 도주 포지션 승률 36~54% (±9%p 허용)
- [ ] **중거리 승률 — 선행/선입 각 35%**: 중거리 100회 → 선행 28~42%, 선입 28~42%
- [ ] **장거리 승률 — 추입 45%**: 장거리 100회 → 추입 36~54%
- [ ] **GameSettingsV4 Inspector 표시**: v4_speedStatFactor, v4_drainBaseRate 등 핵심 계수가 Inspector에서 편집 가능
- [ ] **밸런스 파라미터 실시간 반영**: Inspector에서 수치 수정 후 백테스트 재실행 → 즉시 반영 (재컴파일 불필요)

### 예외 케이스

- [ ] **백테스트 1000회 실행**: 크래시 없음, 결과 파일/로그 정상 저장
