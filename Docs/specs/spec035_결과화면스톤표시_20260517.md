# SPEC-035 — 결과 화면 점수 → 도파민 스톤 획득 표시

**날짜**: 2026-05-17 | **상태**: 구현 진행
**오너 요청**: 결과 화면이 "총점"으로 나오는데, 실제 게임은 도파민 젤리 배팅 → 도파민 스톤 획득 구조.
"도파민 스톤 n개 획득 (총: n개)" 형태로 변경.

## 배경

- 현재 결과 화면(`SceneBootstrapper.Result.cs ShowResult()` 273~280줄)은
  `ScoreManager.CurrentGameScore` 누적 점수를 `str.result.score_win/lose`로 표시
- 실제 통화 흐름: `BetAmountModal.TryBet()` → 베팅, 적중 시
  `BetReward.stone = betAmount` (1:1) → `WalletManager.Reward()` → `Stone` 누적
- 점수(score)는 게임 핵심 통화가 아님 → 스톤 획득량 표시가 직관적

## 변경

| 파일 | 변경 |
|------|------|
| `Assets/Scripts/Manager/GameManager.cs` | `LastRoundStoneGain` 프로퍼티 신규. `CalcScore()`에서 적중=`reward.stone`, 미적중·비통화=0 세팅 |
| `Assets/Scripts/Manager/UI/SceneBootstrapper.Result.cs` | 273~280줄: 점수 텍스트 → 스톤 획득량(`gm.LastRoundStoneGain`) + 총 보유(`WalletManager.Instance.Stone`) |
| `Assets/Resources/Data/StringTable.csv` | `str.result.score_win/lose` 내용을 스톤 문구로 교체 (7개 언어) |

## 문구 (StringTable)

| 키 | ko | 인자 |
|----|-----|------|
| `str.result.score_win` | `도파민 스톤 +{0}개 획득!  (총: {1}개)` | {0}=이번획득 {1}=총보유 |
| `str.result.score_lose` | `스톤 획득 실패  (총: {0}개)` | {0}=총보유 |

en/jp/cn/de/es/br 동일 의미로 번역.

## 승/패 판정

`score > 0`(적중) 유지 — 타이틀·색상 로직 불변. 스톤 획득량만 표시 라인 교체.
점수 공식 라인(`resultScoreFormulaText`, 256~271줄)은 배당 산식 표시로 유지(범위 외).

## 검증

- 적중 라운드: "도파민 스톤 +N개 획득! (총: M개)" 표시
- 미적중: "스톤 획득 실패 (총: M개)"
- AutoRacer(betAmount=0): 0개 — 정상 (통화 흐름 미적용)
