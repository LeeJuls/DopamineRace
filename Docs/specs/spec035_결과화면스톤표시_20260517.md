# SPEC-035 — 결과 화면 점수 → 도파민 스톤 획득 표시

**날짜**: 2026-05-17 | **상태**: 구현 진행
**오너 요청**: 결과 화면이 "총점"으로 나오는데, 실제 게임은 도파민 젤리 배팅 → 도파민 스톤 획득 구조.
"도파민 스톤 n개 획득 (총: n개)" 형태로 변경.

## 배경

- 현재 결과 화면(`SceneBootstrapper.Result.cs ShowResult()` 273~280줄)은
  `ScoreManager.CurrentGameScore` 누적 점수를 `str.result.score_win/lose`로 표시
- 실제 통화 흐름: `BetAmountModal.TryBet()` → 베팅, 적중 시
  `BetReward.stone = betAmount` (1:1) → `WalletManager.Reward()` → `Stone` 누적
- 점수(score)는 게임 핵심 통화가 아님 → 스톤 획득량 표시가 직관적

## 승/패 판정

`score > 0`(적중) 유지 — 타이틀·색상 로직 불변. 스톤 획득량만 표시 라인 교체.
점수 공식 라인(`resultScoreFormulaText`, 256~271줄)은 배당 산식 표시로 유지(범위 외).

## 추가 결정 (오너) — 스톤 = 배율 보상

> ⚠️ **SPEC-047(2026-06-27)로 개정됨:** 아래 "`stone = jelly = ceil(betAmount × odds)`" 결정 중 **젤리 동시 지급 부분은 폐기**. 적중 시 젤리는 반환하지 않고(배팅 시 소멸) 스톤만 `ceil(betAmount × odds)` 지급. 스톤 공식 자체(배율 반영)는 유지.

기존: `stone = betAmount`(1:1) / `jelly = ceil(betAmount×odds)`.
오너 피드백: 결과 보상이 베팅액 그대로라 직관적이지 않음.
→ ~~**`stone = jelly = ceil(betAmount × odds)`** 로 변경.~~ (SPEC-047: 젤리 반환 폐기, 스톤만 `ceil(betAmount × odds)`)
`odds`는 `OddsCalculator.CalcComboOdds`에서 캐릭터 인기도 + 베팅타입 난이도(Win=단일, Exacta=곱×계수 등) 모두 반영됨.

| 파일 | 변경 |

| `BettingData.cs` | `CalculateReward`: `stoneGain = betAmount` → `stoneGain = jellyGain` |
| `SceneBootstrapper.Result.cs` | 산식 라인 `basePt×odds=score`(레거시) → `betAmt × 배당 = stoneGain` |

> 환전(스톤→젤리)·경제는 숫자 스케일만 커짐 (메커니즘 불변). 레거시 score 시스템은 잔존(타이틀 승패 판정용 `score>0`)하나 화면 표시는 스톤 기준.
