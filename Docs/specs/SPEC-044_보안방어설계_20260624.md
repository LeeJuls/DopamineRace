## 1. 위협 모델

리더보드 점수 = `WalletManager.Stone`(메모리 정수)이 그대로 서버 제출됨. 서버(Cloudflare Worker)는
WRITE_TOKEN + 점수범위(0~150k)만 검증 → 토큰만 추출하면 임의 점수 통과.

**근본 한계**: 레이스가 100% 클라이언트 RNG 시뮬레이션 → 서버가 점수를 재계산해 검증하는 것은 구조적으로 불가능.
따라서 목표는 **"완전 차단"이 아니라 "캐주얼 치터 차단 + 작정한 리버서 최대한 지연"**.

| 공격 | 현재 | 방어 후 |
|------|------|---------|
| 트래픽 스니핑 후 점수 JSON 수정·재전송 | 매우 쉬움 | HMAC 서명 깨짐 → 차단 |
| 빌드 DLL 디컴파일로 토큰/키 추출 | 쉬움(Mono) | IL2CPP → 네이티브 리버싱 필요 |
| "그냥 150000 보내기" | 통과 | rounds 기반 상한 → 거부 |
| Cheat Engine 메모리 스캔 | 쉬움 | SecureInt 마스킹 → 값 탐색 실패 |
| PlayerPrefs·leaderboard.json 직접 편집 | 쉬움 | HMAC 체크섬 → 변조 감지 |

> 남는 한계(정직히): IL2CPP 네이티브 바이너리를 리버싱해 SIGN_KEY 추출 + 서명 로직 재현하는 숙련 공격자는
> 여전히 위조 가능. 완전 차단은 서버 권위 시뮬레이션(레이스를 서버에서 실행)뿐 — 범위 밖.

## 2. 방어 4레이어 (핵심)

### L1 — HMAC 본문 서명 (가장 직접적)
- 클라가 보내는 **raw 바디 바이트 전체**에 `HMAC-SHA256` 서명 → 헤더 `X-Sig`(소문자 hex).
- 서버는 `request.text()`로 받은 **그대로** 검증 후 `JSON.parse`. → C#(`JsonUtility`)↔JS 직렬화 불일치 원천 차단.
- `SIGN_KEY`는 **WRITE_TOKEN과 별도 신규 시크릿**. Worker secret + `LeaderboardConfig.asset`(skip-worktree).
- 효과: 점수 1바이트만 바꿔도 서명 깨짐 → 스니핑+수정 공격 차단. 공격자는 바이너리 리버싱 강제.

### L2 — IL2CPP 빌드 전환
- 현재 Standalone = **Mono**(`Assembly-CSharp.dll` 즉시 디컴파일 → 키 노출). IL2CPP 네이티브로 추출 난이도 ↑.
- 사전: Windows IL2CPP 모듈 + VS C++ 빌드툴 설치(빌드 PC마다).

### L3 — SecureInt 메모리 난독화
- `_mask`(랜덤) + `_enc(value^mask)` + `_check`(이중 마스크) 구조체. `Value` getter에서 복호 + 변조 감지.
- `WalletManager._jelly`/`_stone` 백킹 필드만 교체. `Jelly`/`Stone` 프로퍼티 시그니처 불변 → 호출부 무수정.
- 인스펙터 표시는 `#if UNITY_EDITOR` 디스플레이 미러로 보존(권위값 아님).
- 의존성 0 자작(프로젝트 ethos). 효과: Cheat Engine 정수 스캔이 `value^mask`만 발견 → 실패.

### L4 — 저장 무결성 (PlayerPrefs · leaderboard.json)
- 저장 시 `HMAC(localKey, 전 값 정준결합)` 체크섬 동봉. 로드 시 재계산 비교 → 불일치면 리셋/무시 + 경고.
- 리더보드 제출 경로는 아니지만(점수=메모리 Stone) 통계·로컬랭킹 위조 방지.

### 서버 보강 — rounds 기반 상한
- `MAX_SCORE` 플랫(150k) → `maxScoreFor(rounds)` 룩업. 정확한 수치는 **balance 에이전트 산정**
  (현 7라운드 이론최대 ≈69,930). 임의 공식 발명 금지.

## 3. `.claude/` 설정 위생 (이번 감사 — D등급)

`settings.local.json` allow 목록에 임의 실행 와일드카드 누적:

| 등급 | 항목 | 문제 |
|------|------|------|
| CRITICAL | `powershell -Command:*`·`python:*`·`python3:*`·`mcp__Windows-MCP__PowerShell`·`Desktop_Commander__start_process` | 무프롬프트 임의 코드 실행 → 인젝션 시 RCE |
| HIGH | `reg delete *` | 레지스트리 삭제 자동 승인 |
| HIGH | `git push:*` | 자동 push(오너 정책 "수동 push"와 모순) |
| MEDIUM | `npx -y @context7/mcp` · 서브에이전트 "All tools" | 공급망 / 무제한 툴 |
| GOOD | 설정 파일 하드코딩 시크릿 0 · `.mcp.json` 비어있음 | 시크릿 분리 양호 |

권고: 임의실행 와일드카드 제거·구체 명령 재범위화, `reg delete *` 삭제, `git push:*` 재고.

## 4. 시크릿 로테이션 Runbook

1. 신규 토큰: `node -e "console.log(require('crypto').randomBytes(24).toString('hex'))"`
2. 서버 등록: `npx wrangler secret put WRITE_TOKEN`(또는 `SIGN_KEY`) → 값 붙여넣기
3. `LeaderboardConfig.asset`(skip-worktree) + `SECRETS.local.txt` 갱신
4. 구 토큰은 그 시점부터 403 거부. **다른 PC는 skip-worktree라 pull로 안 바뀜 → 수동 입력**

> 시크릿은 코드·git 절대 금지. 노출 발견 시 즉시 위 절차로 로테이션(git 히스토리는 되돌릴 수 없음).

## 5. 6대 보안 점검 체크리스트 (단일 출처)

1. **크리덴셜 노출** — Assets/·config·git history에 token/key/secret 하드코딩 0
2. **리더보드 통신** — HMAC `X-Sig` 본문서명 · 토큰 헤더 · 서버 타당성(rounds 상한)
3. **저장 무결성** — PlayerPrefs/파일 HMAC 체크섬
4. **재화·배팅** — SecureInt 메모리 난독화 · `Reward()` 상한 · 입력 clamp
5. **`.claude/` 위생** — allowlist 임의실행 · 시크릿 분리 · 인젝션 surface
6. **빌드 하드닝** — IL2CPP · 코드 스트리핑 · 에디터 전용 디버그 가드(`#if UNITY_EDITOR`)

## 6. 구현 순서 (DEFERRED — 별도 착수)

1. `CryptoSign.cs`(HMAC hex) → `LeaderboardService.Adapter_BuildSubmit` `X-Sig` 추가
2. Worker `handleSubmit` 서명 검증(`request.text()`) + rounds 상한 + `wrangler secret put SIGN_KEY` 배포
3. `SecureInt.cs` → `WalletManager` 적용
4. PlayerPrefs / `leaderboard.json` 체크섬
5. Standalone IL2CPP 전환 + 릴리스 빌드 1회 검증
6. 크로스랭귀지 HMAC 테스트벡터(C#↔JS 일치) + curl 변조 테스트(X-Sig 없음→403, 바디 수정→403)
