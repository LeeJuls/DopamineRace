## 요구사항

### R1. 폰트 언어 기반 적용
- TitleScene 언어 선택 기준으로 폰트 분기
  - KR 선택 → `koreanFont` (PFstardust3_0_ExtraBold) 전체 적용
  - 그 외 언어 → `mainFont` (NeoDunggeunmoPro-Regular) 전체 적용
- CharacterInfoPopup 내 모든 Text 포함
- RadarChart 동적 생성 Text에도 적용 (플리커 없이)

### R2. 경기기록 스크롤 동기화
- 단거리 / 중거리 / 장거리 RanksScroll 3개
- 어느 하나를 스크롤하면 나머지 2개도 동시에 같은 위치로 이동
- 수평 스크롤 기준

### R3. 스탯 패널 다국어
- CharacterInfoPopup 스탯 이름 (속도/힘/용기/침착/인내/운)
- StringTable 키 기반으로 7개 언어 지원
- 하드코딩 금지 — `Loc.Get("str.ui.char.stat.*")` 사용

### R4. myBetText 크기 개선
- 레이싱 HUD의 내 배팅 표시 텍스트
- 기존 대비 30% 크게 (fontSize 20 → 26)

## 설계

### 폰트 시스템

```
FontHelper.GetFontForCurrentLanguage()
  └─ Loc.CurrentLang == "ko" → GameSettings.koreanFont
  └─ 그 외 → GameSettings.mainFont
```

- `SceneBootstrapper.Awake()` → 씬 전체 Text에 한 번 적용
- `CharacterInfoPopup.ApplyFontAll()` → 팝업 열릴 때 + RadarChart 생성 후 재적용
- `RadarChart.theme.common.font` → 레이블 생성 전에 미리 설정

### 스크롤 동기화 (LateUpdate 폴링)

`onValueChanged` 리스너가 Unity 6 환경에서 불안정 → LateUpdate 폴링 방식 채택

```csharp
// LateUpdate 매 프레임 실행
float sH = _shortScroll.horizontalNormalizedPosition;
// 변경 감지 → 3개 동시 설정
_shortScroll.horizontalNormalizedPosition = targetH;
_midScroll.horizontalNormalizedPosition   = targetH;
_longScroll.horizontalNormalizedPosition  = targetH;
```

## Pixem 업데이트 정책

Pixem 패키지 업데이트 시 커스텀 코드 소실 방지:

1. 업데이트 전 `Docs/backup/Pixem_backup_YYYYMMDD/` 에 수정 파일 백업
2. 업데이트 후 diff 비교 → 우리 코드 머지
3. 수정 대상 파일: `PixemManager.cs`, `PixemSaveDialog.cs`

### 커스텀 기능 목록 (항상 보존)
- `_currentPrefabName` 필드 (커스텀 이름 저장)
- `CreateCharacterPrefab()` → `PixemSaveDialog.Show()` 호출
- `DoSaveCharacterPrefab(string customName)` 메서드
- `GetCharacterFolderPath(string name)` 오버로드
- `LoadPrefabList()` 커스텀 이름 정렬 로직
- `RefreshDataNumber()` null 체크
- `ProcessAll()` `_currentPrefabName` 기반 경로
