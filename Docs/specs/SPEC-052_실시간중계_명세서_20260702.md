## 0. 개요 / 오너 확정 결정

레이스의 두 상태 "트랙 대기(Betting)"·"레이스 중(Racing)"에서 **황제 도파미나우스 3세가 전광판으로 실시간 해설**하는 관측 전용 중계 시스템. 세계관 정본상 황제는 "황실 공식 해설 아나운서 1등급"이라 시스템 자체가 세계관의 게임플레이 페이오프.

| # | 결정 항목 | 확정 |
|---|-----------|------|
| 1 | 캐스팅 | **황제 1인 단독** (FACT형 담백 / COLOR형 개그 2계열) |
| 2 | 정보 수위 | **공개 정보만**(배당·인기·복병). 공략 힌트 없음 → balance EV 검토 불필요 |
| 3 | 연출 | **상단 전광판 티커** 1줄 + 황제 초상 |
| 4 | 유저 호칭 | **2인칭 사용**("시민들이여") — 포고령 77호 근거 |
| 5 | 1인칭 | **"마이크 모드=짐"** — 중계=짐 / 프롤로그 대면=나 (방송 페르소나) |

**기술 타당성(사전 조사 결론)**: 두 이슈 모두 초록불. ① 상태·이벤트 관측은 신규 훅 없이 기존 인프라로 가능. ② 다국어 텍스트는 "템플릿 + `{0}` 치환"으로 규칙 준수하며 구현 가능.

## 2. ★황제 화법 정본 (setting·scenario 리뷰 핵심)

> 중계는 상시 노출 장치라 화법 정본 없이 P2/P3로 증식하면 톤 드리프트가 확정된다. 아래를 SPEC 규범으로 고정한다.

### 2-1. 1인칭 — 마이크 모드=짐
- **중계/전광판 방송 = "짐"**, 프롤로그 대면 대사 = "나" 유지. 모순은 **"방송 페르소나"**로 캐릭터화(황립 해설 아카데미 수석 졸업생 — 아카데미를 본인이 설립했다는 정본 개그와 맞물림).
- **선행 의존 작업**: 프롤로그 전광판 장면(SCENE 3) 대사를 "짐"으로 소폭 조정 → scenario 핸드오프(§10). 대면 장면은 "나" 보존.

### 2-2. 언어별 1인칭 캐스팅표 (번역 착수 전 확정 필수)
| 언어 | 황제 1인칭 |
|------|-----------|
| ko | 짐 |
| en | royal We ("We are not surprised") |
| jp | 朕 + 尊大語 |
| de | Wir (Pluralis Majestatis) |
| cn·tw | 朕 / 칙령체(欽此) |
| es·br | 장엄복수 사어 → "Yo, el Emperador" 등 3인칭 자칭 대안 |

### 2-3. 유머 3원칙
- **(a) 유머 엔진 = "극도의 진지함 × 대상의 사소함".** 과녁은 제국 제도(역사서·국고·칙령·배당판)이며 **소녀(선수) 폄하 절대 금지** — 정본 황제는 부상자에게 쾌유 카드를 쓰는 인물.
- **(b) FACT형도 황제 선언체 유지** (중립 캐스터체 금지). FACT/COLOR 차이는 어조가 아니라 **내용**: FACT=사실만, COLOR=관료장치 개그. 예) "{0}, 지금 제국의 맨 앞."(FACT)가 "맨 앞이다!"보다 황제답고 티커에서 강함.
- **(c) 재치·자기언급 개그는 레어 변주(1~2%)로 강등** — 격식이 드물게 깨져야 웃김(반복 시 예능 아나운서로 평평해짐).

### 2-4. 금지어
| 금지 | 사유 | 대체 |
|------|------|------|
| "~마"(인기마·복병마) | 헌법 제8조 동물레이스 금지, 주자는 소녀 | "인기 선수", "복병"(군사 은유 OK) |
| 연차·기간 수치("수십 년간") | 타임라인 붕괴(황제 재위 6년차) | 회피 |
| 시민 대상 도파민 가든 위협 | 포고령 77호(시민 환대 의무) 충돌 | 가든 언급은 대신·학자 대상 과거형만 |

## 4. 멘트 템플릿 설계

### 4-1. 바리에이션·반복 내성 (계열 비대칭)
- **FACT**(정보): 반복돼도 무해. 고빈도 트리거당 4~5개 + L1 타입 4버킷 얹으면 실효 풀 ~12 → 세션 내 충분.
- **COLOR**(농담): 2회차에 죽음. 트리거당 **≥4개 + 세션당 동일 COLOR 1회 + 셔플백**(순수 랜덤 금지, 최대 간격 보장).

### 4-2. 치환 규칙 (8언어 생존)
- 최대 **2치환**: `{0}`=캐릭터명(DisplayName), `{1}`=순위(`Loc.GetRank`)·트랙명·수치.
- **황금 구문 = 작성 규칙 1번**: `"선두 교체 — {0}!"`(대시+동격) → ko 조사·de 관사·es/br 성수를 동시에 회피.
- **ko 조사**: 교체형 조사만 회피(이/가·은/는·을/를·과/와). **불변 조사(에게·의·도·만·부터·처럼)는 `{0}` 뒤 안전.** 2치환의 "과/와"는 쉼표·"그리고"로 대체.
- **es/br 여성형 고정** (출전자 전원 소녀).
- **문장 전체 = 키 1개.** 구 단위 짜깁기 금지.

### 4-3. 개성 3계층
| 계층 | 방식 | 비용 | Phase |
|------|------|------|-------|
| L0 범용 | 누구에게나 동일 템플릿 | 기준 | P1 |
| L1 타입 특화 | 스타트·스퍼트 등 타입 개성이 갈리는 2~3개 상황만 4타입 분기 | 해당 상황 키 4배 | P2 |
| L2 별명 | `str.char.NNN.nickname` 32키 — `{0}` 자리에 이름 대신 확률 삽입(첫 노출 시 이름 병기). 황제 취미 "선수 별명 작명"이 근거, 황제 작명이라 화법 준수 | +32키(1회성) | P3 |

> **캐릭터 전용 멘트(L3)는 비목표.**

### 4-4. 관계망 2치환 범용 템플릿 (scenario 고가성비 레버)
- 근접 페어 감지(거리 X·Y초) × 관계 4버킷(❤호감·⚔경쟁·★존경·✗비호감) × 2~3변주 = **8~12키로 관계 매트릭스 100+쌍 전부 커버**. 치환자 2개 규칙과 정합.
- **"무내용 암시" 기법**(번역 리스크 0): "저 둘의 사연을 짐은 안다. 안다고만 해두겠다." — 시나리오를 읽은 유저에겐 페이오프, 안 읽은 유저에겐 떡밥.
- 전용 페어키(침묵⚔황금호 등)는 **특급 3쌍 상한으로 P3 격리**(조합폭발 차단).

### 4-5. 라운드 결과 = 인기/복병 분기 (필수)
"예상대로…라고 기록하라" 아이러니는 **복병 우승 때만** 작동. 인기 우승 시 재생되면 개그 오발 → 배당 데이터로 분기(공개 정보 범위라 결정 #2와 무충돌).

## 5. StringTable 키 체계

**명명**: `str.cast.{상태}.{트리거}.{n}` (기존 `str.{영역}.{기능}.{항목}` 준수, 신규 영역 `cast`). 타입 분기는 `str.cast.race.start.runner.{n}`.

| 구간 | 구성 | 키 수 |
|------|------|-------|
| P1 대기 | opening 3·track 3·favorite 3·longshot 2·recap 2·countdown 1 | 14 |
| P1 실황 | start 3·lead 4·surge 3·finallap 2·reversal 2·lastplace 3·photo 2·finish1st 3·finish_userpick 2·podium 2 | 26 |
| P1 결과·UI | hit 3·miss 3·roundend 2·화자명/직함 등 3 | 11 |
| **P1 합계** | | **~51키** |
| P2 증분 | crit·skill·collision·slipstream·타입분기·유저픽 전용·관계망 8~12 | +40± |
| P3 증분 | nickname 32·특급 페어·게스트 | +40± |

**길이 예산(검증 게이트)**: `str.cast.*`는 **ko/jp/cn/tw ≤22자, en/de/es/br ≤55자**. 위반 시 검증 실패(런타임 축소는 2차 방어).

## 6. 연출 / UI (design 리뷰 핵심)

### 6-1. 슬롯 — "동일 위치"는 화면좌표가 아니라 **동일 컴포넌트 재사용**
현재 상단은 이미 점유됨(Betting: TopArea 상단 18% 전폭 / Racing: `myBetText`·라운드·타이머). 따라서:
- **Racing**: `myBetText` 자리를 티커가 **승계**. 티커 유휴 시 기본 표시문("짐이 보건대 그대의 픽은 {0}")으로 흡수 → 기존 UI 1개 순감.
- **Betting**: TopArea 내부 하단에 전용 밴드(높이 ~40px) 추가.

### 6-2. 캔버스 계층 규칙
- 티커 루트 = 상태 UI(betting/racing)의 **형제로 독립**(SetActive 토글에 딸려 소멸 금지).
- countdownUI 딤·모달보다 **아래**. **Countdown 중 숨김**(멘트는 직전 소진). **Result/Finish/GameOver 진입 시 큐 flush.**

### 6-3. 렌더 규격
- **전용 반불투명 플레이트 필수**(Racing 상단 무배경 → 픽셀 스프라이트 위 판독 불가). "전광판" 프레임으로 명분화.
- 교체형(마키 스크롤 금지 — 픽셀폰트 뭉갬·언어별 속도 편차). 페이드 150~250ms 1회.
- **울트라와이드**: 중앙 앵커 고정폭(≤900px), 클램프 패턴 준수(RacingUI엔 레터박스 없음).
- **폰트**: `FontHelper.GetMainFont()`/MkText 경로(CJK 라우팅 SPEC-048 상속). 티커 하한 **14~16px**(전역 10px는 판독불가 하한).
- **아이콘+이름 선행 규격**("안 읽혀도 의미 전달"): 이미 레포 유입된 `star.png`(인기)·`darkhorse.png`(복병) 아이콘 재활용(텍스트는 "선수"로, 아이콘만 시각 기호). FACT형 = "이름 선행 + 판정 후행".
- **톤 시각화**: 텍스트 색 변경 금지(색약). **초상 표정 2프레임 스왑**(FACT=근엄 / COLOR=씨익).
- 프리팹은 `BettingUIPrefabCreator`(MkChild/MkTextObj) + Decorator 패턴 편입.

## 7. 타이밍 / 스팸 제어 (수치 명시)

| 장치 | 규칙 |
|------|------|
| 우선순위 | **P0** 완주·역전·최종바퀴 > **P1** 선두교체·유저픽 > **P2** 스킬·크리 > **P3** 충돌·슬립스트림 |
| 유저 픽 부스트(G1) | 유저 배팅 캐릭터 이벤트 우선순위 +1. **커버리지(노출 빈도)이지 응원 아님** — 황제 중립성 유지(관계도 "공식적으로 편들지 않음") |
| **히스테리시스**(틀린 중계 방지) | 선두교체 = **1.0s 이상 유지 + 진행도 격차 임계 통과** 시만 발화. 순위급상승 = **N계단(≈5)·2s 내** |
| 큐 | 최대 2개, 초과분 오래된 순 drop. P0는 선두 삽입 + 현재 멘트 조기 종료 허용 |
| **이벤트 TTL** | 모달 홀드 후 재개 시 stale 컷오프(닫은 뒤 "10초 전 선두교체" 재생 금지) |
| 쿨다운 | 유형별 전역(크리티컬 6s·충돌 8s) + 동일 캐릭터 언급 5s(P0 제외) |
| **상태별 지속시간** | 대기 6~8s / 실황 3~4s, 최소노출 1.5s |
| **HideInfo 상호작용** | `DR_HideCharInfo` ON 시 인기/복병 계열 멘트 억제 or 중립 변형(유저가 끈 정보 재주입 금지) |
| 밀도 목표 | 실황 ≤1개/5s / 대기 8~12s당 1 |
