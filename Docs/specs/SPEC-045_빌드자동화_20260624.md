## 1. 목표
Steam 출시용 **IL2CPP** Windows 빌드 + 빌드 자동화(`[MenuItem]`/batchmode). Mono는 `Assembly-CSharp.dll` 즉시 디컴파일 → 보안 Layer C(SPEC-044) + 성능 + Steam 표준 위해 IL2CPP.

## 2. IL2CPP 전환 (Phase 1 — 완료)
`unity_execute_code`로 PlayerSettings 정정 + `ProjectSettings.asset` 커밋(spec023 유실 재발 방지).

| 항목 | 이전 | 변경 |
|------|------|------|
| Standalone scriptingBackend | **키 부재**(기본 Mono2x) | IL2CPP (키 신규 생성) |
| applicationIdentifier | com.Company.ProductName | com.Dopamine.DopamineRace |
| bundleVersion | 0.1 | 0.1.0 |
| managedStrippingLevel(Standalone) | 키 부재 | **Disabled 명시** |
| Il2CppCompilerConfiguration | Release | Release |
| stripEngineCode | **1** | **0** |

## 3. 검토 교훈 (build 엔지니어 코드 실측 — 정정 5건)
1. Standalone scriptingBackend 키 자체가 없었음 → "전환"이 아니라 "신규 set". stripping도 명시 set 안 하면 IL2CPP가 Low 폴백 → 보안 직렬화 strip.
2. `stripEngineCode:1` 이미 ON → IL2CPP에서 `Cryptography`/`UnityWebRequest` 백엔드 strip 위험 → 0 (안정 후 점진 복원).
3. **첫 IL2CPP 빌드 MCP `unity_build` 금지** — 첫빌드>timeout → 좀비+산출물락. 에디터GUI/batchmode 1회 워밍 후 incremental.
4. AhnLab V3 빌드 예외 선등록(`Builds/`·`Library/Bee/`·`Library/il2cpp_cache/`·`%TEMP%`) — 빌드락 + HMAC/XOR 산출 exe 오탐.
5. 🟢 보안코드 AOT 안전 확인(`HMACSHA256` 구체생성자·`SecureInt` struct·구체 `[Serializable]`) → stripping Disabled면 첫 빌드부터 동작. 깨지면 원인은 AOT 아닌 2/3/4.

## 4. 빌드 스크립트 (완료)
- `Assets/Scripts/Editor/DopamineBuilder.cs` — `[MenuItem]` `DopamineRace/Build/`(Dev·Release·Release+SteamCopy) + **batchmode 진입점 `CIRelease`**(MCP timeout 회피). `EnsureIL2CPPSettings`로 빌드 직전 백엔드/stripping/engineCode 강제(인스펙터 변경 무력화). steam content 복사 시 `*_BurstDebugInformation_DoNotShip/`·`*.mac`·`*.pdb` 제외, 경로는 `EditorPrefs`.
- `Assets/Scripts/Editor/DopamineBuildVerify.cs` — 화이트(`GameAssembly.dll`·`il2cpp_data`·exe·`UnityPlayer.dll`) + 블랙(`Assembly-CSharp.dll`·`MonoBleedingEdge`) 게이트. 빌드 직후 자동 + 메뉴 수동.

## 5. 검증 게이트 (qa 매트릭스)
- **Phase 1 빌드마커(BM)**: GameAssembly.dll 존재 + Assembly-CSharp.dll/MonoBleedingEdge 부재 + il2cpp_data.
- **Phase 2 기록보관 회귀(RG, 최우선 Critical)**: IL2CPP 실행본에서 leaderboard.json+`.mac` 저장/로드(FP0), PlayerPrefs StatsMAC, `Stone==제출점수`, deviceUniqueIdentifier 안정성. **AOT 무음실패는 예외 안 던짐 → 데이터 존재·일치 직접 대조.**
