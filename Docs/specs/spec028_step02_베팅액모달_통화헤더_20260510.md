## 배경

Phase 1에서 만든 통화 시스템(WalletManager·BetReward·1.1 클램프)을 UI에 노출. 유저가 직접 베팅 금액을 정하고, 보유 통화를 항상 확인할 수 있게 한다. **기존 베팅 화면은 거의 손대지 않고**, START 버튼 클릭 시 별도 모달을 띄우는 단순화된 흐름 채택 (오너 결정).

## 요구사항

- **R2.1** 베팅 화면 최상단 헤더에 `🟦 젤리 X / 💎 스톤 Y` 표시 — **기존 `총점` 영역을 교체** (UC9).
- **R2.2** 베팅 타입별 보상 라벨을 "획득 포인트" → "획득 도파민 스톤(💎)"으로 변경 (UC10).
- **R2.3** START 버튼 클릭 시 **베팅액 입력 모달** 팝업 (별도 화면 X, 모달 오버레이).
- **R2.4** 베팅액 입력 UI: **슬라이더 + 키보드 입력 필드 + 4개 빠른 버튼(-10/-1/+1/+10)** 3중 동기화.
- **R2.5** 베팅액 기본값: 보유 젤리의 25% (예: 100 → 25).
- **R2.6** 미리보기: 적중 시 받는 젤리 (`+N🟦 (+N💎)`) / 실패 시 손실 (`-N🟦`) 실시간 갱신.
- **R2.7** 잔액·범위 검증: 베팅액 < 1 또는 보유 젤리 초과 시 자동 클램프 + Start 버튼 비활성.
- **R2.8** 보유 젤리가 1 미만이면 모달 진입 자체 차단 → 즉시 GAME OVER 화면(Phase 3) 호출.
- **R2.9** `[배팅 확정]` 클릭 후 베팅 변경 불가 (즉시 라운드 시작).
- **R2.10** 모든 신규 텍스트는 StringTable UID 키로 등록 → 7언어 (한·영 우선, 나머지 fallback).

### 흐름 (시퀀스)

```
1. [유저] 배팅 타입 선택 (단승/연승/...)
2. [유저] 캐릭터 선택 (1~3마리)
3. [유저] START 버튼 클릭
4. [SceneBootstrapper.OnStartButtonClick]
   → if (Wallet.Jelly < 1) → GameState.GameOver 진입 (Phase 3)
   → else → BetAmountModal.Show(currentBet)
5. [BetAmountModal] 모달 표시. 기본값 = ceil(Wallet.Jelly * 0.25)
6. [유저] 슬라이더/키보드/빠른 버튼으로 베팅액 조정
7. [BetAmountModal.RefreshPreview] 매 변경마다 미리보기 갱신
8. [유저] [배팅 확정] 클릭
9. [BetAmountModal.OnConfirm]
   → currentBet.betAmount = _amount
   → Wallet.TryBet(_amount)         // jelly 차감
   → callback: SceneBootstrapper.StartRound(currentBet)
   → Hide()
10. 레이스 진행 (기존)
```
