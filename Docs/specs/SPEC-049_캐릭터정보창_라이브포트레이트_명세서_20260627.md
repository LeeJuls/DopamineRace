# SPEC-049 캐릭터 정보창 라이브 포트레이트 (PNG → mp4)

> 캐릭터 정보창 일러스트를 살아 움직이는 mp4로. 8개 비디오 + 24개 PNG 하이브리드, Standalone 전용.
> 총괄: `feature-director`. plan: `.claude/plans/4-adaptive-meteor.md`.

## 1. 목표
"유저가 캐릭터 보는 맛이 우마무스메처럼 쫄깃한가" — 정보창에서 캐릭터가 생동감 있게 움직이도록. 비디오 재생 인프라(VideoPlayer→RenderTexture→RawImage) 신규 구축.

## 2. §0 선행 측정 결과 (2026-06-27, MCP 실측)

| N | 비디오(w×h) | 종횡비 | fps | 길이 | PNG | Δ종횡비 |
|---|------------|--------|-----|------|-----|---------|
| 0 | 480×480 | 1.000 | 16 | 5.1s | 256×256 | 0.0% |
| 1 | 480×480 | 1.000 | 32 | 5.0s | 256×256 | 0.0% |
| 2·3·5·7·23·26 | 640×640 | 1.000 | 24 | 5.0s | 256×256 | 0.0% |

**확정**:
- **레이아웃**: `AspectRatioFitter.HeightControlsWidth` 유지 (비디오·PNG 모두 1:1 정사각, Δ0% → 마스크 크롭/흰여백 리스크 해소, 24 PNG 회귀 0)
- **RT 해상도**: 640×640 (최대 소스, 정사각), `filterMode=Point`
- **showMaskGraphic**: 현행(1) 유지 — 종횡비 일치로 여백 노출 없음
- **코덱**: H264. fps 16/24/32 혼재 → §E에서 24 통일 검토(1차 원본 유지)

## 3. 확정 결정
1. 범위: 캐릭터 정보창만, 하이브리드 폴백(비디오 없으면 PNG)
2. 플랫폼: Standalone만 (WebGL 미대상 → 자동 PNG 폴백)
3. 데이터: `CharacterDB_V4.csv` `char_video` col20 추가, 빈 값이면 `char_illustration`(PNG)
4. 영상 크기: BettingPanel 프리팹에서 디자이너 인스펙터 조절 (런타임 RectTransform 미변경)

## 4. 데이터 — char_video col20
실측 헤더 col0~col19(마지막 char_passive). **char_video를 col20 맨끝 추가** → 기존 인덱스 불변, 오프셋 0.
- 8개 행(0·1·2·3·5·7·23·26)만 `Icon/Char/video/Character_N`, 24개 빈칸
- `CharacterData`/`CharacterDataV4` 파서 `cols.Length>20` 가드
- `VideoResourcePath` getter: charVideo 있으면 경로, 없으면 null → PNG 폴백

## 5. 아키텍처 — Image + RawImage 오버레이
- `IllustrationMask` 자식에 `IllustrationVideo`(RawImage) 추가, 기존 `Illustration`(Image, PNG)과 형제
- 비디오 O → RawImage 활성 + `illustration.enabled=false` / X → Image만 (PNG 무손상)
- `VideoPlayer`(RenderTexture·audioOutputMode=None·isLooping·waitForFirstFrame·skipOnDrop) → RT(640², Point, 1개 재사용) → RawImage(maskable, raycastTarget=false)
- RectTransform 프리팹 값(런타임 미변경), 패처는 초기값만 Illustration 복사
- **PNG↔비디오 크로스페이드 150~200ms**, charId 가드, errorReceived→PNG 폴백
