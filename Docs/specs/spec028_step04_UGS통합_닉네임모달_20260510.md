# SPEC-028 step04 — Phase 4: UGS 통합 + 닉네임 입력 모달

**날짜**: 2026-05-10
**상태**: 작성 (구현 대기)
**선행**: spec028_step01 (Phase 1 통화) + spec028_step03 (Phase 3 Finish 화면)
**후속**: spec028_step05 (글로벌 랭킹 뷰)
**마스터**: spec028 (개요)

## 배경

이번 Phase가 백엔드 연결 핵심. UGS (Unity Gaming Services) Authentication·Leaderboards를 통합해 누적 스톤을 글로벌 랭킹에 제출하고, 마지막 라운드 완주 + 1만등(또는 운영값) 컷오프 진입 시 **옛날 오락실 점수 등록** 스타일 닉네임 모달을 띄운다.

## 요구사항

- **R4.1** UGS Package 설치: `com.unity.services.core` + `authentication` + `leaderboards`.
- **R4.2** UGS Dashboard 프로젝트 생성 + Leaderboard ID `dopamine_stone_global` 발급 (capacity 10000, sort DESC) — **오너 작업**.
- **R4.3** TitleScene 진입 시 백그라운드 비동기로 UGS 초기화 + 익명 로그인 (PlayerID 자동 발급).
- **R4.4** `LeaderboardService` 신설 — 4개 메서드: `GetCutoffScoreAsync`, `SubmitStoneAsync`, `GetTopAsync`, `GetMyRankAsync`.
- **R4.5** 모든 LeaderboardService 메서드는 try-catch + 오프라인 폴백 (네트워크 실패 시 로컬 저장 + 알림).
- **R4.6** 마지막 라운드 완주 시 cutoff 비동기 조회 → 본인 스톤 ≥ cutoff면 닉네임 모달 표시, 미진입이면 "랭킹 미진입" 안내.
- **R4.7** 닉네임 모달: 1~8자 길이 + 한·영·숫자만 + 차단어 검사 + **중복 검사** 통과 시 등록.
- **R4.8** `BadwordList.csv` 신설 — 운영 가능한 차단어 목록 (오너 작성, 빈 파일도 OK).
- **R4.9** `BadwordFilter` 클래스 — 부분 일치 검사 (`bool IsClean(string)`).
- **R4.10** 점수 제출 성공 시 `WalletManager.ResetStoneOnly()` 호출 (스톤 0으로 리셋, 다음 게임 준비).
- **R4.11** 닉네임 변경 불가 — 한 번 등록되면 재로그인 시에도 동일 닉네임 사용 (PlayerPrefs `registered_nickname` 키).
- **R4.12** 모든 신규 텍스트는 StringTable UID 키로 등록 (7언어).

## 비고

- UGS 무료 한도(50K MAU)는 100 DAU 미만 가정에서 충분.
- Unity 6에서 UGS Leaderboards `GetEntriesAsync` 페이지네이션 사용 (offset, limit).
- `GetCutoffScoreAsync` 구현 시 entries 수가 capacity 미만이면 0 반환 (모두 진입 가능 의미).
- 닉네임 중복 검사는 클라이언트 1차 + UGS PlayerName 등록 시 서버 측 중복 검출 둘 다 활용.
- 예외 처리 강화 (UC5 반영): 모든 검증 단계는 순차 실행 + 사용자 친화적 에러 메시지.

## 학습된 제약 (반영)

- Unity 6의 UGS는 `using Unity.Services.Authentication;` 등 새 namespace 사용
- `AddPlayerScoreAsync`는 default로 "update only when greater" 정책 — Reset 필요 없음
