# SPEC-028 step05 — Phase 5: 글로벌 랭킹 뷰 + TitleScene 진입점

**날짜**: 2026-05-10
**상태**: 작성 (구현 대기)
**선행**: spec028_step04 (Phase 4 UGS 통합)
**후속**: 없음 (마지막 Phase)
**마스터**: spec028 (개요)

## 배경

UGS Leaderboard에 등록된 점수를 유저가 직접 확인할 수 있는 **글로벌 랭킹 뷰**. 메인 메뉴(TitleScene) START 버튼 위쪽에 진입점 버튼 추가 (오너 결정 UC8 — 대기 화면에서 즉시 도파민 자극). Finish 화면에서도 등록 후 자동으로 랭킹 화면 진입.

## 요구사항

- **R5.1** TitleScene **START 버튼 바로 위**에 "🌍 글로벌 랭킹" 버튼 추가 (대기 화면 진입점).
- **R5.2** Finish 화면(랭킹 등록 직후 또는 미진입 시점)에 "글로벌 랭킹 보기" 버튼 추가.
- **R5.3** 기존 로컬 Top100 버튼은 "**내 기록**"으로 리네임 (혼동 방지).
- **R5.4** 글로벌 랭킹 뷰: Top 10,000 페이지네이션 (50개씩 × 200페이지).
- **R5.5** 본인 위치 자동 하이라이트 + "본인 페이지로 이동" 버튼.
- **R5.6** [새로고침] 버튼 → `GetTopAsync()` + `GetMyRankAsync()` 재호출.
- **R5.7** 네트워크 오류 시 빈 리스트 + 에러 토스트 + 새로고침 버튼 강조.
- **R5.8** 화면 진입 시 자동 1페이지 로드 (또는 본인 위치가 명확하면 본인 페이지).
- **R5.9** 모든 신규 텍스트는 StringTable UID 키로 등록 (7언어).
- **R5.10** 페이지네이션 50개 × 200 = Top 10,000까지 조회 가능 (Phase 4 capacity와 일치).

## 학습된 제약 (반영)

- ScrollRect의 ContentSizeFitter + LayoutGroup 조합 시 RectTransform 갱신 1프레임 지연 → `LayoutRebuilder.ForceRebuildLayoutImmediate` 호출 필요
- UGS GetEntriesAsync는 한 번에 최대 100개 권장 (50개씩이면 API 호출 부담 적음)
- 본인 위치는 entries 페이지네이션 결과로 직접 검색 가능 (`PlayerId == 본인` 비교)
- 랭킹 항목 50개를 매 페이지 재생성하면 GC 부담 → ObjectPool 검토 (선택)
