# SPEC-030 — 결승선 도착 시 트랙 이탈 버그 수정

**날짜**: 2026-05-16
**상태**: ✅ 구현 완료 (GATE A·B·D PASS, C는 오너 수동)
**개발 방식**: Harness Engineering (사전 진단 → 수정 → 결정적 검증)

## 배경

일부 레이서가 결승선 도착 후 트랙(레인) 밖(잔디/안쪽)에 **정지한 채** 표시됨.

## 근본 원인

도착 스냅 좌표 = `WP0 + outward*(laneOffset + deviationOffset)` (`RacerController.GetOffsetPosition`).
`deviationOffset`(랜덤 경로이탈)이 도착 진입 시 **0으로 초기화·클램프되지 않아** 그대로 합산.

- `GameSettings.asset` 실제값: `pathDeviation=0.3`, `laneOffset=0.19` (코드 default와 다름)
- 레인 밴드 = `RACER_COUNT/2 * laneOffset` = ±0.855
- 수정 전 도착 최악: `|laneOffset|(0.855) + dev(0.3)` = **1.155 → 레인밴드 0.300 초과 이탈**
- 비입상 레이서는 `RaceManager.OnRacerFinished`에서 재배치 안 됨(`GetPodiumPosition`은 입상권만) → 빗나간 스냅 고정 노출

## 요구사항

- **R30.1** 결승 진입(`headingToFinish`) 후 랜덤 경로이탈 정지 → 레인 정렬로 도착
- **R30.2** 횡오프셋을 레인 밴드 내로 클램프 → 어떤 경우에도 트랙 이탈 0
- **R30.3** 레이스 순위/속도(HP·CP) 무영향 (deviation은 시각 횡오프셋 전용)

## 변경 파일

| 파일 | 변경 |
|------|------|
| `Assets/Scripts/Racer/RacerController.cs` | `UpdateDeviation()` 최상단 `headingToFinish` 시 `deviationOffset=0` 후 return / `GetOffsetPosition()` `dev = headingToFinish?0:deviationOffset` + `Mathf.Clamp(laneOffset+dev, ±laneBand)` |
| `RaceBacktestWindow.cs` | **변경 없음** — 시각 좌표 미모델링(속도/HP만 백테스트). CLAUDE.md 동시수정 규칙은 속도/HP 로직 대상이라 본 시각 수정은 비해당 |

부가 효과: `LateralOffset`(`laneOffset+deviationOffset`)도 도착 시 안정 → 트랙바 UI 원 위치도 정상화.

## 비고

- 클램프는 레이스 중에도 작동(최외곽+큰 deviation 시 레인 끝으로 당김) → 항상 트랙 내 유지. 순위는 웨이포인트 진행도 기반이라 영향 없음
- 교훈: GameSettings.asset 실제값이 코드 default와 다름(`pathDeviation` 0→0.3) — 검증 전 asset 값 확인 필수
