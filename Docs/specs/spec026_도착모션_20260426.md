## 배경

레이스 완주 시 모든 레이서가 일률적으로 `Idle` 모션을 재생하던 것을, 사용자의 배팅 입상권 여부에 따라 차등화. 입상권은 환호(`AttackMagic`), 권외는 패배(`Death`) 모션으로 명확한 시각적 피드백 제공.

---

## 요구사항

### R1. 입상권 → AttackMagic 모션
- 사용자가 배팅한 타입의 입상 순위 안에 든 캐릭터는 골인 직후 `AttackMagic` 트리거

### R2. 입상권 외 → Death 모션
- 입상권 밖 캐릭터는 골인 직후 `Death` 트리거

### R3. 배팅 타입별 입상 컷
| BetType | 입상 컷 | 의미 |
|---------|---------|------|
| Win (단승) | 1등 | 1명 |
| Place (연승) | 1~2등 | 2명 |
| Quinella (복승) | 1~2등 | 2명 |
| Exacta (쌍승) | 1~2등 | 2명 |
| Trio (삼복승) | 1~3등 | 3명 |
| Wide (복연승) | 1~3등 | 3명 |

---

## 설계

### 흐름
```
레이서 결승선 도착 → isFinished=true → OnFinished?.Invoke(this)
        ↓
RaceManager.OnRacerFinished(racer)
  → finishCount++; racer.FinishOrder = finishCount
  → prizeCut = CurrentBet.PrizeRankCut
  → racer.PlayFinishMotion(finishCount <= prizeCut ? "AttackMagic" : "Death")
```

### 아키텍처 결정
| 결정 | 이유 |
|------|------|
| `RacerController`에서 Idle 호출 제거 | 입상 판정 정보가 RaceManager에만 있음 |
| `BetInfo.PrizeRankCut` 헬퍼 추가 | 입상 컷 로직 중앙 집중화 |
| `RacerController.PlayFinishMotion(string)` 공개 | RaceManager → RacerController 단방향 호출 |
