# SPEC-050: 아이템 정보 팝업 (도파민 젤리·스톤 클릭 설명)

**작성일**: 2026-06-29 | **상태**: 구현  
**관련 파일**: `ItemInfoPopup.cs` · `ItemInfoTrigger.cs` · `Assets/Prefabs/UI/ItemInfoPopup.prefab`

---

## 1. 배경 / 목적

배팅 화면 우상단의 도파민 젤리·스톤 보유량 표시(CurrencyHeader)는 수치만 보여줄 뿐,
각 재화가 무엇인지 설명이 없다. 신규 유저가 두 재화의 역할을 이해하기 어렵다.

→ **아이템 아이콘 클릭 시 이름·설명을 보여주는 팝업**을 추가한다.
트랙 설명창(TrackInfoPanel)과 같은 스타일의 박스로, 다른 화면의 아이템에도 재사용 가능하게 **모듈화**한다.

---

## 2. 요구사항

| # | 요구 | 처리 |
|---|------|------|
| 1 | 젤리·스톤 클릭 → 이름+설명 박스 표시 | `ItemInfoTrigger` + `ItemInfoPopup` |
| 2 | 트랙 설명창 같은 스타일 (이름 위, 설명 아래) | 흰 9-slice 배경 + 다크 텍스트 |
| 3 | 이름·설명은 StringTable 키로 다국어 | `str.dopamine.item.name/desc.xxx` 8개 언어 |
| 4 | 다른 곳의 같은 아이템도 동일 동작 | 모듈식 트리거 — 키만 지정하면 재사용 |
| 5 | 코드 절차 생성 금지, 프리팹으로 | `ItemInfoPopup.prefab` 인스턴스 배치 |

---

## 3. StringTable 키 (신규)

> **명명 규칙 확정**: 앞으로 아이템 이름·설명은 `str.dopamine.item.{name|desc}.{아이템id}` 형식.

| 키 | 내용(ko) |
|----|----------|
| `str.dopamine.item.name.dopaminejelly` | 도파민 젤리 |
| `str.dopamine.item.desc.dopaminejelly` | 아직 결과를 모르는 기대 에너지가 응결된 통화. 배팅에 사용되며 매 게임 100개가 주어진다. |
| `str.dopamine.item.name.dopaminestone` | 도파민 스톤 |
| `str.dopamine.item.desc.dopaminestone` | 기대가 현실이 됐을 때 결정화된 에너지. 배팅 적중 시 획득하며 글로벌 랭킹 점수가 된다. |

설명 출처: `Docs/specs/SPEC-040_세계관설정집_20260619.md` §8 (아레나 경제).

## 4. 구조 (모듈)

```
ItemInfoPopup (root, RectTransform full-stretch, ItemInfoPopup 컴포넌트)
  Backdrop (Button + 투명 Image, 전체 stretch) ─ 바깥 클릭 시 Hide
  Box (Image: Img_BG_ListSlot_BG_01, Sliced, 우상단)
    NameLabel (Text, 상단, 다크블루)
    DescLabel (Text, 하단, 자동 줄바꿈, 다크그레이)
```

- **`ItemInfoPopup.Instance`** — 씬당 1개. 트리거가 이걸 통해 `Show`/`Hide`.
- **`ItemInfoTrigger`** — `itemNameKey`·`itemDescKey`만 가진 클릭 핸들러(IPointerClickHandler).
  같은 아이템 재클릭 시 토글로 닫힘. 클릭 감지 위해 노드에 raycast Image 필요.

## 7. 향후 확장

- 캐릭터 정보창, 상점 등 다른 화면 아이템에 `ItemInfoTrigger` 부착 + 키 지정만으로 재사용.
- 신규 아이템 추가 시 `str.dopamine.item.*` 키 발급 → 트리거에 연결.
