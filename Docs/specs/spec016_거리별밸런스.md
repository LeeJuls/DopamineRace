## 1. 설계 목표

| 거리 | 유리 타입 | 전략 이유 |
|------|----------|----------|
| 단거리 (2바퀴) | 도주(Runner) | 초반 버스트로 선두 확보 → 짧은 레이스에서 역전 불가 |
| 중거리 (3바퀴) | 균형 | 버스트 구간이 고르게 분포 |
| 중장거리 (4바퀴) | 선입/추입 | 후반 버스트 + 스퍼트 HP 보너스 |
| 장거리 (5바퀴) | 추입(Reckoner) | HP 비축 → 스퍼트 진입 시 폭발적 속도 보너스 |

## 2. 거리별 버스트 구간 랩 환산

진행도 1.0 = 전체 레이스 (랩 수 무관).

| 타입 | 진행도 구간 | 2바퀴 | 3바퀴 | 5바퀴 |
|------|-----------|-------|-------|-------|
| 도주 | 0.10~0.24 | 0.2~0.48랩 | 0.3~0.72랩 | 0.5~1.2랩 |
| 선행 | 0.25~0.40 | 0.5~0.8랩 | 0.75~1.2랩 | 1.25~2.0랩 |
| 선입 | 0.43~0.66 | 0.86~1.32랩 | 1.29~1.98랩 | 2.15~3.3랩 |
| 추입 | 0.50~0.80 | 1.0~1.6랩 | 1.5~2.4랩 | 2.5~4.0랩 |
| 스퍼트 | 0.80~1.00 | 1.6~2.0랩 | 2.4~3.0랩 | 4.0~5.0랩 |

## 3. 핵심 구현: Emergency Drain LapScale

### 3.1 문제점 (도입 전)

단일 `emergencyDrainMul` 값으로는 전 거리 동시 밸런스 불가:
- 2L에서 적당한 drain 값 → 5L에서 Runner HP 과소모 (진동 현상)
- 5L에서 적당한 drain 값 → 2L에서 Runner HP 과보존 (도주 독주)

### 3.2 해결책: lapScale 스케일링

```csharp
// RacerController_V4.cs (및 RaceBacktestWindow.cs 미러)
// 3바퀴 기준, 거리에 비례한 emergency drain 자동 조정
float lapScale = Mathf.Pow((float)GetTotalLaps() / 3f, 0.5f);
drain *= gs.GetV4EmergencyBurstDrainMul(charDataV4.charType) * lapScale;
```

### 3.3 lapScale 값표

| 거리 | lapScale | 효과 |
|------|----------|------|
| 2L | √(2/3) = **0.816** | drain 낮음 → 도주 단거리 유리 |
| 3L | √(3/3) = **1.000** | 기준값 |
| 4L | √(4/3) = **1.155** | drain 증가 |
| 5L | √(5/3) = **1.291** | drain 높음 → 도주 장거리 불리 |

## 6. 튜닝 인사이트

### 6.1 거리 구조 (lapScale)
- **lapScale √(laps/3)**: 자연스럽게 단거리 도주 강, 장거리 도주 약 구현
- **leaderTargetMax**: 정수값이라 3↔4가 이진 스위치처럼 작동 → 4 유지 + drain으로 조절

### 6.2 Drain Mul 스위트 스팟 (사이클 2~5 도출)
- **runnerEmergencyDrainMul = 4.1**: 3.8은 도주 과잉(32%), 4.3은 과약(23%), 4.1이 26~27% 정확히 달성
- **chaserEmergencyDrainMul = 0.9**: 1.3은 C 과약(19%), 0.9는 C 지속 경쟁력 확보 (다만 4L에서 37%로 편향 가능)
- **reckonerEmergencyDrainMul = 1.0**: 0.55는 K 5L 지배(42%), 1.0에서 5L 37%로 완화. 0.8~0.9가 이상적일 수 있음
- **leaderEmergencyDrainMul = 4.5**: 튜닝 사이클 2~5 내내 변경 불필요 — 선행 20~25% 안정

### 6.3 스킬시스템 연동 교훈
- **P_LowHP:35 임계가 너무 느슨**: 세크레트가 2L에서 수시 발동 → HP<25%로 상향 필요
- **Always 패시브 effectValue 유효 범위**: +4%는 약간 부족, +6%는 적정, +8%는 과강(테스트 안됨)
- **SkillDB 중앙화 이후 튜닝 속도 ↑**: CSV 1행만 수정하면 전체 effect 조정 가능
