# SPEC-028 step01 — Phase 1: 통화 시스템 + 배당 1.1 클램프

**날짜**: 2026-05-10
**상태**: 작성 (구현 대기)
**선행**: 없음 (최초 Phase)
**후속**: spec028_step02 / spec028_step03
**마스터**: spec028 (개요)

## 배경

도파민 젤리/스톤 룰을 게임에 도입하기 위한 백엔드 골격. 이 Phase는 UI·UGS 무관하게 **오프라인 단독으로 통화 흐름과 배당 보정이 100% 동작**해야 한다. 모든 후속 Phase의 기반.

## 요구사항

- **R1.1** `CurrencyItem` ScriptableObject 타입 정의 + DopamineJelly/Stone 인스턴스 2개 생성.
- **R1.2** `WalletManager` 싱글턴 — 게임 단위 젤리/스톤 보유, 베팅 차감, 보상 가산, 게임 시작 시 리셋.
- **R1.3** `BettingCalculator.CalculateReward(BetInfo, List<int> rankings, int betAmount)` 신설 — `Mathf.CeilToInt(amount × odds)` + 스톤 = 베팅액.
- **R1.4** `OddsCalculator.CalcComboOdds`의 5개 타입(Place·Quinella·Exacta·Trio·Wide)에 `Mathf.Max(1.1f, odds)` 안전망 추가.
- **R1.5** `GameSettings.asset`의 `oddsMin_*` 5개 값을 모두 1.1로 정규화 (YAML 직접 편집).
- **R1.6** `GameManager.GameState.GameOver` 신규 enum + `CalcScore()`에서 `WalletManager.Jelly == 0` 체크 분기.
- **R1.7** `enableRoundResume = false` 강제 + 도중 종료 = GAME OVER 처리 (관련 PlayerPrefs 키 사용 중단).
- **R1.8** `BetInfo`에 `int betAmount` 필드 추가 (직렬화 X, 런타임 전용).
- **R1.9** 기존 `payout*` 6개 필드는 그대로 유지 (OddsCalculator base 값으로 활용).
- **R1.10** Unity MCP 단위 테스트 5건 PASS — 50×1.1=55 / 33×1.1=37(올림) / 0.9 클램프 / Jelly 0 → GameOver / Wallet 리셋.

## 설계

### 클래스 다이어그램

```
[CurrencyItem : ScriptableObject]
  + string id
  + string nameKey
  + string descKey
  + Sprite icon
  + Color defaultColor

[WalletManager : MonoBehaviour (싱글턴)]
  - int _jelly
  - int _stone
  - CurrencyItem _jellyItem
  - CurrencyItem _stoneItem
  + int Jelly { get }
  + int Stone { get }
  + event Action OnChanged
  + bool TryBet(int amount)        // 잔액 < amount 면 false
  + void Reward(int jelly, int stone)
  + void ResetForNewGame()         // jelly = 100, stone = 0
  + void ResetStoneOnly()          // 랭킹 제출 후 호출 (Phase 4에서 활용)

[BetReward : struct]
  + int jelly       // 적중 시 받는 젤리
  + int stone       // 적중 시 받는 스톤
  + bool hit        // 적중 여부

[BettingCalculator : static (확장)]
  + BetReward CalculateReward(BetInfo bet, List<int> rankings, int betAmount)
    // 1. CalcComboOdds(bet, rankings, popularity, gameSettings) 호출
    // 2. Mathf.Max(1.1f, odds) 적용 (Phase 1 안전망)
    // 3. hit = HitCheck(bet, rankings)
    // 4. hit ? new BetReward { jelly = CeilToInt(amount * odds), stone = amount, hit = true }
    //       : new BetReward { jelly = 0, stone = 0, hit = false }

[OddsCalculator : static (수정)]
  + float CalcComboOdds(BetInfo bet, List<int> popularity, GameSettings settings)
    // 기존 로직 + 결과에 Mathf.Max(1.1f, ...) 클램프 추가

[GameManager (수정)]
  + enum GameState { Init, Betting, Racing, Finish, GameOver }   // GameOver 추가
  + void CalcScore()
    // 1. BettingCalculator.CalculateReward로 BetReward 계산
    // 2. WalletManager.TryBet(betAmount) 이미 호출됨 (BetAmountModal에서)
    //    여기서는 hit 시 WalletManager.Reward(jelly, stone)
    // 3. WalletManager.Jelly == 0 체크 → ChangeState(GameOver)
```

### 데이터 흐름

```
[유저 베팅 확정]
   ↓ (Phase 2 BetAmountModal에서 호출 — Phase 1은 호출 인터페이스만 준비)
WalletManager.TryBet(betAmount) → jelly -= betAmount
   ↓
[레이스 진행]
   ↓
[CalcScore]
   ↓
BettingCalculator.CalculateReward(bet, rankings, betAmount) → BetReward
   ↓ (hit이면)
WalletManager.Reward(reward.jelly, reward.stone)
   ↓
WalletManager.Jelly == 0 ? GameState.GameOver : 다음 라운드
```

### Unity MCP 실행 예시 (TC-A2-01)

```csharp
// MCP execute_csharp_script_in_unity_editor
var bet = new BetInfo(BetType.Win) { betAmount = 33 };
bet.selections.Add(0); // 1번 캐릭터
var rankings = new List<int> { 0, 1, 2 }; // 1번이 1등
var settings = GameSettings.Instance;
var reward = BettingCalculator.CalculateReward(bet, rankings, 33);
// 자연 배당이 1.1이라면 33 * 1.1 = 36.3 → CeilToInt = 37
Debug.Log($"[TC-A2-01] reward.jelly={reward.jelly} (expected 37)");
Debug.Log($"[TC-A2-01] reward.stone={reward.stone} (expected 33)");
Debug.Log($"[TC-A2-01] reward.hit={reward.hit} (expected true)");
```
