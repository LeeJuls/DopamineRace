### M2 구현 방식: 구간제 속도 (원래 FSM에서 변경)

원래 SPEC의 FSM(Positioning → Cruising → Spurt) 대신, **OverallProgress 기반 구간제 속도** 방식으로 구현됨.

```
[타입별 부스트 구간]  — 각 타입 고유의 전략 구간, Vmax × burstSpeedRatio(2.0) 전력질주
  도주(Runner)    : 6 ~ 86%   (테스트: 사실상 풀버스트)
  선행(Leader)    : 26 ~ 45%  (초중반)
  선입(Chaser)    : 46 ~ 65%  (중반)
  추입(Reckoner)  : 66 ~ 85%  (후반)

[기본 달리기]         — 부스트/최종스퍼트 외 구간, Vmax × normalSpeedRatio(1.0)

[최종 스퍼트]         — 86~100%, 전 타입 Vmax × spurtVmaxBonus(1.5) + Accel × spurtAccelBonus(2.0)
```

### 현재 파라미터 실측값 (GameSettingsV4.asset)

| 파라미터 | 값 | 설명 |
|---------|-----|------|
| `v4_staminaBase` | 80 | Stamina 0일 때 기본 HP |
| `v4_staminaPerStat` | 3 | Stamina 1당 추가 HP |
| `v4_drainPerLap` | 22.4 | 바퀴당 절대 HP 소모 (기본 달리기 기준) |
| `v4_burstDrainMul` | **1.8** | 부스트 구간 HP 소모 배율 |
| `v4_spurtDrainMul` | **2.5** | 최종 스퍼트 HP 소모 배율 |
| `v4_slipstreamDrainMul` | 0.70 | 슬립스트림 드레인 감소 배율 |
| `v4_exhaustSpeedFloor` | 0.80 | HP 0% 시 Vmax의 80% |
| `v4_normalSpeedRatio` | 1.0 | 기본 달리기 속도 (Vmax 대비) |
| `v4_burstSpeedRatio` | 2.0 | 부스트 속도 배율 (Vmax 대비) |
| `v4_spurtVmaxBonus` | 1.5 | 스퍼트 Vmax 보너스 배율 |
| `v4_spurtAccelBonus` | 2.0 | 스퍼트 Accel 보너스 배율 |
| `v4_runnerBurstEnd` | 0.86 | Runner 버스트 종료 (테스트값 — 원래 0.25) |

### 드레인 공식 (현재 구현)

```
progressDelta = currentProgress - lastProgress   // 이번 프레임 진행량

drain = v4_drainPerLap × totalLaps × progressDelta
      × (1.8 if Burst / 2.5 if Spurt / 1.0 if Normal)
      × (0.70 if Slipstream)
      × (1.80 if Panicking)

결과: Stamina 20 기준 (MaxHP 140)
  2바퀴 → 약 68% HP 잔여
  3바퀴 → 약 52% HP 잔여
  5바퀴 → 약 20% HP 잔여  ← 캘리브레이션 기준
```

## 1. V4 스탯 체계

### 1.1 스탯 일람 (1~20, 합계 90)

| 스탯 | 역할 | 핵심 공식 |
|------|------|----------|
| **Speed** | 최고속도 상한 | `Vmax = 1.0 + Speed × 0.02` |
| **Accel** | 가속 폭발력 | `Lerp(cur, target, Accel × 0.05 × dt)` |
| **Stamina** | 자원(HP) 총량 | `MaxHP = 80 + Stamina × 3` |
| **Power** | 충돌 돌파력 | `V_drop = 0.35 × (1 - myPower/(myPower+opponentPower))` |
| **Intelligence** | AI 판단력 | `P_smart = min(0.92, Int/(Int+10))` |
| **Luck** | 크리/회피 | `critChance = Luck × 0.005` (3초 주기) |

### 1.2 스탯별 계산 상세

#### Speed → Vmax
```
기본 Vmax = 1.0 + Speed × v4_speedStatFactor(0.02)
스퍼트 Vmax = 기본 Vmax × v4_spurtVmaxBonus(1.20)

| Speed | 기본 Vmax | 스퍼트 Vmax |
|-------|----------|-----------|
| 20    | 1.40     | 1.68      |
| 15    | 1.30     | 1.56      |
| 10    | 1.20     | 1.44      |
```

#### Accel → 가속 Lerp 가중치
```
가속 = Accel × v4_accelStatFactor(0.05)
스퍼트 가속 = 가속 × v4_spurtAccelBonus(2.0)

currentSpeed = Lerp(currentSpeed, targetSpeed, 가속 × dt)

| Accel | 기본 가속 | 스퍼트 가속 |
|-------|----------|-----------|
| 20    | 1.00     | 2.00      |
| 15    | 0.75     | 1.50      |
| 10    | 0.50     | 1.00      |
```

#### Stamina → HP
```
MaxHP = v4_staminaBase(80) + Stamina × v4_staminaPerStat(3)
드레인 = currentSpeed × v4_drainBaseRate(0.08) × 슬립스트림보정 × 오버페이스보정

| Stamina | MaxHP |
|---------|-------|
| 20      | 140   |
| 15      | 125   |
| 10      | 110   |

스태미나 0 도달 시:
  - Vmax → Vmax × v4_exhaustSpeedFloor(0.50)
  - 추가 감속: v4_exhaustDecel(2.0) 초당
```

#### Power → 충돌 페널티 감소 (상대 비교 방식)
```
// 충돌 발생 시 양쪽 모두 피해 (단, Power 높은 쪽이 덜 받음)
V_drop_A = v4_collisionBasePenalty(0.35) × (1 - PowerA / (PowerA + PowerB))
V_drop_B = v4_collisionBasePenalty(0.35) × (1 - PowerB / (PowerA + PowerB))

// 이유: 절대값 방식(1 - Power/20)은 Power 20이면 피해 0% → 사실상 무적
//      상대 비교 방식은 Power 차이가 클수록 격차가 생기되 둘 다 피해를 받음

예시 (basePenalty = 0.35):
| 상황 (A vs B) | A의 V_drop | B의 V_drop |
|--------------|-----------|-----------|
| 20 vs 20 (동등) | 0.175 | 0.175 |
| 20 vs 10    | 0.117     | 0.233     |
| 20 vs 5     | 0.070     | 0.280     |
| 10 vs 5     | 0.117     | 0.233     |
```

> **설계 이유**: 절대값 방식은 Power 20 캐릭터가 피해 0% → 충돌 전략이 없어짐.
> 상대 비교 방식은 Power 강자가 유리하되 완전 무적은 없어 전략 다양성 유지.

#### Intelligence → 3가지 판단 수치

**① 사전 회피 확률 (Soft-Cap)**
```
P_smart = min(v4_smartMaxLimit(0.92), Int / (Int + v4_smartK(10)))

| Int | P_smart | 의미 |
|-----|---------|------|
| 20  | 0.667   | 67% 확률로 마군 회피 성공 |
| 15  | 0.600   | 60% |
| 10  | 0.500   | 50% |
| 5   | 0.333   | 33% |
```

**② 오버페이스(흥분) 발동 확률**
```
P_panic = v4_panicBase(0.30) × (1 - Int / v4_intelligenceStatMax(20))

| Int | P_panic | 의미 |
|-----|---------|------|
| 20  | 0.00    | 절대 흥분 안 함 |
| 15  | 0.075   | 7.5% 확률 |
| 10  | 0.150   | 15% 확률 |
| 5   | 0.225   | 22.5% 확률 → 스태미나 급소모 |

오버페이스 발동 시:
  - 드레인 × v4_panicDrainMul(1.80) (80% 추가 소모)
  - 지속: v4_panicDuration(2.0)초
```

**③ 판단 틱 주기**
```
T_tick = v4_thinkTickBase(1.0) - (Int / v4_intelligenceStatMax(20)) × v4_thinkTickBonus(0.80)

| Int | T_tick | 의미 |
|-----|--------|------|
| 20  | 0.20초 | 0.2초마다 전방 탐색 |
| 15  | 0.40초 | |
| 10  | 0.60초 | |
| 5   | 0.80초 | 거의 1초에 한 번만 탐색 |
```

#### Luck → 크리/회피 (V1과 동일 구조)
```
critChance = Luck × v4_luckCritChance(0.005) (3초 주기)
critBoost = v4_luckCritBoost(1.30), v4_luckCritDuration(1.5)초
dodgeChance = Luck × v4_luckDodgeChance(0.02)
```

## 2. 레이스 페이즈

### 2.1 페이즈 구분 (OverallProgress 0~1 기준)

```
[Phase 1: Positioning]  0 ~ positioningEndRatio(0.15)
    ↓
[Phase 2: Cruising]     0.15 ~ 마지막 랩 spurtTrigger 전
    ↓
[Phase 3: Spurt]        마지막 랩의 남은 X% 이하
```

### 2.2 Phase 1: Positioning (자리잡기)

- **기간**: 레이스 시작 ~ OverallProgress 15%
- **목표**: 각 타입이 가상 페이스메이커 대비 목표 순위 범위에 진입
- **로직**:
  - 현재 순위가 목표 범위보다 앞 → 감속 (타겟 속도 낮춤)
  - 현재 순위가 목표 범위보다 뒤 → 가속 (스태미나 소모 증가)
  - **도주(Runner)**: 예외 — 무조건 선두 추구, 스태미나를 당겨 써서 1위 선점
- **지능 영향**: 오버페이스 판정 시작 (지능 낮으면 자리잡기 중 스태미나 낭비)
- **스태미나 소모 상한**: 별도 캡 없음 — `v4_drainBaseRate` 튜닝으로 Phase 3 진입 시 충분한 잔량 보장 (밸런싱으로 해결)

### 2.3 Phase 2: Cruising (페이스 유지)

- **기간**: 포지셔닝 완료 ~ 스퍼트 트리거 전
- **목표**: 스태미나 보존하며 목표 순위 유지
- **타입별 로직**:
  - 가상 페이스메이커 기준으로 목표 거리 유지
  - 슬립스트림 탐색 (지능 기반 — 앞 캐릭터 Y축 추종)
  - **도주(Runner) — 체력 안배 모드** ← 2026-03-16 확정
    - 1~3위 유지 시: Vmax의 80~90%로 제한 → 스태미나 절약 (Phase 3 대비)
    - 4위 이하로 밀릴 위기 (추격자가 `runnerDefendRange` 이내 접근) → **도주 본능 발동**: 목표 속도를 100% Vmax로 순간 가속
    - Phase 1에서 선두 스퍼트했으므로 Phase 2는 선두 방어 모드
    - **도주 본능 해제 조건**: 추격자가 `runnerDefendRange × 1.5f` 이상 벌어지면 자동 해제 → 체력 안배 모드 복귀 (해제 없으면 HP 조기 소진)
  - **추입(Reckoner)**: Vmax의 70~80% 속도로 의도적 제한 → 극단적 스태미나 비축
  - **선행(Leader)**: 도주마의 뒤를 추종, 슬립스트림 활용
- **지능 영향**:
  - 판단 틱마다 슬립스트림 명당자리 탐색
  - 오버페이스 발동 시 불필요한 급가속 → 스태미나 낭비

> **도주 Phase 2 설계 이유**: 처음부터 Vmax 전력이면 HP 소진 → 후반 역전 드라마 없음.
> 체력 안배로 Phase 3 스퍼트 폭발력 확보 + 도주 본능으로 선두 방어 긴장감 유지.

### 2.4 Phase 3: Spurt (전력질주)

- **트리거 조건**:
  - **도주**: Phase 1에서 이미 스퍼트 진입. Phase 2 체력 안배 후, Phase 3 진입 시 **남은 스태미나를 모두 소진하는** 최종 스퍼트
  - 선행: 마지막 랩, 남은 거리 20%
  - 선입: 마지막 랩, 남은 거리 30%
  - 추입: 마지막 랩, 남은 거리 40%
- **스퍼트 효과**:
  - Vmax × `spurtVmaxBonus`(1.20) — 속도 상한 해제
  - Accel × `spurtAccelBonus`(2.0) — 폭발적 가속
  - 드레인도 함께 증가 (높은 속도 → 높은 소모)
- **동적 스퍼트 (지능 보정)**:
  - 지능 높은 캐릭터: 남은 스태미나 / 예상 소모율 계산 → 최적 타이밍에 스퍼트
  - 지능 낮은 캐릭터: 고정 트리거 비율에서 발동 (타이밍 비효율)
- **마군 돌파**:
  - 선입/추입이 스퍼트 진입 후 전방 캐릭터를 만나면:
    - 지능 P_smart 확률 → 성공: 부드럽게 Y축 차선 변경 (속도 유지)
    - 실패: 충돌 → Power 기반 페널티 계산
  - 추입은 외곽(아웃코스)으로 크게 돌아 마군 우회 가능 (차선 2칸 이동)

## 3. 가상 페이스메이커

### 3.1 개념
- 보이지 않는 투명 오브젝트가 트랙을 일정 속도로 달림
- 캐릭터들은 **서로가 아닌 페이스메이커 대비** 상대 위치를 유지
- 데드락 방지: 1위가 늦어져도 페이스메이커는 계속 달림 → 뒤 캐릭터도 밀려남 없음

### 3.2 구현
```csharp
// 페이스메이커 속도 = globalSpeedMultiplier × pacemakerSpeedRatio
float pacemakerProgress += pacemakerSpeed × dt;

// 각 캐릭터의 목표 OverallProgress
float targetProgress = pacemakerProgress - typeOffset;
// 도주: offset = -0.02 (페이스메이커보다 약간 앞)
// 선행: offset = +0.03 (약간 뒤)
// 선입: offset = +0.08 (중위권)
// 추입: offset = +0.15 (후미)
```

### 3.3 거리별 보정
- **단거리(2바퀴 이하)**: 페이스메이커 빠르게 → 포지셔닝 짧음 → 도주 유리
- **장거리(5바퀴 이상)**: 페이스메이커 느리게 → 크루징 길어짐 → 추입 유리

## 4. Y축 차선 시스템

### 4.1 차선 구조
```
Y축 차선 (9명 기준, 5차선):
  차선 1 (인코스, Y 최상단)  ← 이동 거리 최단
  차선 2
  차선 3 (중앙)
  차선 4
  차선 5 (아웃코스, Y 최하단) ← 이동 거리 최장, 마군 회피용
```

### 4.2 차선과 게임 로직
- **슬립스트림**: 같은 차선 + 앞 캐릭터와 일정 거리 내 → 드레인 × 0.70
- **블로킹**: 같은 차선 + 앞 캐릭터가 느릴 때 → 속도 강제 하향 (추월 불가)
- **회피**: 지능 판단 틱 성공 → 빈 차선으로 Y축 이동 (속도 유지)
- **충돌**: 차선 이동 중 다른 캐릭터와 겹침 → Power 판정

### 4.3 차선 변경 규칙
- 차선 변경 중에는 약간의 속도 감소 (횡이동 비용)
- 한 번에 1차선만 이동 가능 (추입 아웃코스 우회 시 2차선 가능)
- 차선 변경 쿨다운: 0.5초

### 4.4 인코스 OverallProgress 보정 — V4 제외 (추후 개발)
> ⚠️ **V4에서는 인코스 진행도 보정을 구현하지 않는다.**

- **이유**: 현재 `OverallProgress`는 단순 선형 진행도. Y축(차선)별로 실제 이동 거리를 다르게 계산하려면 트랙 구조 전면 수정 필요 → MVP 복잡도 초과
- **영향**: 인코스가 아웃코스보다 실제로 빠른 이점 없음 → 차선은 순수 슬립스트림/블로킹/회피 전략용으로만 사용
- **추후 개발**: 별도 SPEC에서 `LaneProgressMultiplier` 도입으로 처리 예정

## 5. 슬립스트림 (기존 확장)

### 5.1 발동 조건
```
같은 Y축 차선 AND 앞 캐릭터와의 진행도 차이 ≤ slipstreamRange
```

### 5.2 효과
```
드레인 = baseSpeed × drainBaseRate × slipstreamDrainMul(0.70)
→ 30% 스태미나 절약
```

### 5.3 타입별 의미
- **도주**: 선두라 슬립스트림 불가 → 스태미나 소모 최대 (풍압 100%)
- **선행**: 도주 뒤에 붙으면 슬립스트림 → 효율적 크루징
- **선입/추입**: 마군 속에서 슬립스트림 → 스태미나 극적 절약
- **지능 높을수록**: 슬립스트림 위치를 능동적으로 찾아감

## 6. 충돌 시스템 (기존 공식 교체)

### 6.1 발동 조건 (V1과 동일)
- 같은 차선 + 거리 내 + 쿨다운 경과 + 확률 판정

### 6.2 V4 공식 (상대 비교 방식 — 2026-03-16 확정)
```csharp
// CollisionSystem.cs ResolveCollision() 내부 — V4 분기
if (gs.useV4RaceSystem && gs.v4Settings != null)
{
    float pWinner = winner.V4Power;                    // RacerController에 추가할 프로퍼티
    float pLoser  = loser.V4Power;
    float total   = Mathf.Max(1f, pWinner + pLoser);   // 분모 0 방어

    float winnerPenalty = gs.v4Settings.v4_collisionBasePenalty * (1f - pWinner / total);
    float loserPenalty  = gs.v4Settings.v4_collisionBasePenalty * (1f - pLoser  / total);

    winner.ApplyCollisionPenalty(winnerPenalty, gs.winnerPenaltyDuration);
    loser.ApplyCollisionPenalty(loserPenalty,   gs.loserPenaltyDuration);
}
else
{
    // 기존 V1/V3 방식 — 그대로 유지
    float winnerPenalty = gs.collisionBasePenalty * 0.5f * trackPenaltyMul;
    winner.ApplyCollisionPenalty(winnerPenalty, gs.winnerPenaltyDuration);
    float loserPenalty = gs.collisionBasePenalty * trackPenaltyMul;
    loser.ApplyCollisionPenalty(loserPenalty, gs.loserPenaltyDuration * trackLoserDurMul);
}

// 예: v4_collisionBasePenalty=0.35
// Power 20 vs 20 → 양쪽 모두 0.175 (동등 피해)
// Power 20 vs 10 → 강자 0.117, 약자 0.233
// Power 20 vs 5  → 강자 0.070, 약자 0.280
```

> **구현 방식**: 별도 V4CollisionSystem.cs 파일 불필요.
> `ResolveCollision()` 내부 페널티 계산 블록에 `if (gs.useV4RaceSystem)` 분기만 추가.
> 스킬 충돌(aArmed/bArmed), Luck 회피, 슬링샷, VFX는 V4에서도 그대로 유지.
> `winner.V4Power` 프로퍼티는 M3a-3 구현 시 `RacerController_V4.cs`에 추가.

### 6.3 Luck 회피 (유지)
```
dodgeChance = Luck × 0.02
→ Luck 20: 40% 확률로 충돌 회피
```

| **차선 5 경계 초과** | 추입이 아웃코스 2칸 이동 시 현재 차선 4이면 차선 6 접근 → 배열 OOB | `Mathf.Clamp(newLane, 1, 5)` 가드 확인 |
| **Accel×dt Lerp 불안정** | `Lerp(cur, target, Accel×0.05×dt)` — 스파이크 프레임 dt가 크면 가중치 > 1 → 속도 target 역전 | `Mathf.Clamp01(accel × dt)` 가드 없는지 확인 |
| **다중 라운드 V4 초기화** | 2라운드 시작 시 `v4CurrentStamina`, `v4Phase`, `v4IsSpurting` 등 미리셋 → 1라운드 상태 잔류 | 2라운드 진입 시 모든 V4 상태 변수 초기값 확인 |
