# SPEC-028 — 도파민 젤리/스톤 + UGS 글로벌 랭킹 (마스터)

**날짜**: 2026-05-10
**상태**: 작성 (구현 대기)
**관련 히스토리**: (구현 후 작성)
**분할**: 본 마스터 + SPEC-028A~E (Phase 1~5)

## 배경

현재 베팅이 "단발 베팅 → 결과" 구조로 짜릿함이 약함 — 실제 경마처럼 돈을 거는 긴장감 부재라는 주변 피드백. 한정 시드(젤리 100)로 N라운드를 운용하는 로그라이크형 의사결정 게임으로 변환하고, 누적 스톤으로 글로벌 랭킹을 제공해 "한 게임 = 하나의 도전" 사이클의 도파민 루프를 구축한다.

## 요구사항

- **R1** 한 게임 = 도파민 젤리 100개 + 도파민 스톤 0개로 시작 (모든 유저 동일 출발선).
- **R2** 적중 시 받는 젤리 = `Mathf.CeilToInt(베팅액 × 배당)`, 적중 시 받는 스톤 = 베팅액 × 1.
- **R3** 빗나감 시 베팅 젤리 전액 손실, 스톤 변화 없음.
- **R4** 라운드 진행 중 젤리가 0이 되면 GAME OVER (저장/랭킹 X, 재시작 강제).
- **R5** 마지막 라운드 완주 시에만 누적 스톤을 글로벌 랭킹에 제출하고 스톤 리셋.
- **R6** 모든 베팅 옵션의 배당 최소값은 1.1배 (자연 배당 < 1.1 시 1.1로 보정 표시 — 옵션 차단 X).
- **R7** 닉네임은 랭킹 등록 시점에만 1회 입력 (오락실 점수 등록 스타일), 변경 불가.
- **R8** 글로벌 랭킹은 Top 10,000 컷오프 (오픈 직후 100명으로 시작, 추후 확장).
- **R9** 보유 통화(젤리·스톤)는 베팅 화면 상단 헤더에서 **기존 `총점` 영역을 교체**하여 항상 표시.
- **R10** 베팅액 입력은 START 클릭 시 별도 모달 팝업 (기존 UI 유지).
- **R11** 스톤 사용처(향후 캐릭터 튜닝)는 이번 명세에서 미노출.
- **R12** 모든 신규 텍스트는 StringTable에 UID 키로 등록 후 `Loc.Get()` 사용 (한국어 + 영어 우선, 나머지 5언어는 영어 fallback).
- **R13** 도파민 젤리·스톤은 각각 `CurrencyItem` ScriptableObject로 분리 관리 (이름·아이콘 디자이너 교체 용이).
- **R14** 글로벌 랭킹 화면 진입점은 TitleScene **START 버튼 위쪽**에 별도 버튼으로 배치 (대기 화면에서 즉시 접근).
- **R15** 베팅 타입별 보상 라벨(예: "삼복승 획득 포인트")은 "**삼복승 획득 도파민 스톤**" 또는 아이콘(💎) 표기로 교체.

## 설계

### 통화 구조

| 통화 | 휘발성 | 용도 (현재) | 용도 (미래, 미공개) |
|------|--------|-------------|---------------------|
| 🟦 **도파민 젤리** | 게임마다 100 리셋 | 베팅 | 동일 |
| 💎 **도파민 스톤** | 게임 종료 시 리셋 (랭킹 제출 후) | 글로벌 랭킹 점수 | 라운드 사이 캐릭터 튜닝 |

### Currency Item 구조 (ScriptableObject)

```
Assets/Resources/Items/
├─ DopamineJelly.asset    (CurrencyItem)
└─ DopamineStone.asset    (CurrencyItem)

CurrencyItem {
  string id;             // "jelly" / "stone"
  string nameKey;        // StringTable UID
  string descKey;        // StringTable UID
  Sprite icon;           // 디자이너 PNG
  Color defaultColor;    // UI 강조 색상
}
```

### 시스템 계층

```
[유저 입력]
   ↓
[BetAmountModal]──┐
   ↓             │
[WalletManager] ──→ [BettingCalculator.CalculateReward]
   ↓                  ↓ (Mathf.CeilToInt)
[GameManager.CalcScore]
   ↓
[GameState 분기]
   ├─→ Continue (다음 라운드)
   ├─→ GameOver (jelly==0)
   └─→ Finish (마지막 라운드 완주)
            ↓
       [LeaderboardService]
            ↓
       (cutoff 비교)
            ├─→ 진입 → NicknameInputModal → SubmitStone → 스톤 리셋
            └─→ 미진입 → 안내 → 스톤 리셋
```

### 게임 흐름

```
[메인메뉴]
   ↓ (게임 시작)
[게임 시작 화면 — 총 N 라운드 표시, 젤리 100/스톤 0]
   ↓
[1라운드 베팅] → 캐릭터 선택 → START 클릭
   ↓
[BetAmountModal — 베팅액 결정]
   ↓ (배팅 확정)
[레이스 진행]
   ↓
[결과 — 젤리/스톤 갱신]
   ↓
   ├─ 젤리 == 0 ─→ [GameOverScreen — 다시 시작]
   ↓
[2~N-1 라운드 반복]
   ↓
[마지막 라운드 결과]
   ↓
[FinishScreen — 누적 스톤 + 랭킹 제출 분기]
   ├─ Top 10K 진입 → [NicknameInputModal] → 등록 → [GlobalRanking 표시]
   └─ 미진입 → "랭킹 미진입" → 다시 시작
```

## StringTable UID 정책 (강제)

> **모든 텍스트는 예외 없이 UID 키로 등록 → `Loc.Get()` 호출**

UID 키 네이밍:
```
str.{도메인}.{용도}[.{서브}]

예:
  str.currency.jelly.name
  str.currency.jelly.desc
  str.currency.stone.name
  str.bet.modal.title
  str.bet.modal.amount_label
  str.bet.preview.win
  str.bet.preview.lose
  str.gameover.title
  str.gameover.btn.restart
  str.rank.global.title
  str.rank.global.cutoff_notice
  str.nickname.modal.title
  str.nickname.modal.placeholder
```

매 Phase 종료 시 `DopamineRace > Validate StringTable Keys` 실행 → 누락/미사용 0건 필수.
