## 배경

SampleScene Round 1 Betting 상태 진입 시, 캐릭터들이 배팅 화면 동안 트랙 위를 자유롭게 돌아다니는 워밍업 연출. Start 버튼을 누르면 출발선으로 1.2초 Lerp 정렬 후 기존 3-2-1 카운트다운 진입. Round 2 이상은 기존 동작 유지.

---

## 요구사항

### R1. Round 1 자동 배회 시작
- Round 1 Betting 진입 시 캐릭터 자동으로 X:0~9, Y:-3.5~3.5 범위 배회

### R2. 3가지 배회 동작
- Moving(60%): 랜덤 목표 지점으로 Run 트리거 이동
- Pausing(25%): 0.5~2.0초 Idle 정지
- Swaying(15%): 1.0~2.5초 ±0.2 사인파 X 진동

### R3. Start 버튼 → LineUp(Round 1만)
- Round 1: 배회 중지 → 1.2초 SmoothStep Lerp로 출발선 정렬 → StartRace()
- Round 2+: 기존 동작 그대로 (즉시 StartRace())

### R4. 더블클릭 방지
- LineUp 중 startButton.interactable=false, 완료 후 true 복구

### R5. 하위 호환성
- ResetRacer() 호출 시 배회 자동 중단
- Round 2+ Betting 진입 시 배회 없음 (출발선 대기)

---

## 설계

### 흐름 변경

```
현재: Betting → [Start 클릭] → Countdown(3s) → Racing
변경: Betting → 캐릭터 배회 중 → [Start 클릭] → LineUp(1.2s) → Countdown(3s) → Racing
Round 2+: 변경 없음
```

### 배회 범위
- X: 0~9, Y: -3.5~3.5 (카메라 가시 영역 X:-7~12, Y:-5.5~5.5 내부)

### 아키텍처 결정사항
| 결정 | 이유 |
|------|------|
| Walk 트리거 미사용 → Run 대체 | Animator에 Walk 없음 (Sprint 0 확인) |
| partial class 파일 분리 | RacerController 주 파일 오염 방지 |
| `_isWandering` 가드 Update() 최상단 | `isRacing`과 동시 활성 차단 |
| `_wanderCoroutine` 핸들 방식 | StopAllCoroutines() 금지 원칙 준수 |
| GameManager.Start() 타이밍 보정 | `Start()`에서 기존 Betting 상태 체크 추가 |
