# SPEC-027 — 트랙별 날씨 효과 + 트랙 선택 드롭다운

**날짜**: 2026-04-26
**상태**: 완료 (Desert는 롤백)
**관련 히스토리**: his043

## 배경

5개 트랙(Normal/Rainy/Snow/Desert/Highland)이 외관만 다르고 분위기 차이가 부족. **Pixel Weather FX 에셋**으로 트랙별 고유 날씨 연출 추가 + Inspector에서 라운드별 트랙을 **enum 드롭다운**으로 선택 가능하게.

## 트랙별 효과 상세

### Rainy ✅
- **PS_RainStreak** (메인 비) — Pixel Weather FX 기본
- **PS_Splash** (신규, MCP로 직접 생성) — **정지형 플래시 (v4 최종)**
  - Box 18×10.5 화면 전체 랜덤 스폰
  - 수명 0.2~0.35s, 사이즈 곡선 0.4→1.0→0.6 (파문 확장)
  - 알파 0→1→1→0 페이드 (15%/60%/100%)
  - velocityOverLifetime disabled (떨어지지 않음)
- **SE_RainSplash** (원본 서브이미터) — `m_IsActive=0` 비활성

### Snow ✅
- **PS_Snow** (메인 눈) + **PS_Fog** (보너스 옅은 안개)
- **SE_Melt** 서브이미터 비활성 (회색박스 위험)

### Highland ✅
- **PS_Fog** (옅은 안개, 모락모락 펄스)
- 핵심 튜닝:
  - `startLifetime` 25→12s (사이클 빠르게)
  - `rateOverTime` 25→35/s (적절 밀도)
  - 동시 활성 ~420개 (원본 ~625에서 33% 감소 → 펄스감 살아남)
  - ShapeModule.scale (50, **0**, 3) → (22, **12**, 3) 화면 전체 커버

### Desert ❌ 롤백
- Wind01_Pixel + Fog02 시도 → 회색 점만 찍히고 움직임 없음
- **근본 원인**: VelocityModule.x/y가 minMaxState=3 (TwoCurves) 곡선이라 prefab override의 scalar 변경 무시됨
- **교훈**: 곡선 기반 모듈은 prefab override로 변경 불가 — Constant 필드만 안전

## 공통 오버라이드 패턴 (모든 트랙)

```yaml
PS:
- prewarm: 0
- cullingMode: 3 (AlwaysSimulate)
- SubModule.enabled: 0
- CollisionModule.enabled: 0
Renderer:
- m_CastShadows / m_ReceiveShadows / m_DynamicOccludee: 0
- m_ReflectionProbeUsage / m_LightProbeUsage: 0
- m_SortingOrder: -50 (BG -100 위, 캐릭터 0 아래)
```

## 트랙 선택 드롭다운

```csharp
public enum TrackChoice {
    Random = 0, Normal = 1, Rainy = 2, Snow = 3, Desert = 4, Highland = 5
}
```

`GameSettings.roundTracks` 필드가 string[] → TrackChoice[] 마이그레이션. **GameSettings.asset YAML 직접 편집 필수** (Unity 컴파일 후엔 string 값 전부 default(0)으로 초기화됨).

## 부수 변경

### 입상 순위 sortingOrder
- `RaceManager.SetRacerSortingOrder(racer, 50 - finishCount)`
- 1등 49 > 2등 48 > 3등 47 > ... 9등 41
- `RespawnRacers()`에서 라운드 재시작 시 0으로 리셋

### 배회 bounds 축소 (트랙 밖 캐릭터 방지)
- 이전: center(3.5, 0), size(12, 6)
- 현재: center(3.0, 0), size(10, 4.6) — Highland 같은 작은 트랙 기준

### 타이틀 스폰 가속
- `FIRST_SPAWN_DELAY` 1.0 → 0.15s
- `SPAWN_INTERVAL_MIN/MAX` 0.8/1.5 → 0.4/0.75s

## 학습된 제약

| 제약 | 회피 방법 |
|------|----------|
| Prefab override는 nested struct (Color.a, MinMaxCurve curve) 변경 불가 | Constant scalar 필드만 override |
| 곡선 기반 모듈 (VelocityModule 등) override 불가 | 곡선 필드 회피, 기본값 사용 |
| Unity는 string[] → enum[] 변경 시 asset 값 초기화 | YAML 직접 편집 (Editor 종료 상태) |
| Pixel Weather FX 프리팹의 sub-emitter는 충돌 기반 | `m_IsActive=0` 비활성 (회색박스 사고 방지) |
