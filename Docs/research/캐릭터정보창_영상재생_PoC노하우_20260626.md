## 2. 핵심 노하우

### 2-1. SampleScene UI는 전부 런타임 동적 생성
- `SceneBootstrapper`가 Play 시점에 Canvas·`CharacterInfoPopup`·일러스트를 코드로 생성.
- **Edit 모드엔 해당 오브젝트가 없음** → MCP `search_by_name`/Hierarchy 0건, 프리팹 직접 수정 불가.
- 결론: **런타임 주입(또는 SceneBootstrapper/생성기 코드 수정)** 이 유일한 통합 경로.

### 2-2. CharacterInfoPopup 일러스트 구조
```
CharacterInfoPopup (BettingPanel 하위, 런타임 생성)
 └ Layout2_Left
    └ IllustrationMask        ← UnityEngine.UI.Mask (showMaskGraphic=false), 크롭 담당
       ├ Illustration         ← Image + AspectRatioFitter(HeightControlsWidth)
       └ (WinRateBg 등 오버레이가 형제로 올 수 있음 — 가리지 말 것)
```
- 정적 일러스트는 **Image** 컴포넌트. 영상은 **RawImage** 필요 → 직접 교체 대신 **오버레이 추가**가 코드 변경 최소.

### 2-3. 일러스트 실측 크기 & 정규화 앵커 (가장 중요)
- 게임 캔버스: **ScreenSpaceOverlay + CanvasScaler(레퍼런스 3840×2160, 4K)**.
- `IllustrationMask` 실측 = **711 × 731 px** (4K 레퍼런스 공간 기준, 비율 ≈ 0.97 거의 정사각형).
- ⚠️ **절대 픽셀로 맞추면 해상도 변경 시 어긋남** → 반드시 **정규화 앵커(화면 비율 0~1)** 사용.

| 정규화 앵커 | X | Y |
|------------|------|------|
| anchorMin | 0.3256 | 0.2679 |
| anchorMax | 0.5109 | 0.6061 |

- **측정 방법** (Play 모드 필수, 런타임에만 존재):
  ```
  maskRT.GetWorldCorners(mc); canvasRT.GetWorldCorners(cc);
  nMin = (mc[0] - cc[0]) / (cc[2] - cc[0]);   // bottom-left
  nMax = (mc[2] - cc[0]) / (cc[2] - cc[0]);   // top-right
  ```
- 적용: 오버레이 RectTransform에 `anchorMin/Max = nMin/nMax`, `offsetMin/Max = 0`.

### 2-4. Unity UI 영상 재생 체인
```
VideoPlayer(renderMode=RenderTexture, source=Url, isLooping)
   → RenderTexture(영상 w×h, prepareCompleted 콜백에서 생성)
   → RawImage.texture
```
- `url` = `Application.streamingAssetsPath` 하위 경로 (StandaloneWindows에서 `File.Exists` 동작).
- `Prepare()`는 **비동기** → `prepareCompleted` 콜백에서 RT 생성 후 `Play()`.
- 오디오 불필요 시 `audioOutputMode = None`.

### 2-5. 커버 크롭(프레임 꽉 채우기)
- `AspectRatioFitter.AspectMode.EnvelopeParent`, `aspectRatio = 영상 w/h`.
- Mask가 넘치는 부분을 크롭 → 정사각 프레임에 세로 영상도 자연스럽게 채워짐.

## 3. MCP / Play 작업 주의점

| 상황 | 주의 |
|------|------|
| ScreenSpaceOverlay 캡처 | `unity_graphics_game_capture`(카메라)엔 **안 잡힘** → `unity_screenshot_game`(실제 게임뷰) 사용 |
| Play 진입/종료 | 도메인 리로드로 `"fetch failed"` 일시 에러 — `editor_state`로 재확인하면 정상 |
| Play 모드 변경분 | 만든/수정한 오브젝트·RectTransform은 **Stop 시 롤백** → 영구 적용은 **Edit 모드**에서 |
| 측정 타이밍 | 일러스트는 런타임 생성 → **Play에서 측정**, 측정값(정규화 앵커)만 빼서 Edit/코드에 반영 |
| 팝업 표시 트리거 | `SceneBootstrapper.OnRacerClicked(uiIndex)`(private) → `charInfoPopup.Show(...)`. 검증 시 리플렉션 호출 가능 |

## 4. 재설계 시 권장 아키텍처 (미구현)

> 기존 코드 0 수정 + 언제든 제거 가능 원칙 유지 시:

1. **독립 컴포넌트**(영속 호스트 오브젝트)가 런타임에 `CharacterInfoPopup` 탐색 후 `IllustrationMask` 안에 `RawImage` 오버레이 주입.
   - 배치: `Illustration` **바로 위 sibling** (승률 등 후순위 오버레이는 안 가림).
2. **VideoPlayer는 영속 호스트에 배치** (팝업 비활성 영향 회피), RenderTexture를 오버레이에 연결.
3. **상태 구동**: 팝업 `IsShowing`/`CurrentCharId` 폴링
   - 표시 → 재생 / 숨김 → 일시정지 / 캐릭터 전환 → 영상 교체.
4. **prepare 지연 폴백**: 준비 중엔 오버레이 숨겨 **기존 정적 일러스트** 노출.
5. **영상 경로 규칙**: `StreamingAssets/char_videos/{charId}.mp4` → 없으면 `default`.

## 5. 정식 진행 전 결정 필요 사항

| 항목 | 검토 포인트 |
|------|-----------|
| 빌드 용량 | 캐릭터당 ~1.8MB × 32명 ≈ **58MB**. 압축/해상도/길이 정책 |
| prepare 지연 | 캐릭터 전환 시 첫 프레임 공백 → 정적 일러스트 폴백 UX 확정 |
| 코덱 | **H.264 권장** (Unity VideoPlayer 호환성) |
| 스타일 일관성 | 32명 AI 영상 간 톤·구도 일관성 확보 방안 |
| 영상 vs 정적 | 사용자 토글 옵션 제공 여부 (저사양/취향) |
| 제작 파이프라인 | Kling / OCMaker AI 등 — 입력(기존 일러스트) 활용 가능 여부 |
