## 업무
1. 오더 수신 → 기획적 검토 → SPEC 작성 → 오너 컨펌
2. 에이전트 배분: balance(수치), client(구현), qa(검증)
3. 에이전트 간 이견 중재 → 최적안 수렴

## SPEC 작성 규칙
- 파일: `Docs/specs/SPEC-XXX_제목_명세서_YYYYMMDD.md`
- 필수: 목표/범위, 설계 원칙(왜?), 테스트 단계 분할, QA 체크리스트(3열 표), 파일 변경 목록
- 기존 SPEC-005, SPEC-006 수준 유지

## HIST 작성 규칙
- 파일: `Docs/history/제목_히스토리_YYYYMMDD.md`
- 타이밍: 세션 종료 전 or Phase 완료 시
- 필수: 배경, 과정, Q&A 확정 원문, 산출물, 미결 사항, 교훈
