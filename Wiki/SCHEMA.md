## 1. 위키 구조 (3계층)

```
Raw Sources (불변)
  └─ D:/Unity_Project/DopamineRace/Docs/specs/   ← SPEC 명세서 원본
  └─ D:/Unity_Project/DopamineRace/Docs/history/ ← 히스토리 원본

Wiki Layer (LLM 관리) ← 이 볼트
  ├── SCHEMA.md          ← 이 파일 (도메인 규칙)
  ├── Index.md           ← 전체 페이지 허브

  └── 도파민 프로젝트/   ← 엔티티/개념 페이지

Schema (이 파일)
  └─ SCHEMA.md          ← LLM 유지보수 지침
```

## 2. 세션 시작 필수 루틴

**매 세션에서 첫 작업으로 반드시 다음 3개 파일 읽기:**

1. `SCHEMA.md` — 도메인 규칙 파악
2. `Index.md` — 존재하는 페이지 확인
3. `log.md` 최근 30개 항목 — 최근 활동 확인

→ 중복 페이지 생성 방지 + 누락 교차 참조 방지

## 3. 필수 페이지 프론트매터

```yaml
---
title: 페이지 제목
created: YYYY-MM-DD
updated: YYYY-MM-DD
type: concept | entity | comparison | query | summary
tags: [태그분류법의 태그들]
related: [[연관_페이지]]
sources: [Docs/specs/spec012_V4레이스시스템.md]
confidence: high | medium | low
contested: true           # 모순이 있을 때만
contradictions: [other-slug]  # contested: true 일 때만
---
```

**confidence 기준:**
- `high` — 공식 SPEC/CLAUDE.md 기반, 검증됨
- `medium` — 히스토리 기반, 대체로 정확하나 변경 가능
- `low` — 추정/불확실, 재검증 필요

## 5. 3가지 핵심 작업

### Ingest — 새 SPEC/히스토리 추가 시

1. 관련 위키 페이지 **1~3개** 업데이트 (또는 신규 생성)
2. [[Index]] 에 신규 페이지 링크 추가
3. [[도파민 프로젝트/SPEC_인덱스]] 에 행 추가
4. [[도파민 프로젝트/히스토리/개발_타임라인]] 에 마일스톤 추가
5. `log.md` 에 `[INGEST]` 항목 기록
6. 모든 신규 페이지: **최소 2개 교차 링크 `[[]]` 필수**

### Query — 질문에 답할 때

1. `Index.md` 읽고 관련 페이지 식별
2. 다음 기준 중 2개 이상이면 `queries/YYYYMMDD_질문요약.md` 로 저장:
   - 같은 질문이 반복될 것 같다
   - 여러 페이지 지식을 종합한 답변이다
   - 미래 결정에 참고가 된다
3. `queries/_index.md` 의 목록 테이블에 행 추가
4. `log.md` 에 `[QUERY]` 항목 기록

### Lint — 주기적 정리

점검 항목:
- 고아 페이지 (다른 페이지에서 링크 없는 것)
- 깨진 `[[링크]]`
- 교차 링크 2개 미만 페이지
- 프론트매터 누락 필드
- 90일 이상 미갱신 페이지
- `confidence: low` 페이지 재검증
- 모순(`contested: true`) 처리 여부
- `log.md` 500개 초과 여부

## 6. 페이지 생성/분할/아카이빙 기준

| 조건 | 처리 |
|------|------|
| 2개 이상 출처에서 언급 | ✅ 페이지 생성 |
| 특정 출처의 핵심 내용 | ✅ 페이지 생성 |
| 일회성 언급, 부수적 세부사항 | ❌ 생성 금지 |
| 200줄 초과 | 하위 페이지로 분할 |
| 내용이 완전히 대체됨 | `_archive/` 로 이동 |
| 10개 이상 페이지 대량 업데이트 | 사용자 확인 후 진행 |

---

## 7. 로그 관리

- `log.md` 항목 500개 초과 → `log-YYYY.md` 로 이름 변경 후 새 `log.md` 시작
- 로그 형식: `- **[ACTION]** 내용 (`파일경로`) ← 출처`
- ACTION 종류: `INGEST` · `QUERY` · `LINT` · `UPDATE` · `ARCHIVE`
