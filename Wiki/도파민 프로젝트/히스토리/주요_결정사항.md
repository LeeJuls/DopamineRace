## 배당 정밀도 — decimal 사용

**문제**: `Mathf.CeilToInt(33 × 1.1f)` — float 오차로 잘못된 결과 가능

## 치팅 방어 — "완전 차단" 포기, 서명+난독화 채택 (2026-06-24)

**문제**: 레이스가 100% 클라 RNG 시뮬 → 서버가 점수를 재계산해 검증 불가(구조적 한계). 점수=`WalletManager.Stone` 단일 메모리 값이 그대로 제출됨.
**결정**: 목표를 "완전 차단"이 아니라 **"캐주얼 치터 차단 + 리버서 최대 지연"**으로 설정. 방어 = HMAC **raw 바디 서명**(`X-Sig`) + IL2CPP + SecureInt + 저장 체크섬.
**이유**: 암호화(기밀성)는 위협이 아님 — 위협은 위조(무결성)라 서명이 본질. 키가 클라에 있는 한 리버서는 못 막지만, 스니핑+수정 같은 흔한 공격은 차단.
**HMAC 키 포인트**: JSON 문자열이 아닌 **전송 raw 바디 바이트**에 서명 → C#(`JsonUtility`)↔JS 직렬화 불일치 회피.
→ 상세: [[../시스템/보안_치팅방어]], SPEC-044

## 시크릿은 git 절대 금지 (2026-06-24)

**사건**: WRITE_TOKEN이 git 히스토리에 노출 발견 → 즉시 로테이션.
**결정**: 시크릿은 `LeaderboardConfig.asset`(skip-worktree) + `SECRETS.local.txt`(gitignore)에만. 노출 시 키 폐기가 유일한 대응(히스토리 되돌리기 불가).
**교훈**: skip-worktree라 **다른 PC는 pull로 토큰이 안 바뀜 → 수동 입력 필요**.
