# 보안 & 치팅 방어

> 리더보드 점수 위조 방어 + 시크릿 관리 + `.claude/` 설정 위생. 담당: [[에이전트_가이드|security 에이전트]].

---

## 핵심 한계 (먼저 이해할 것)

리더보드 점수 = `WalletManager.Stone`(메모리 정수)이 그대로 서버 제출됨. 그리고
**레이스는 100% 클라이언트 RNG 시뮬레이션** → 서버가 점수를 재계산해 검증하는 것은 **구조적으로 불가능**.

→ 목표는 **"완전 차단"이 아니라 "캐주얼 치터 차단 + 작정한 리버서 최대 지연"**. 브래깅용 리더보드에 서버 권위 시뮬레이션까지는 과방어.

## 위협 모델

| 공격 | 현재 | 방어 후 |
|------|------|---------|
| 트래픽 스니핑 후 점수 JSON 수정·재전송 | 매우 쉬움 | HMAC 서명 깨짐 → 차단 |
| 빌드 DLL 디컴파일로 토큰/키 추출 | 쉬움(Mono) | IL2CPP → 네이티브 리버싱 필요 |
| "그냥 150000 보내기" | 통과 | rounds 기반 상한 → 거부 |
| Cheat Engine 메모리 스캔 | 쉬움 | SecureInt 마스킹 → 값 탐색 실패 |
| PlayerPrefs·leaderboard.json 직접 편집 | 쉬움 | HMAC 체크섬 → 변조 감지 |

---

## 방어 4레이어 (설계 — 구현 DEFERRED)

1. **HMAC 본문 서명** — 클라가 보내는 **raw 바디 바이트 전체**에 `HMAC-SHA256` → 헤더 `X-Sig`. 서버는 `request.text()`로 받은 그대로 검증 후 `JSON.parse`(C#↔JS 직렬화 불일치 회피). `SIGN_KEY`는 WRITE_TOKEN과 **별도** 신규 시크릿.
2. **IL2CPP 전환** — 현 Mono(`Assembly-CSharp.dll` 즉시 디컴파일) → 네이티브로 키 추출 난이도 ↑.
3. **SecureInt 메모리 난독화** — `_jelly`/`_stone`을 `value^mask` 저장 구조체로. [[주요_클래스|WalletManager]] 프로퍼티 시그니처 불변.
4. **저장 무결성** — PlayerPrefs/`leaderboard.json`에 HMAC 체크섬 → 변조 감지.

서버 보강: `MAX_SCORE` 플랫(150k) → **rounds 기반 상한**(수치는 balance 산정).

> 진화 경로는 [[에이전트_가이드|server 에이전트]] 문서의 "토큰 → HMAC 본문서명 → Steam 티켓"과 정합.

## 시크릿 관리

- 시크릿은 코드·git **절대 금지** → `LeaderboardConfig.asset`(git skip-worktree) + `SECRETS.local.txt`(gitignore)에만.
- 노출 발견 시 **즉시 로테이션**: 신규 생성 → `wrangler secret put` → asset/SECRETS 갱신. git 히스토리는 되돌릴 수 없음 → 키 폐기가 유일한 대응.
- **다른 PC는 skip-worktree라 pull로 토큰이 안 바뀜 → 수동 입력 필요.**
- 2026-06-24: WRITE_TOKEN 유출 발견 → 로테이션 완료(구 토큰 403 무효). 상세: [[../히스토리/개발_타임라인]].

## `.claude/` 설정 위생 (2026-06-24 감사 — D등급)

`settings.local.json` allow 목록에 임의 실행 와일드카드 누적:

| 등급 | 항목 |
|------|------|
| CRITICAL | `powershell -Command:*` · `python:*` · `mcp__Windows-MCP__PowerShell` · `Desktop_Commander__start_process` → 무프롬프트 임의 코드 실행(인젝션 시 RCE) |
| HIGH | `reg delete *`(레지스트리 삭제) · `git push:*`(오너 "수동 push" 정책과 모순) |
| GOOD | 설정 파일 하드코딩 시크릿 0, 시크릿 분리 양호 |

권고: 임의실행 와일드카드 제거·구체 명령 재범위화.

## 6대 점검 체크리스트

1. 크리덴셜 노출 (Assets/·config·git history)
2. 리더보드 통신 (HMAC `X-Sig`·토큰·rounds 상한)
3. 저장 무결성 (PlayerPrefs/파일 체크섬)
4. 재화·배팅 (SecureInt·`Reward()` 상한·입력 clamp)
5. `.claude/` 위생 (allowlist·시크릿 분리·인젝션)
6. 빌드 하드닝 (IL2CPP·스트리핑·`#if UNITY_EDITOR` 가드)

> 감사 실행: `/security-audit` 스킬 또는 [[에이전트_가이드|security 에이전트]] 디스패치.
