# 사운드 시스템

> **상태**: ✅ BGM 듀얼 크로스페이드 + 상태별 전환 완료 / ✅ SFX 툴화 완료(SFXSettings, 12키)  
> **핵심**: `BGMManager`(BGM, DontDestroyOnLoad 싱글턴 + AudioSource A/B 크로스페이드) / `SFXManager`+`SFXSettings`+`SFXKeys`(SFX, 키 기반 데이터 주도)

---

## BGM 클립 목록

| 파일 (Resources/) | 재생 상황 |
|------------------|----------|
| `Audio/Title_Bgm` | 타이틀 씬 |
| `Audio/BGM` | 배팅 라운드 중 |
| `Audio/bgm_exciting` | Countdown 시작 → 레이싱 → 결과 화면까지 |
| `Audio/bgm_ending` | Finish / GameOver 화면 |

## BGM 상태 전환 흐름

```
TitleScene → Title_Bgm (FadeIn)
  ↓ START
Betting(라운드1) → BGM (CrossFade 1.5s)
  ↓ 배팅 확정
Countdown → bgm_exciting (CrossFade 1.5s)
  ↓
Racing → bgm_exciting 유지
  ↓
Result → bgm_exciting 유지 (다음 라운드 누를 때까지)
  ↓ 라운드 2+ 배팅 진입
Betting → BGM 복귀 (CrossFade 1.5s)
  ↓ 전체 라운드 종료
Finish / GameOver → bgm_ending (CrossFade 1.5s)
  ↓ 타이틀 복귀
TitleScene → Title_Bgm (FadeIn)
```

## BGMManager 아키텍처

```
BGMManager (DontDestroyOnLoad 싱글턴)
├─ AudioSource _sourceA  ← _active  (현재 재생 중)
└─ AudioSource _sourceB  ← _inactive (크로스페이드 대상)
```

### 듀얼 AudioSource 동시 크로스페이드

`CrossFade(path, duration)` 호출 시:
1. `_inactive`에 새 클립 로드 → `Play()`
2. 코루틴에서 매 프레임: `_active.volume ↓` + `_inactive.volume ↑` (동시 진행)
3. 완료 후: 역할 교체 (`_active ↔ _inactive`), 이전 소스 Stop + clip=null

```csharp
// CrossFadeRoutine 핵심
from.volume = Mathf.Lerp(fromStart, 0f,        t);
to.volume   = Mathf.Lerp(0f,        targetVol, t);
```

> 이전 구현(순차 FadeOut→FadeIn)과 달리 두 AudioSource가 **동시에** 재생되므로  
> duration 구간 동안 두 BGM이 겹쳐 들림.

## Public API

| 메서드 | 설명 |
|--------|------|
| `CrossFade(path, duration)` | 동시 크로스페이드 (주 전환 수단) |
| `PlayBGM(path, loop)` | 즉시 재생 (같은 클립 중복 방지) |
| `FadeIn(vol, dur)` | active 채널 볼륨 페이드인 |
| `FadeOut(dur, callback)` | active 채널 페이드아웃 → Stop |
| `StopBGM()` | 전체 정지 |
| `Mute(bool)` | 음소거 ON/OFF (양쪽 채널 동시 적용) |
| `SetVolume(vol)` | active 채널 볼륨 직접 설정 |
| `IsPlaying` | active 채널 재생 중 여부 |

## 주의사항 (BGM)

- **MCP 중 재생 자제**: Play/Recompile 중 BGMManager 좀비 인스턴스 발생 가능 → 씬 재로드로 해소
- **크로스페이드 중단**: 새 CrossFade 호출 시 진행 중인 코루틴 자동 취소 + inactive 정지
- **Enter Play Mode**: `OnDestroy`에서 `Instance = null` 처리 → Domain Reload OFF 환경 안전

## SFX 시스템 (2026-07-01~02 신설)

**동기**: BGM처럼 사운드 경로를 코드에 하드코딩(`"Audio/bgm_exciting"` 등)하면 파일 교체 때마다 재컴파일 필요 → 인스펙터에서 자유롭게 교체 가능한 데이터 주도 방식으로 재설계.

### 아키텍처 3층
```
SFXKeys (static class)          — 키 문자열이 존재하는 유일한 곳. 코드는 SFXKeys.RaceCollisionHit 심볼만 참조(오타=컴파일 에러)
SFXSettings (ScriptableObject)  — key→SFXEntry 매핑. Assets/Resources/SFXSettings.asset. 파일 경로는 코드에 전혀 없음(AudioClip 직접 참조)
SFXManager (기존 클릭전용 매니저 확장) — PlaySFX(key)/PlayLoop(key)/StopLoop()/GetEntry(key) API
```

### SFXEntry 필드
| 필드 | 설명 |
|---|---|
| `key` | SFXKeys 상수값과 동일 문자열 |
| `clips[]` | 재생 후보(드래그앤드롭). 비어있으면 **무음**(에러 없음 — 사운드 미제작 상태에서도 게임 진행 지장 없음) |
| `volume` | 개별 볼륨 배율, `[Range(0,3)]`(최대 3배), `GameSettings.sfxVolume`에 곱연산 |
| `loop` | 체크 시 지속 반복, 해제 시 1회만 |
| `maxConcurrent` | 동시 재생 상한(0=무제한) — 원샷 폴리포니 완화용 |
| `delay` | 재생 시작 지연(초) — 이벤트-사운드 타이밍 미세조정 |
| `loopInterval` | 루프 반복 사이 간격(초) — 0이면 끊김없이 바로 이어붙임, 예 0.1이면 매 반복마다 0.1초 쉬었다 재생 |

### 재생 경로 2종
- **원샷**(`PlaySFX`): `sfxSource.PlayOneShot` — `delay>0`이면 코루틴 지연 후 재생, `maxConcurrent` 카운터(코루틴, clip 길이만큼 자동 감소)
- **루프**(`PlayLoop`/`StopLoop`): 전역 루프 채널 1개(마네키네코 등). **코루틴 기반**(`LoopRoutine`) — `while(true) { Play(); yield WaitForSeconds(clip.length + loopInterval); }` 방식이라 반복 사이 간격 조절 가능(Unity 기본 `AudioSource.loop=true`는 gapless라 간격 조절 불가라 직접 구현)
- **레이서 로컬 루프**(발소리): `RacerController`가 자기 전용 `AudioSource`로 동일한 `RunLoopRoutine` 코루틴 재생. `GameManager.CurrentBet.selections.Contains(racerIndex)`로 **베팅 선택 캐릭터만** 재생(최대 3명, Trio/Wide) — 9명 전원이 발소리를 내면 소음이 되는 문제 해결

### 12개 키 (트리거 지점)
| 키 | 트리거 | 대상 |
|---|---|---|
| `sfx.race.collision.hit/dodge/slingshot` | `CollisionSystem.cs` 충돌 3종 | 9명 전원(쿨다운으로 실제 빈도 낮음) |

| `sfx.race.run` | `RacerController.StartRacing()` | 베팅선택 캐릭터만, loop 체크박스로 토글 |

### 하드코딩 방지 원칙
사운드 **파일 경로**는 코드에 전혀 없음(AudioClip 직접 참조로 원천 차단). **키 이름 문자열**도 각 호출부에 흩어지지 않고 `SFXKeys` 1곳에만 존재 — 호출부는 `SFXManager.Instance?.PlaySFX(SFXKeys.RaceCollisionHit)`처럼 상수 심볼만 참조.

### 클릭음 선택적 억제
`SFXManager.ScanAndAttachButtons()`(0.5s~2s 주기)가 씬의 모든 Button에 자동으로 클릭음을 붙이는데, 전용 SFX가 있어 일반 클릭음이 필요없는 버튼은 `SFXManager.SuppressAutoClick(button)`을 `OnEnable()`에서 미리 호출 — `SFXClickTag` 컴포넌트를 선점시켜 자동 스캔이 건너뛰게 함.

### 검증 도구 — `SFXSettingsValidator`
`StringTableValidator` 패턴 재사용, 단 **reflection 기반**(정규식 리터럴 스캔보다 신뢰도 높음) — `typeof(SFXKeys)`의 모든 const 필드를 직접 수집해 `SFXSettings.asset` 등록 키와 대조. 메뉴 `DopamineRace > Validate SFX Keys`. 검출: 누락/중복key/미할당clip(경고, 무음은 유효상태라 에러 아님)/미사용key.

### 미러 규칙 예외
`RacerController`/`RacerController_V4` 수정 시 `RaceBacktestWindow` 동시수정 원칙(CLAUDE.md)의 **명시적 예외** — 헤드리스 `SimRacer`는 `MonoBehaviour`를 상속하지 않는 순수 POCO라 `AudioSource`/`GetComponent` 자체가 존재하지 않음. 사운드는 프레젠테이션 레이어라 미러 대상이 아니며, 실수로 미러에 추가 시도해도 컴파일 에러로 즉시 차단됨.
