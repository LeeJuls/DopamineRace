# 통화 시스템

> **상태**: ✅ Phase 1~3 완료, Phase 4~5 (UGS) 대기  
> **핵심**: 젤리(베팅 자원) + 스톤(랭킹 점수) 이중 통화 + 럭키 잭팟(스톤→젤리)  
> **⚠️ SPEC-047 (2026-06-27)**: 경제 단방향 전환 — 적중 시 젤리 반환 없음  
> **⚠️ SPEC-051 (2026-07-01)**: 환전/구제/고양이의 힘 → 마네키네코 럭키 잭팟 통합 + 0젤리 게임오버 버그픽스

---

## 통화 구조

| 통화 | 아이콘 | 휘발성 | 현재 용도 |
|------|--------|--------|----------|
| 🟦 **도파민 젤리** | `DopamineJelly.asset` | 게임마다 100 리셋 | 베팅 소모 자원 (환전 받는 통화) |
| 💎 **도파민 스톤** | `DopamineStone.asset` | 랭킹 제출 후 리셋 | 글로벌 랭킹 점수 + 환전 재원 |

## 게임 시작 조건

모든 유저 동일 출발선: **젤리 100, 스톤 0**

## 보상 공식 (SPEC-047 이후)

```
적중: 스톤 += CeilToInt(베팅액 × 배당)  ← 젤리는 반환하지 않음
빗나감: 젤리 -= 베팅액 (전액 손실), 스톤 변화 없음
```

> **설계 의도**: 젤리는 배팅 시 소멸. 적중해도 젤리가 늘지 않아 복리 증식이 불가.
> 희소한 젤리를 쪼개며 스톤을 쌓는 로그라이크 긴장감 유지.

### SPEC-047 이전 구공식 (폐기됨)

```
[구] 적중: 젤리 += CeilToInt(베팅액 × 배당), 스톤 += 베팅액
```

구공식은 젤리가 복리로 증가해 배팅 압박감·구제 시스템·고양이 환전 의미가 희석됨 → 단방향으로 전환.

## 럭키 잭팟 (스톤 → 젤리) — SPEC-051 (2026-07-01)

> 구 환전/구제/고양이의 힘(라운드별 비율 2~10:1)을 **"마네키네코" 럭키 잭팟** 단일 메커니즘으로 재설계.

### 규칙

| 항목 | 값 |
|------|-----|
| 산식 | 보유 스톤 **전부 × N 젤리** (N=2~8 가중 `[18,18,18,16,14,10,6]`), **무캡** |
| 성격 | **항상 이득** (1스톤 → 최소 2젤리, 손해 불가) — 럭키 리워드 |
| 빈도 | **게임당 1회** (자발 고양이 클릭 + 0젤리 자동 트리거가 1회 공유) |
| 0젤리 자동 | game over 대신 **강제 잭팟 모달**(닫기·backdrop·ESC 차단), 최소 `floor 5젤리` 보장 |
| 구제 | 잭팟에 통합 (별도 rescue 제거) |

### Game Over 조건 (SSOT 재정의 — ★버그픽스)

```csharp
public bool CanJackpot()     => JackpotUsesLeft > 0 && _stone >= 1;
public bool ShouldGameOver() => _jelly == 0 && !CanJackpot();
```

→ 0젤리라도 **(잭팟 횟수 남음 && 스톤≥1)** 이면 잭팟으로 생존. **버그**: 구 `TryShowBetAmountModal`이 `Jelly<1`에서 스톤 무시하고 즉시 game over 하던 모순 → 잭팟 제시로 교체.

| 젤리 | 스톤 | 잭팟 횟수 | 결과 |
|------|------|----------|------|
| >0 | - | - | ❌ 정상 |
| 0 | ≥1 | 남음 | 잭팟 모달 (자동) |
| 0 | 0 | - | **✅ Game Over** |
| 0 | ≥1 | 0(소진) | **✅ Game Over** (P1) |

### 무캡 안전성 (balance 40~60만회 백테스트)

최종 리더보드 점수 기준 **무캡 ≈ 캡 ≈ 잭팟없음** (전 분위 구분 불가). 3중 자기교정:
스톤=LeaderboardScore(선소모) + 하우스엣지(오버라운드 1.20 → −17%EV) + **2라운드 천장**.
> ⚠️ **재검증 트리거**: ① roundLaps 대폭↑ ② 배당 +EV 전환 ③ 잭팟 2회+/게임.

### GameSettings 필드

`jackpotMultiplierMin/Max(2/8)` · `jackpotWeights[18,18,18,16,14,10,6]` · `jackpotRescueFloorJelly(5)` · `jackpotUsesPerGame(1)`. (캡 필드 없음 — 무캡)

### UI (마네키네코)

- 제목 **마네키네코**(招き猫), 버튼 **고양이의 보은**(猫の恩返し).
- 연출: 검정 반투명 판 + 밝은 노랑 롤(115pt) + **입력 잠금**(`_isRolling`) → 결과 "**+{N} 도파민 젤리 획득**" + 실제 젤리 아이콘(RewardBurst, `🟦` 이모지 픽셀폰트 미렌더 → 스프라이트).
- 키: `str.jackpot.*` (8언어).

## 관련 클래스

- `WalletManager` — 젤리/스톤 상태, **`CanJackpot()`·`TryJackpot(autoFloor)`·`RollJackpotN()`·`JackpotUsesLeft`·`LastJackpotJellyGain`·`ShouldGameOver()`(재정의)** (SPEC-051 — 구 환전/구제/고양이 API 제거), `Reward(int stoneGain)`
- `GameManager.CalcScore` — 레이스 결과 → `Reward(stone)`만 호출 (SPEC-047)
- `BettingData.BetReward` — `stone` + `hit` 필드 (SPEC-047에서 `jelly` 필드 제거)
- `ExchangeModal` — **럭키 잭팟 UI**(룰렛 연출·`ShowAuto()` 강제 모달) (SPEC-051)
- `RewardBurst` — 잭팟 획득 연출 "+N + 젤리 아이콘"
- `CurrencyHeader` — 상단 통화 표시

## 아이템 정보 팝업 (SPEC-050)

CurrencyHeader의 젤리·스톤 아이콘 클릭 → 이름+설명 박스(트랙 설명창 스타일).

| 컴포넌트 | 역할 |
|----------|------|
| `ItemInfoPopup` | 팝업 본체. `Instance` 싱글턴, `Show(nameKey,descKey)`/`Hide()` |
| `ItemInfoTrigger` | 클릭 트리거(모듈). nameKey·descKey만 지정하면 어느 아이템에든 재사용. 토글 |
| `ItemInfoPopup.prefab` | 프리팹 — BettingPanel 자식, Canvas(`overrideSorting=true, sortingOrder=2000`)+GraphicRaycaster |

- **정렬**: `sortingOrder=2000` → BetAmountModal(1000)보다 위에 렌더. BettingPanel(~0)에서도 최상단.
- 키 규칙: `str.dopamine.item.{name|desc}.{아이템id}` (예: `...dopaminejelly`). 향후 아이템 표준.
- 설명 내용 출처: SPEC-040 세계관설정집 §8 (젤리=기대 에너지, 스톤=결정화된 믿음).
- **BetAmountModal 재사용 (2026-06-30)**: `BettableClickOverlay`(투명 Image + `ItemInfoTrigger`)가 모달 우상단 전체 영역의 클릭을 수신 → 동일 팝업을 모달 위에 표시.

→ 상세: `Docs/specs/SPEC-050_아이템정보팝업_명세서_20260629.md`
