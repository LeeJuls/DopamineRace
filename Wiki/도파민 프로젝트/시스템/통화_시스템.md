# 통화 시스템

> **상태**: ✅ Phase 1~3 완료, Phase 4~5 (UGS) 대기  
> **핵심**: 젤리(베팅 자원) + 스톤(랭킹 점수) 이중 통화 + 환전 시스템

---

## 통화 구조

| 통화 | 아이콘 | 휘발성 | 현재 용도 |
|------|--------|--------|----------|
| 🟦 **도파민 젤리** | `DopamineJelly.asset` | 게임마다 100 리셋 | 베팅 자원 + 환전 받는 통화 |
| 💎 **도파민 스톤** | `DopamineStone.asset` | 랭킹 제출 후 리셋 | 글로벌 랭킹 점수 + 환전 재원 |

---

## 게임 시작 조건

모든 유저 동일 출발선: **젤리 100, 스톤 0**

---

## 보상 공식

```
적중: 젤리 += CeilToInt(베팅액 × 배당), 스톤 += 베팅액
빗나감: 젤리 -= 베팅액 (전액 손실), 스톤 변화 없음
```

---

## 환전 시스템 (스톤 → 젤리)

### 규칙

| 조건 | 처리 |
|------|------|
| 매 라운드 시작 시 | 환전 비율 2~10:1 랜덤 결정 |
| 라운드당 1회 | `ExchangedThisRound` 플래그 |
| 정상 환전 | 스톤 ≥ 비율 → 비율만큼 차감, 젤리 +1 |
| 구제 환전 | 젤리=0 AND 스톤≥1 AND 스톤<비율 → 스톤 전부 → 젤리 1 |

### Game Over 조건

```csharp
public bool ShouldGameOver()
{
    if (Jelly > 0) return false;
    return !CanExchange();  // 환전도 불가하면 종료
}
```

→ 사실상: **젤리=0 AND 스톤=0** 또는 **이미 환전 사용** 상황에서만 종료

### 시나리오 예시 (비율=5 가정)

| 젤리 | 스톤 | Game Over? |
|------|------|-----------|
| >0 | - | ❌ |
| 0 | 0 | **✅** |
| 0 | 1~4 | ❌ (구제) |
| 0 | 5+ | ❌ (정상) |

## ScriptableObject 구조

```
Assets/Resources/Items/
├─ DopamineJelly.asset    (CurrencyItem)
└─ DopamineStone.asset    (CurrencyItem)

CurrencyItem {
  string id;          // "jelly" / "stone"
  string nameKey;     // StringTable UID
  string descKey;     // StringTable UID
  Sprite icon;        // 디자이너 PNG 교체 가능
  Color defaultColor; // UI 강조 색상
}
```

## CurrencyHeader UI

배팅 화면 상단 기존 `총점` 영역 → 젤리/스톤 실시간 표시로 교체.

## 관련 클래스

- `WalletManager` — 젤리/스톤 상태 관리, `CanExchange()`, `TryExchange()`
- `GameManager.CalcScore` — 레이스 결과 → Wallet 업데이트
- `ExchangeModal` — 환전 팝업 UI
- `CurrencyHeader` — 상단 통화 표시
