# 트랙 시스템

> **상태**: ✅ 5트랙 선택·배경·충돌 차별화 작동 / ✅ 데이터 주도 스탯 상성 4트랙 완비·**격리 백테스트 검증 완료** — 사막=Power0.04·눈=Stamina0.10·고원=Int0.04·비=Luck0.07 (2026-07-02)
> **핵심**: `TrackDB.csv`(런타임 소스) → `TrackInfo.ToTrackData()` → `GameSettings.currentTrack`. 스탯 상성은 `stat_affinity` 콜론 인코딩 → 공유 헬퍼로 실엔진·백테스트 동일 계산.

## 5트랙 개요

| 트랙 | 아이콘 | weight | 살아있는 특성(V4) | 스탯 상성 |
|------|-------|--------|------------------|-----------|
| normal | 🏟️ | 100 | 기준(모든 배율 1.0) | — |
| rainy | 🌧️ | 80 | 충돌범위↑1.4·페널티↓0.6·슬링샷↓0.7·luck_mul 1.3 | **Luck:0.07** (비=운, luck_mul 시너지) |
| snow | ❄️ | 60 | 충돌페널티↑1.5·슬링샷↑1.6·luck_mul 0.7 | **Stamina:0.10** |
| desert | 🏜️ | 70 | luck_mul 1.2·loser_penalty↑1.6 | **Power:0.04** |
| highland | 🏔️ | 50 | 충돌범위↓0.7·페널티↓0.7·luck_mul 0.8 | **Intelligence:0.04** |

> **4트랙 4스탯 완비**(2026-07-02): 사막=Power·눈=Stamina·고원=Int·비=Luck 서로 다른 스탯 → 언더독 다양성. normal은 무상성(순수 스탯 기준). 계수는 goyo는 의도적 최강+희박등장이라 baseline 유지, **언더독 이득에 초점** → 격리 백테스트로 확정: desert 0.08→**0.04**, 고원 Int 0.10→**0.04**(goyo 과증폭 차단), 비 Luck 0.10→**0.07**(goyo push 반감), 눈 Stamina 0.10(유지). 상세 [밸런스](#밸런스-2026-07-02-격리-백테스트-검증-완료) 참조.

**선택 규칙**(`TrackDatabase`): R1=normal 고정 → R2+ 이전 트랙 제외 가중 랜덤. `GameSettings.roundTracks`로 라운드별 강제 지정 가능.

## ⚠ V4 레버 Live/Dead 현황 (2026-07-02 실측)

TrackDB.csv에 24개 배율 컬럼이 있으나 **V4 실엔진에서 상당수가 死**(기획은 V1-V3 스탯 기준, V4 재설계 때 Brave/Calm/Endurance 스탯 소멸). `CalcSpeedV4`/`ConsumeStaminaV4`는 트랙을 stat_affinity 외엔 참조하지 않음.

| 레버 | V4 상태 | 비고 |
|------|--------|------|
| collision_range/penalty/slingshot/loser_dur | **Live** | `CollisionSystem`·백테스트 `SimCollisions`. 단 입력이 레거시 스탯(charBasePower/charBaseBrave) |
| luck_mul | **Live(오배선)** | `TryDodge`의 Int 회피 확률에 곱해짐(이름과 불일치) |
| **stat_affinity** | **Live (2026-07-02 신설)** | Power/Intelligence → vmax 곱보너스. 아래 상세 |
| speed_mul | **사실상 死** | 스폰 초기값(`GetBaseSpeed`)만, `CalcSpeedV4`가 Lerp로 덮음 |
| noise_mul · fatigue_mul | **死** | 겨냥 스탯(Calm/Endurance)이 V4에 없음. 미배선 |
| has_mid_slow_zone (고원 오르막) | **死** | `GetSlowZoneMultiplier` 호출처 0건 |
| early/mid/late_bonus_mul · power_speed_bonus · brave_speed_bonus | **死** | 미사용. power/brave_speed_bonus는 stat_affinity로 대체됨([DEPRECATED], 차기 컬럼 정리) |

## 데이터 주도 스탯 상성 (stat_affinity)

트랙별 "어떤 스탯이 속도에 유리한가"를 **CSV 데이터만으로** 정의(하드코딩 없음). 프로젝트 패시브 스킬(`char_passive`)의 콜론 인코딩 패턴 미러.

### CSV 문법 (`stat_affinity` 컬럼)
```
"Power:0.06"                    ← 사막: Power 유리
"Intelligence:0.07"             ← 고원: Intelligence 유리
"Power:0.05;Intelligence:0.03"  ← 복합(세미콜론 다중, 중복 스탯 합산)
"" 또는 "none"                  ← 무보너스
```
- 스탯명: `Speed/Accel/Stamina/Power/Intelligence/Luck` (`V4StatType`, 대소문자 무관)
- 계수: `[0, BONUS_MAX=0.15]` 클램프. 파싱 실패 항목은 graceful 무시(무보너스).

### 공식 (`TrackStatAffinity.ComputeVmaxMultiplier`)
```
vmax ×= 1 + Σ (스탯값 × HiddenWeight / 20 × bonus)
```
`/20` = 만렙(20) 정규화(만렙·weight=1일 때 항 = bonus). `HiddenStatWeights` 곱 = V4 스탯 사용처 일관성(밸런스 튜닝 레버 유지).

### 미러 = 공유 헬퍼 (드리프트 원천 차단)
실엔진 `RacerController_V4.CalcSpeedV4`(vmax 선언 직후)와 백테스트 `RaceBacktestWindow.SimTickV4`가 **같은 static 헬퍼 `TrackStatAffinity.ComputeVmaxMultiplier(cd, affs)` 호출** → 코드 복붙 없음 = 미러 드리프트 불가능. (헬퍼는 non-Editor `Assets/Scripts/Data/`에 위치, 매 틱 호출이라 순수 산술·무로그)

### 롤백 스위치 (feature flag)
`GameSettingsV4.v4_enableTrackStatBonus`(기본 true). **off → stat_affinity 완전 무효 → 기존 트랙 동작 즉시 복귀**(다른 트랙 레버는 유지). `if (flag && track != null)` 가드로 일반 트랙/빈 affinity는 원동작 보존(회귀 0).

### 백테스트 CSV 소스화
백테스트 `selectedTrack`을 asset 드래그 → **`RunHeadless(...,trackId)`로 CSV에서 `ToTrackData()` 빌드**(`BuildTrackFromCsv`). 런타임=백테스트 동일 소스(CSV) → asset 이원화·파리티 버그 제거.

## 확장 경계 (T1 데이터만 / T2 코드 필요)

- **T1 (CSV 데이터만, 코드 0)**: 기존 6스탯 × vmax 곱보너스. **실증**: 눈=`"Stamina:0.10"`·비=`"Luck:0.07"`를 코드 0으로 추가해 4트랙 완비(2026-07-02). 추가 트랙/스탯도 CSV 셀만으로 확장.
- **T2 (코드 필요)**: 새 효과종류(fatigue→드레인·충돌 슬링샷·구간 차등·slow_zone 부활). 현 구조 표현력은 `{스탯}×{vmax 곱}` 2차원뿐. 死 레버 부활은 여기 해당(`TrackEffectKind` enum 도입 검토).

## 밸런스 (2026-07-02 격리 백테스트 검증 완료)

**밸런스 철학(오너)**: goyo류 최강캐는 의도적(+등장 희박) → 일부러 불리하게 안 함. 대신 **언더독이 이득 보는 맵**을 지향.

**측정법(핵심)**: 트랙 vs normal 델타는 상성 계수 + 트랙 고유 레버(loser_penalty·collision·luck_mul)가 뒤섞임 → 계수 튜닝엔 부적합. **FULL(상성 활성) vs NOAFF(`statAffinities`만 비움, 다른 레버 유지)를 동일시드 페어드** 비교해 **계수만의 순수 기여(affΔ)**를 격리 → 이것이 올바른 계수 판정 지표. (BuildTrackFromCsv로 FULL/NOAFF 트랙 빌드 후 `RunSimulationCore` 페어드; goyo는 P16/I20/S20/L20이라 모든 유리군에 껴서 별도 분리)

**격리 검증 결과**(N=2500~3000, seed=20260702, r=스탯↔affΔ 상관, affΔ=계수 순수 기여):
| 트랙 | 계수 | r | goyo affΔ | 언더독군 affΔ | 판정 |
|------|------|---|-----------|--------------|------|
| snow | Stamina:0.10 | 0.76 | **+0.6%p** | +1.9%p | ✅ 이상적 언더독맵 |
| desert | Power:0.04 | 0.59 | +3.5%p | +1.7%p | ✅ 양호 |
| rainy | Luck:0.07 | 0.72 | +2.9%p | +0.3%p | ✅ 군 얇음·개별 로또팝 |
| highland | Int:0.04 | 0.65 | +3.8%p | +0.3%p | ✅ goyo 71%(normal↓) 억제 |

- **snow = 정답지**: 상성이 언더독에만 흐름(hongyeom +8%p·goldenday +7%p), goyo 무반응(+0.6). Stamina는 선행마엔 여분 속도라 中위권만 승수로 전환.
- **desert 0.04**: 언더독 +1.7%p, goyo +3.5%p. desert의 goyo 강세(81%)는 대부분 loser_penalty 1.6이지 Power:0.04 아님(noaff 이미 78%).
- **highland Int 0.10→0.04(하향) — 핵심 원리**: 고원의 언더독 성질은 충돌↓ 레버(goyo를 67%로 억제)에서 나옴. Int 상성은 goyo(Int20)에 부익부라 **0.10이면 goyo를 +8~10%p로 78%까지 되돌려 언더독맵을 무효화**. 임계 ≈ **0.048**(goyo=normal 72.5% 교차) — 계수 스윕으로 확증(0.03→goyo 71.9%, 0.04→71.1%, 0.05→73.0%, 0.06→74.8%, 0.10→76.8%). **0.04**면 goyo 71%(normal↓) 유지로 언더독맵 보존 + 최소 Int 향미. Int-언더독군 이득은 계수 무관 +0.4%p로 미미 → **Int는 언더독 보상엔 약한 스탯**(고원의 언더독성은 100% 충돌 레버).
- **rainy Luck 0.10→0.07(하향)**: goyo affΔ 반감(+4.7→+2.9), 언더독군 이득 동일(+0.3). Luck 보유 26/32라 군 변별력 구조적으로 약함. rainy의 goyo 강세(80%)는 rainy 고유 레버(collision 1.4·luck_mul 1.3, noaff 이미 77%)이지 Luck:0.07 아님. 개별 언더독 로또팝(amazon +8%p·zen +6%p, 승률 7~8% 캐릭)이 럭 테마 부합.
- **지배 임계**: desert Power≥0.12 · highland Int≥0.05(goyo>normal)에서 goyo맵化 → 현재값(0.04) 안전 마진.

> 계수는 `TrackDB.csv` `stat_affinity` 셀 1개 변경으로 재튜닝(코드·재컴파일 불필요, CSV 리임포트만). 재검증: BuildTrackFromCsv FULL/NOAFF 페어드로 계수 순수 기여 격리 측정.

## 주의사항

- **하드코딩 금지**: 트랙별 유리 스탯·계수는 CSV `stat_affinity`에만. 코드는 값 무관.
- **미러 규칙**: `CalcSpeedV4` 수정 시 `SimTickV4`도 — 단 stat_affinity는 공유 헬퍼라 자동 정합.
- **Validator**: `ParseList`가 graceful(오타 침묵 제거)이라 오타 스탯명이 무경고로 묻힘 → 커밋 전 `Validate Track Data` 실행 권장.
- **[DEPRECATED] 잔재**: `power_speed_bonus`/`brave_speed_bonus` CSV 컬럼·`TrackData/TrackInfo` 필드는 stat_affinity로 대체됨(현재 무해·미사용, 차기 컬럼 정리 예정).
