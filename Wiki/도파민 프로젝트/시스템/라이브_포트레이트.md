# 라이브 포트레이트 (SPEC-049)

> 캐릭터 정보창의 일러스트(PNG)를 **mp4 비디오**로 전환. 하이브리드 폴백(비디오 없으면 PNG).
> Raw: `Docs/specs/SPEC-049_*` · `Docs/history/라이브포트레이트_히스토리_20260628.md` | 상태: ✅ 구현 완료 (A~G, 빌드 스모크만 owner 잔여)

---

## 1. 개요

- **범위**: 캐릭터 정보창([[배팅_시스템|배팅 화면]]의 `CharacterInfoPopup`)만. 무음·루프 영상.
- **하이브리드 폴백**: `char_video` 있으면 비디오, 없으면 기존 `char_illustration`(PNG).
- **플랫폼**: Standalone 전용. WebGL은 자동 PNG 폴백.
- **8개 캐릭터** 비디오 보유: Character_0, 1, 2, 3, 5, 7, 23, 26.

## 2. §0 측정 (확정)

8개 mp4 전부 **1:1 정사각**(480²·640²), PNG도 1:1(256²) → **Δ종횡비 0%**.

| 항목 | 값 |
|------|-----|
| 레이아웃 | `AspectRatioFitter.HeightControlsWidth` |
| RenderTexture | **640×640**, `filterMode=Point` (픽셀아트 톤) |
| 코덱 | H264, 길이 ~5초, fps 16/24/32 혼재(단계 E에서 통일 검토) |

## 3. 데이터

- `CharacterDB_V4.csv` **col20 = `char_video`** (맨끝 추가 → 기존 인덱스 불변). 빈 값이면 PNG 폴백.
- `CharacterData.VideoResourcePath` / `CharacterDataV4.VideoResourcePath` getter — 경로 정규화(역슬래시→/, `.mp4` 제거), 빈값이면 `null`.
- mp4 위치: `Assets/Resources/Icon/Char/video/Character_N.mp4` ↔ `Resources.Load<VideoClip>("Icon/Char/video/Character_N")`.

## 4. 아키텍처

### CharacterVideoController (런타임 래퍼)
**경로**: `Assets/Scripts/Manager/UI/CharacterVideoController.cs` — [[../아키텍처/주요_클래스]] 참조.
- `VideoPlayer → RenderTexture(640² Point) → RawImage`. 무음(`audioOutputMode=None`)·루프·`waitForFirstFrame`·`skipOnDrop`.
- `TryPlayForCharacter(CharacterData)` — 클립 있으면 prepare 후 true, 없으면 false.
- 이벤트 `OnPrepared(float aspect)`(준비 완료, RawImage 0→1 페이드인) / `OnFallback()`(실패·타임아웃).
- prepare race **토큰 가드**, `errorReceived`/타임아웃(2s)→`OnFallback`, 크로스페이드 0.18s. RectTransform 미변경.

### CharacterVideoPrefabPatcher (에디터)
**경로**: `Assets/Scripts/Editor/CharacterVideoPrefabPatcher.cs` — `DopamineRace > Patch IllustrationVideo (SPEC-049)`.
- `IllustrationMask` 자식에 `IllustrationVideo`(RawImage + AspectRatioFitter) 추가.
- RectTransform 초기값을 **형제 `Illustration`에서 복사**(디자이너 조정값 보존, 하드코딩 금지).
- `LoadPrefabContents → FindDeep → SaveAsPrefabAsset`, try/finally 언로드, 멱등 가드.
- `BettingUIPrefabCreator`에도 동기 생성(전체 재생성 시 노드 유실 방지).

### CharacterInfoPopup 통합
**경로**: `Assets/Scripts/Manager/UI/CharacterInfoPopup.cs`.
- `Init()`: `IllustrationVideo` 탐색 → `CharacterVideoController` get-or-add → 이벤트 구독.
- `Show()`: PNG 가시 복구 후 비디오 분기. `OnPrepared`→PNG 크로스페이드아웃, `OnFallback`→PNG 유지.
- `Hide()`/`OnDisable()`: `StopAndRelease`(씬 전환 누수 가드). `OnDestroy()`: 구독 해제.
- charId 전환은 컨트롤러 내부 토큰 가드 + Show마다 `StopAndRelease`.

## 6. 알려진 이슈 / 함정

- **`Cannot Prepare a disabled VideoPlayer`** (단계 D 버그, 수정됨 `434b256`): `VideoPlayer.Prepare()`는 활성 GameObject에서만 가능. `TryPlayForCharacter`는 반드시 `gameObject.SetActive(true)` **이후** 호출해야 함. 위반 시 첫 정보창 오픈에서 prepare 실패→2s 타임아웃→PNG 폴백(영상 안 뜸).
- ~~**`Color primaries 0 ... unsupported by WindowsMediaFoundation`**~~ **✅ 근본 해소(2026-06-28)**: 원인은 6개 mp4(Character_2·3·5·7·23·26)의 H264 **SPS VUI에 색공간 메타 미지정**. WMF는 SPS VUI만 읽음(mp4 `colr` atom 무시). 해결 = ① ffmpeg `h264_metadata` bsf로 SPS VUI에 bt709 **무손실** 주입(`-c copy`) + ② Unity 트랜스코드 **OFF**(트랜스코드 출력은 메타를 다시 날리므로 원본 직접 재생). Character_0·1은 이미 bt709라 제외. owner Play 검증 경고 0건. 계획: `Docs/plans/colorprimaries_fps_재인코딩_20260628.md`.
- **Media Foundation 코덱 미지원** 환경 → `errorReceived`→PNG 폴백(설계된 동작).
- 트랜스코드는 **`[MenuItem]`/batchmode로 owner 직접 실행** (MCP `SaveAndReimport`는 좀비 위험 — [[../워크플로우/MCP_사용법]]).
