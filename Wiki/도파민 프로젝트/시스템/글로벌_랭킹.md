# 글로벌 랭킹

> **상태**: ⏸ Phase 4 (UGS 통합) + Phase 5 (랭킹 뷰) 미착수  
> **의존**: Phase 4는 오너의 UGS Dashboard 셋업 사전 작업 필요

---

## 개요

마지막 라운드 완주 시 누적 **도파민 스톤** → 글로벌 랭킹 제출.  
"오락실 점수 등록" 스타일: 닉네임 1회 입력, 변경 불가.

---

## 랭킹 규칙

| 항목 | 내용 |
|------|------|
| 제출 조건 | 마지막 라운드 완주 (Game Over 시 제출 불가) |
| 점수 | 누적 도파민 스톤 |
| 컷오프 | 초기 Top 100명 → 1,000 → 10,000 점진 확대 |
| 닉네임 | 랭킹 등록 시 1회 입력 (이후 변경 불가) |
| 스톤 처리 | 랭킹 제출 후 스톤 리셋 (재도전 시 0부터) |

---

## 진입점

- **TitleScene**: START 버튼 위 "글로벌 랭킹" 별도 버튼 (대기 중 즉시 접근)

---

## 분기 흐름

```
마지막 라운드 결과
  ├─ Top 10K 진입
  │     └─ NicknameInputModal → 닉네임 입력
  │           └─ LeaderboardService.Submit(stones, nickname)
  │                 └─ 스톤 리셋 → GlobalRankingView 표시
  └─ 미진입
        └─ "랭킹 미진입" 안내 → 스톤 리셋 → 재시작
```

---

## 기술 구조

### UGS (Unity Gaming Services)

- `UgsBootstrap.cs` — UGS 초기화
- `LeaderboardService.cs` — 점수 제출, 조회
- `BadwordFilter.cs` — 닉네임 차단어 1차 필터 (BadwordList.csv)
- 오프라인 폴백: 랭킹 없이 게임 진행 가능

### 화면 구성

- `GlobalRankingPrefab` — Top 10 + 본인 순위 표시
- `NicknameInputModal` — 닉네임 입력, 차단어 검증
- 페이지네이션 (Phase 5)

---

## 차단어 운영

- 초기: 한국어 위주 ~100단어 (오너 직접 작성)
- 신고 기반 추가 운영
- UGS 측 제재 기능과 별개로 클라이언트 1차 필터
