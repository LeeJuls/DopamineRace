## 텍스트 자동 축소 — UITextFit (2026-07-02)

es/br/de 장문·CJK 세로 삐짐으로 UI 텍스트가 박스를 넘는 문제의 공용 해법. **박스(rect)는 고정, 넘칠 것 같으면 폰트만 축소**(오너 요구). `Assets/Scripts/Utility/UITextFit.cs`, 전부 **런타임 적용**(프리팹 무변경 — 롤백=호출 제거).

| API | 용도 | 규칙 |
|---|---|---|
| `Shrink(t, min=10, max=0)` | 일반 라벨 Best Fit 중앙화 | max≤0이면 **현재 fontSize를 상한**(프리팹 수작업 값 존중). ★ `horizontalOverflow=Wrap` 필수 — **Overflow면 Best Fit이 폭 제약을 무시**(S0 실측: extents100px·202px 텍스트가 Overflow=28 유지 / Wrap=14 축소) |
| `FitRichText(t)` | `<size=N>` 태그 텍스트 | Best Fit은 인라인 size 태그를 못 줄임 → 태그값+fontSize를 **비율 1회 축소**(넘칠 때만, 스타일 보존). 스킬바(30/26)용 |
| `FitFontSize(font, labels, base, budget)` | 라이브러리 소유 Text | XCharts 레이더처럼 Best Fit 불가한 곳(라벨 sizeDelta를 preferredWidth로 강제 재설정)의 `textStyle.fontSize` 계산용 |

**적용처(2026-07-02)**: 캐릭터팝업(타입칩·거리라벨×3·레이더 축라벨·스킬설명) + 배팅바(타입탭×6·배당라벨/공식). 배당라벨은 point앵커+고정 200px → **OddsArea stretch 전환+탭 클리어존**(탭이 OddsArea와 원래 92px 겹침) 선행.

검증(8언어 하네스): ko/en/jp 원크기 유지(회귀 0), cn/tw 스킬 30→26·28 축소로 세로 수납, es/br 칩 26→19·레이더 28→24·배당 겹침 해소. min 하한 10px(픽셀폰트 판독성).

**함정 메모**: ① Best Fit+Overflow는 무동작(Wrap 필수) ② rect 미확정 시점(빌드 직후) 폭 계산 금지 — `Canvas.ForceUpdateCanvases()` 선행 ③ XCharts 라벨은 매 리프레시 sizeDelta 재설정이라 후처리 Best Fit 불가(fontSize 사전 계산만) ④ 레이더 라벨은 한쪽 전폭 확장 케이스가 있어 예산은 반폭(×2 금지).

**차기**: 레이더 스탯 축약 CSV 키(`str.ui.char.stat.*.short`), 16:9 미만 해상도 잔여 리스크, 타 UI 전수 스윕(헬퍼 재사용).

## 캐릭터 시나리오 번역 (2026-07-02)

> 32명 캐릭터 스토리(`str.char.NNN.scenario`)를 ko 원본 → **7개 외국어**(en·jp·cn·de·es·br·tw) 직번역. 애니메이션체 톤. 자세한 배경은 [[../캐릭터/캐릭터_개요]] 참조.

### 규모·방식
- **32명 × 7외국어 = 224셀**. 020 꽃비 파일럿 1행 → 4파벌 병렬 번역 → C# 배치로 CSV 반영.
- 세계관 정본은 [[../캐릭터/캐릭터_개요]]의 **B안 〈도파미나 제국〉** — 번역도 이 정본에 정합.

### 시맨틱 컬러 태그 (콘텐츠 삽입 규약)
스토리 텍스트에는 표시 시 색으로 치환되는 마크업을 넣는다. 렌더링은 `CharacterStoryPopup.FormatStoryText`(→ [[../아키텍처/주요_클래스]]) 담당.

| 태그/마커 | 색 | 용도 |
|-----------|-----|------|
| `[skill]...[/skill]` | 골드 `#A67C00` + bold | 스탯 수치+효과 (숫자·% 있는 것만) |
| `[key]...[/key]` | 오렌지 `#D9622B` | 출신세계 고유명사 + 소원 핵심구 |
| `§TIP§`(마커) | 블루 `#3B6EA5` 이탤릭 | 배경/배팅 팁 자동 분리 |

- **번역 시 태그·마커·수치는 그대로 보존** (내용만 번역, 마크업은 언어 무관 고정).
- 태그 규약: 중첩 금지 · 한 문장 내 · 여는·닫는 쌍 필수 · 캐릭터당 2~5개.

### 용어 일치 규칙
| 항목 | 참조 키 | 규칙 |
|------|---------|------|
| 스탯 용어 | `str.ui.char.stat.*` | 시나리오 내 스탯 명칭은 이 키의 각 언어 표기와 **정확히 일치** |
| 캐릭터명 | `str.char.NNN.name` | 시나리오에서 인물 지칭 시 name 키 표기와 일치 |
| 슬립스트림 | `str.skill.p.*.slip.*`(공식) | 아래 표준어로 통일 |

### ★슬립(slipstream) 용어 표준화
파벌별로 갈렸던 표기(滑行·打滑·Ausrutschen 등)를 **공식 스킬 키**(`str.skill.p.*.slip.*`)의 표기로 통일.

| 언어 | 표준어 |
|------|--------|
| cn (간체) | **气流** |
| tw (번체) | **氣流** |
| de | **Sog** |
| es | **Rebufo** |
| br | **Vácuo** |
| en | Slipstream · jp スリップ (기존 유지) |

### CSV 안전·검증
- CSV 안전: ASCII 큰따옴표 회피 → 전각 「」 사용 (필드 파싱 깨짐 방지).
- 검증(`execute_csharp` 전수): §TIP§ 264개 · 태그쌍 균형 · 전 행 9필드 · **리터럴 누출 0 · 한글 잔존 0**.

### 잔여
- 새벽(032)은 `str.char.032.name` 키 없는 미완성 캐릭터 → 참조 시 Dawn/黎明 임의역, 캐릭터 완성 시 통일.
- 출시 전 전 언어 네이티브 검수 권장.
