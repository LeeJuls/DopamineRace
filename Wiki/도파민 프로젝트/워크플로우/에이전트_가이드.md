# 에이전트 가이드

## leader (PM)

**언제 사용**: 기획 검토, 작업 분배, SPEC/HIST 작성, 에이전트 간 이견 중재

**업무**:
1. 오더 수신 → 기획 검토 → SPEC 작성 → 오너 컨펌
2. balance · client · qa 에이전트 배분·조율
3. 오너 보고 시점: SPEC 완성 · HIST 완성 · push 직전

**기획 기준**: "유저가 레이싱 보는 맛이 우마무스메처럼 쫄깃한가?"

---

## balance (밸런스)

**언제 사용**: 밸런스 분석, GameSettingsV4 수치 조정, HP/CP 시스템 검토

**전문 도메인**: 우마무스메 레이스 메커니즘, 경마 전략, 확률 배팅

**밸런스 기준**:

| 지표 | 목표 | 경고 기준 |
|------|------|----------|
| 타입별 1위율 | 10~13% 균등 | 편차 >5% |
| 캐릭터별 승률 | 5~20% | >30% or <2% |
| 역전 발생 구간 | 70~100% | <5% or 초반만 |
| 트랙별 편차 | 트랙마다 다른 유리 타입 | 한 타입 50%+ |

**워크플로우**: 백테스트 → 가설 → `GameSettingsV4.asset` 수정 → 100~500회 재검증

---

## client (Unity 구현)

**언제 사용**: SPEC 기반 기능 구현, RacerController 수정, 프리팹 생성기 수정, MCP 검증

**도구**: Unity MCP (execute_csharp + read_console), 파일 작성

---

## qa (품질 보증)

**언제 사용**: 구현 완료 후 검증, 회귀 위험 분석, PASS/FAIL 판정

**워크플로우**:
1. TC 실행 (Unity MCP `execute_csharp_script_in_unity_editor`)
2. 결과 보고 → leader에 전달
3. 에러 0건 + 모든 TC PASS 확인

---

## design (UX/UI)

**언제 사용**: UI 설계, 비주얼 일관성 검토, 디자인 스펙 작성

**범위**: 모달·패널 레이아웃, 픽셀아트 톤 일관성, 인터랙션 피드백, 색약 대비

---

## security (보안)

**언제 사용**: 보안 감사, 치팅 방어 설계, 시크릿 관리, `.claude/` 설정 위생, 빌드 하드닝

**원칙**: 클라가 보낸 모든 값은 위조 가능 데이터. 레이스=클라 시뮬 → 서버 재계산 불가 → "완전 차단" 아닌 "캐주얼 차단 + 리버서 지연".

**6대 점검**: 크리덴셜 노출 · 리더보드 통신(HMAC) · 저장 무결성 · 재화·배팅 · `.claude/` 위생 · 빌드 하드닝

**도구**: `/security-audit` 스킬. 등급 CRITICAL/HIGH/MEDIUM/LOW + `파일:라인` + 권고. → [[../시스템/보안_치팅방어]]

## SPEC-028 협업 모델 (다중 에이전트 예시)

```
Phase 시작
  └─ leader → 에이전트 브리프
        ├─ balance (수치 검토)
        ├─ client (Unity 구현)
        └─ qa (TC 실행)
              └─ leader → 오너 게이트 보고
```

→ 자동화 훅: StringTableValidator (컴파일 완료 시), Unity 콘솔 에러 체크, TC 일괄 실행

## 위임 기준

- 기획 모순/헛점 발견 → `leader`
- 백테스트 수치 → `balance`
- Unity 코드/프리팹 수정 → `client`
- 구현 검증 → `qa`
- UI 레이아웃 → `design`
- Steam 스토어 → `marketing`
- 리더보드 백엔드/REST 계약 → `server`
- 보안·치팅 방어·시크릿 → `security`
