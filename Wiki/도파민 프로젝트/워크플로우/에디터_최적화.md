# 에디터 최적화 — Enter Play Mode & Domain Reload

> Play 진입 속도 **4.1초 → 0.8~1.5초** (Domain Reload 비활성화 효과)

---

## Enter Play Mode 설정

| 설정 | 값 | 위치 |
|------|-----|------|
| `enterPlayModeOptionsEnabled` | `true` | `ProjectSettings/EditorSettings.asset` |
| `enterPlayModeOptions` | `DisableDomainReload` (=1) | 동일 |
| `DisableSceneReload` | **미적용** | SceneBootstrapper `Start()` 재진입 감사 선행 필요 |

메뉴: `DopamineRace > Enable Fast Enter Play Mode` (`EnterPlayModeSetup.cs`)

---

## Domain Reload OFF 핵심 주의

Domain Reload가 꺼지면 **static 필드가 Play 세션 간에 유지**된다. 오염 방지를 위해 `[RuntimeInitializeOnLoadMethod(SubsystemRegistration)]`로 리셋해야 한다.

### 실행 순서

```
SubsystemRegistration → BeforeSceneLoad → AfterSceneLoad → RuntimeInitialized
```

- static 필드 리셋: **`SubsystemRegistration`** (가장 이름, Awake 전)
- 씬 오브젝트 필요 시: `BeforeSceneLoad` 이후

---

## 안전화 패턴

### 1. DontDestroyOnLoad 싱글턴 OnDestroy

```csharp
private void OnDestroy()
{
    CancelInvoke();           // SFXManager 등 InvokeRepeating 해제
    StopAllCoroutines();      // 코루틴 정리
    _fetchInFlight = false;   // 인플라이트 플래그 초기화
    if (Instance == this) Instance = null;
}
```

**적용 파일** (2026-06-30): BGMManager · SFXManager · SceneTransitionManager · LeaderboardService · CharacterDatabase · TrackDatabase

### 2. static 필드 SubsystemRegistration 리셋

```csharp
[RuntimeInitializeOnLoadMethod(RuntimeInitializeLoadType.SubsystemRegistration)]
private static void ResetStaticState()
{
    trackPanelOpen = true;
#if UNITY_EDITOR
    if (_circleSprite != null)
    {
        UnityEngine.Object.DestroyImmediate(_circleSprite.texture);
        UnityEngine.Object.DestroyImmediate(_circleSprite);
    }
#endif
    _circleSprite = null;
}
```

**적용 대상**:
| 클래스 | static 필드 | 리셋 이유 |
|--------|-------------|-----------|
| `SceneBootstrapper` | `trackPanelOpen`, `_circleSprite` | 라운드 간 유지값 초기화, Texture2D 누수 방지 |
| `HiddenStatWeights` | `_loaded` | 에디터 CSV 수정 후 재Play 즉시 반영 |

### 3. SteamManager s_everInitialized 리셋

```csharp
[RuntimeInitializeOnLoadMethod(RuntimeInitializeLoadType.SubsystemRegistration)]
private static void ResetForDomainReloadOff()
{
    // Bootstrap()의 BeforeSceneLoad보다 먼저 실행
    if (s_instance == null) s_everInitialized = false;
}
```

> [!warning] P0 버그
> 리셋 없으면 2회차 Play에서 `throw new Exception("[SteamManager] SteamAPI_Init 중복 호출 시도")` 발생.

## 안전한 것 / 위험한 것

| 구분 | 대상 | 이유 |
|------|------|------|
| ✅ 안전 | `OddsCalculator`, `SkillRegistry` | Clear()/null-check 재생성 패턴 |
| ✅ 안전 | `Loc.cs` | `Load()` 호출마다 딕셔너리 재구축 |
| ✅ 안전 | `WalletManager`, `ScoreManager` | DontDestroyOnLoad 없음, 씬 파괴 시 자동 소멸 |
| ❌ 위험 | `SteamManager.s_everInitialized` | → SubsystemRegistration 리셋 필수 |
| ⚠️ 주의 | `SceneBootstrapper._circleSprite` | Texture2D 참조 잔류 → DestroyImmediate 필요 |

---

## DestroyImmediate 규칙

```csharp
#if UNITY_EDITOR
    UnityEngine.Object.DestroyImmediate(sprite.texture);
    UnityEngine.Object.DestroyImmediate(sprite);
#endif
```

- 빌드에서 `DestroyImmediate` 금지 → `#if UNITY_EDITOR` 필수
- Domain Reload 비활성화는 **에디터 전용** — IL2CPP 빌드에 영향 없음

---

## Scene Reload OFF (미적용)

`DisableSceneReload`는 이번 범위 제외. 적용 전 `SceneBootstrapper.Start()` 재진입 안전성 별도 감사 필요.
